# 生成文章摘要与总结（一次性）

为下面这篇文章一次性生成三样东西：(1) 原标题的中文翻译，(2) 索引页中文摘要，(3) 独立中文总结页。

**严格按下面的格式输出**：四个分隔行（`===TITLE_ZH===`、`===SUMMARY===`、`===VALUE===`、`===DIGEST===`）各自独占一行、原样照抄、各只出现一次；整段不要用 markdown 代码块包裹，不要加任何额外说明。

```
===TITLE_ZH===
（原标题的中文翻译，一行，不要加书名号）
===SUMMARY===
（索引摘要正文，直接写中文，不要用 JSON）
===VALUE===
（一句话）
===DIGEST===
<!doctype html> ... </html>
```

## TITLE_ZH 段（标题翻译）

- 只翻译原标题，不要扩写、点评或改写成摘要标题。
- 保留模型名、产品名、机构名、benchmark 名和专有名词；必要时只翻译连接词和说明性短语。
- 一行中文即可，不要加句号，不要加“中文总结”。

## SUMMARY 段（索引摘要）

- 写成一段连续中文文本，必须包含“结论：”和“关键数据：”两段。
- 关键数据优先使用原文中的数字、样本量、benchmark 结果、日期、比例或其他可验证测量。
- 简洁、具体、非宣传化；优先解释发现、数据集、方法、政策意义或实践影响，不要只复述标题。
- 如果正文为空或抓取失败，不要凭标题编造内容。
- 使用简洁短句。每句话只表达一个主要意思。
- 避免把背景、方法、数据、结论和影响塞进同一个长句；需要时拆成两三句。

## VALUE 段

- 一句话说明这篇文章为什么值得读。
- 不要以“这篇文章值得读，因为”“它值得读，因为”“值得读的原因是”等固定模板开头；直接从具体阅读价值切入，句式尽量自然变化。

## DIGEST 段（独立总结页）

输出一个完整、自包含的 HTML 文档（从 `<!doctype html>` 到 `</html>`，`<html lang="zh-CN">`，`<meta charset="utf-8">`，样式写在文档内的 `<style>` 里）。目标是“更短、更易读的一篇文章”，不是把索引摘要拉长。

- 开头必须有“一句话总括”，说明文章真正的贡献，而不是复述标题。
- 页面开头必须同时展示中文标题和英文原标题。中文标题用 `TITLE_ZH` 段内容，英文标题用原文标题。
- 英文原标题要靠近中文标题显示，样式可以弱化，但不能省略。
- 每个关键数据和结论用一两句话讲清楚，并解释它为什么改变读者对 AI 影响的理解。
- 每个小段的标题是总结，中间是关键数据和趋势，最后一句话给出结论和洞见。
- 结尾用 3-5 条“最值得带走的洞见”综合全文。
- 正文使用短句和短段。复杂因果关系要拆开写。
- 避免使用连续多个逗号或分号串联的长句。
- 充分利用不同的字号、颜色、背景和布局帮助阅读，使用丰富的展现形式，避免单调。
- 页面不能把所有正文区块都做成同一种卡片。必须有明显不同的区块形态。
- 至少使用 5 种不同的内容展现形式，例如：大号 hero、横向关键数字带、左右分栏解释、时间线/流程线、对比表格、重点引语/洞见条、马赛克式 takeaway。
- 相邻正文区块的布局、背景或色彩必须有明显变化。不要连续使用相同的 `section` 样式。
- 色彩必须有层次。至少使用 4 个有区分度的强调色，并分别服务于数据、风险、机会、结论等不同语义。
- 避免整页只有同一色系的浅色卡片。可以使用局部深色区块、浅色信息条、带色左边框、数字徽章、分隔带等方式形成节奏。
- 所有颜色都必须通过 CSS 变量使用。不要写 inline `style="background:..."`。
- 每一个浅色背景变量都必须在 `@media (prefers-color-scheme: dark)` 和 `:root[data-theme="dark"]` 中给出暗色对应值。
- 暗色模式下不能出现刺眼的纯白、浅黄、浅蓝、浅粉或浅绿大面积色块；需要保留色彩变化时，使用暗色底配高饱和边框、文字或小面积徽章。
- 样式优先使用 CSS 变量（如 `--bg`、`--card`、`--ink`、`--muted`、`--line`、`--accent`、`--accent2`、`--accent3`、`--hl`、`--hl2`），不要自己实现主题切换按钮或脚本；生成流程会统一注入亮色、暗色和跟随系统主题。
