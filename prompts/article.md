# 生成文章摘要与总结（一次性）

为下面这篇文章一次性生成两样东西：(1) 索引页中文摘要，(2) 独立中文总结页。

**严格按下面的格式输出**，用两个分隔行隔开。分隔行各自独占一行、原样照抄，整段不要用 markdown 代码块包裹、不要加任何额外说明：

```
===SUMMARY===
{"summary_zh": "...", "value_zh": "..."}
===DIGEST===
<!doctype html> ... </html>
```

## SUMMARY 段（输出一个 JSON 对象）

- `summary_zh` 必须包含“结论：”和“关键数据：”两段。关键数据优先使用原文中的数字、样本量、benchmark 结果、日期、比例或其他可验证测量。
- `value_zh` 用一句话说明这篇文章为什么值得读。
- 简洁、具体、非宣传化；优先解释发现、数据集、方法、政策意义或实践影响，不要只是复述标题。
- 如果正文为空或抓取失败，不要凭标题编造内容。

## DIGEST 段（输出一个完整、自包含的 HTML 文档）

目标是“更短、更易读的一篇文章”，不是把索引摘要拉长。

- 从 `<!doctype html>` 到 `</html>`，`<html lang="zh-CN">`，`<meta charset="utf-8">`，样式写在文档内的 `<style>` 里。
- 开头必须有“一句话总括”，说明文章真正的贡献，而不是复述标题。
- 每个关键数据和结论用一两句话讲清楚，并解释它为什么改变读者对 AI 影响的理解。
- 每个小段的标题是总结，中间是关键数据和趋势，最后一句话给出结论和洞见。
- 结尾用 3-5 条“最值得带走的洞见”综合全文。
- 充分利用字号、颜色、背景和布局帮助阅读。
