GPT-5.4 的真正变化：从模型能力到可执行工作流

一句话总括：GPT-5.4 的贡献在于把推理、编码、电脑操作、工具搜索和长上下文整合成更可靠的“专业工作代理”，让 AI 更接近在真实软件环境中完成端到端任务。

专业知识工作

它把“生成答案”推进到“交付工作成品”

GPT-5.4 在 GDPval 中达到 83.0% wins or ties，高于 GPT-5.2 的 70.9%。这个指标覆盖 44 个职业和美国 GDP 贡献最高的 9 个行业，任务包括销售演示、会计表格、排班、制造图和短视频等真实工作产物。

内部投行表格建模任务中，GPT-5.4 得分 87.3%，而 GPT-5.2 为 68.4%；演示文稿评测中，人类 68.0% 的时间更偏好 GPT-5.4 输出。结论是，模型升级的重点正在从知识问答转向可审阅、可交付、可迭代的办公成果。

事实性

更强的专业能力必须同时降低错误成本

OpenAI 称，在用户标记过事实错误的去标识化提示集合上，GPT-5.4 的单条 claim 相对 GPT-5.2 少 33% 虚假，完整回答含错率少 18%。这类改进对法律、财务、研究和文档密集型场景尤其重要，因为错误不是小瑕疵，而会直接影响交付可信度。

这改变了读者对 AI 影响的理解：更有价值的模型不是更会“自信表达”，而是能在复杂材料中维持准确性，并减少人工复核负担。

电脑使用与视觉

原生电脑使用让代理能力进入真实界面

GPT-5.4 是 OpenAI 首个具备原生计算机使用能力的通用模型。它可以通过截图、鼠标键盘动作、Playwright 等方式操作网页和软件，并允许开发者通过确认策略调整不同风险级别的安全行为。

工具生态

工具搜索解决了大工具集的上下文浪费

过去，系统会把所有工具定义一次性塞进提示词；当工具很多时，这会带来数千甚至数万 token 的成本和上下文污染。GPT-5.4 的 tool search 改为先给轻量工具列表，模型需要时再检索具体工具定义。

在 250 个 MCP Atlas 任务、36 个 MCP server 全开的测试中，工具搜索配置减少 47% 总 token 使用量，同时保持相同准确率。这个结果的含义是，大规模工具生态不再只能靠更长上下文硬塞进去，而可以靠按需检索降低成本与延迟。

编码与网页研究

编码、搜索和长流程能力被合并进同一个主线模型

GPT-5.4 在 SWE-Bench Pro Public 达到 57.7%，略高于 GPT-5.3-Codex 的 56.8% 和 GPT-5.2 的 55.6%。它还支持 Codex 中实验性的 1M token 上下文，适合长时间规划、执行和验证任务。

网页研究方面，GPT-5.4 在 BrowseComp 达到 82.7%，高于 GPT-5.2 的 65.8%；GPT-5.4 Pro 达到 89.3%。这表明模型不仅更会写代码，也更能在多轮搜索中坚持查找难定位信息，并整合成可用答案。

部署与成本

价格更高，但目标是用效率抵消总成本

GPT-5.4 已在 ChatGPT、API 和 Codex 推出，API 模型名为 gpt-5.4，Pro 版本为 gpt-5.4-pro。ChatGPT Plus、Team、Pro 用户可使用 GPT-5.4 Thinking，GPT-5.2 Thinking 将在 Legacy Models 中保留三个月，并于 2026 年 6 月 5 日退役。

API 标准价格方面，gpt-5.4 为输入 $2.50/M tokens、缓存输入 $0.25/M tokens、输出 $15/M tokens；gpt-5.4-pro 为输入 $30/M tokens、输出 $180/M tokens。结论是，OpenAI 把 GPT-5.4 作为更贵但更省步骤、更省 token 的专业工作模型来定位。

安全

更强代理能力也带来更高网络安全要求

OpenAI 继续按 Preparedness Framework 中的 High cyber capability 部署 GPT-5.4，并配套网络安全监控、可信访问控制，以及部分 ZDR 场景的请求级阻断。文章也承认分类器仍可能带来误报，并会继续校准拒绝和拦截策略。

同时，OpenAI 发布 CoT controllability 开源评测，用于衡量模型是否能故意隐藏推理以逃避监控；文章称 GPT-5.4 Thinking 控制自身 CoT 的能力较低。这里的关键洞见是，代理越能操作真实系统，安全设计越需要进入模型能力本身和部署流程。

最值得带走的洞见

  • GPT-5.4 的核心不是单一榜单领先，而是把编码、办公文档、网页研究、电脑操作和工具调用合成一个面向真实工作的模型。
  • GDPval、表格建模、演示文稿和事实性数据说明，AI 对知识工作的影响正在从辅助写作转向交付可用成果。
  • 原生电脑使用和 1M 上下文让代理能处理更长链路，但也让验证、权限和安全确认变得更重要。
  • 工具搜索的 47% token 降幅提示未来代理系统的关键架构：不是把所有工具塞给模型，而是让模型按需发现工具。
  • GPT-5.4 更贵，但 OpenAI 的论点是通过更少轮次、更少 token、更高成功率来降低整体工作成本。
