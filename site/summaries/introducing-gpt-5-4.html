Introducing GPT-5.4

Original title: Introducing GPT-5.4

GPT-5.4 的真正贡献，是把推理、编码、视觉、电脑操作、工具选择和长上下文整合成一个更能完成专业工作的代理模型。

它不是只让回答更聪明。它试图让模型更少来回确认，更会调用工具，更能在真实软件环境里执行、检查和交付。

专业工作能力的变化，不只是跑分上涨

GPT-5.4 在 GDPval 上达到 83.0%。这个评测覆盖 44 个职业，任务包括销售演示、会计表格、排班、制造图和短视频。

这改变了读者对 AI 影响的理解：模型正在从“写答案”进入“产出可交付物”。评估对象也从单题正确率转向真实工作产品。

文档、表格、演示 是文章反复强调的核心场景。

内部投行建模任务中，GPT-5.4 平均得分 87.3%，GPT-5.2 为 68.4%。演示文稿评估中，人类 68.0% 的时间偏好 GPT-5.4 的输出。

这说明改进集中在专业交付的完整度、审美和结构稳定性上。

代理工作流被拆成四个能力环节

1

规划

ChatGPT 中的 GPT-5.4 Thinking 可先给出工作计划。用户可以在生成途中调整方向。

2

执行

API 和 Codex 中加入原生电脑使用能力。模型可以通过截图、鼠标、键盘和 Playwright 完成任务。

3

找工具

工具搜索让模型不必一次塞入全部工具定义。需要时再查工具，可降低成本和上下文压力。

4

验证

更长上下文和更强视觉能力让模型能跨长流程保持状态，并检查页面、文档和软件环境。

“工具搜索”的意义，是让模型面对大型工具生态时不再被全部说明书淹没。 在 250 个 MCP Atlas 任务、36 个 MCP server 上，工具搜索减少 47% 总 token 用量，同时保持相同准确率。

安全和可用性也说明模型走向更高风险场景

OpenAI 将 GPT-5.4 按 Preparedness Framework 视为 High cyber capability。这意味着更强能力也带来更严格的监控、访问控制和高风险请求处理。

文章还提到 CoT controllability 评估。GPT-5.4 Thinking 控制或隐藏推理链的能力较低，被视为有利于安全监控的性质。

可用性：GPT-5.4 在 ChatGPT、API 和 Codex 推出。API 模型名为 gpt-5.4，Pro 版本为 gpt-5.4-pro。
迁移：GPT-5.2 Thinking 会在付费用户的 Legacy Models 中保留三个月，并于 2026 年 6 月 5 日退役。

最值得带走的洞见

1. 代理式 AI 的瓶颈正在从模型智力转向工作流完整性。

GPT-5.4 强调电脑使用、工具搜索、长上下文和验证。它要解决的是“能否完成任务”，不只是“能否回答问题”。

2. 专业交付物成为新战场。

表格、演示和文档的评估被放到核心位置。AI 产品价值更接近办公室里的实际产出。

3. 工具搜索会影响成本结构。

大型 MCP 生态下，少放工具定义本身就是能力提升。省 token 也能降低延迟。

4. 视觉和电脑使用让模型更接近操作系统层代理。

OSWorld 75.0% 是重要信号。它意味着模型开始能在界面中看、点、输入和修正。

5. 更强模型也需要更明确的安全边界。

High cyber capability 的部署方式说明，越能执行真实任务，越需要细粒度确认和风险控制。
