推出 GPT-5.4

Introducing GPT-5.4

一句话总括：GPT‑5.4 的真正贡献，是把知识工作、编程、计算机操作和工具调用整合成一个更可靠的通用工作代理，同时降低完成复杂任务所需的交互和 token。

专业工作能力明显增强

GPT‑5.4 在覆盖 44 种职业的 GDPval 中达到 83.0%，高于 GPT‑5.2 的 70.9%。这意味着提升不只发生在问答中，也体现在演示文稿、财务模型、排班表和制造图等实际交付物上。

内部投行建模评测达到 87.3%。演示文稿盲评中，人类评审有 68.0% 的时候更偏好 GPT‑5.4。结论是，模型开始更稳定地处理带有格式、审美和业务约束的成品工作。

通用模型获得原生计算机操作能力

GPT‑5.4 在 OSWorld‑Verified 上达到 75.0%，超过 GPT‑5.2 的 47.3%，也略高于报告中的人类水平 72.4%。它既能编写 Playwright 代码，也能根据截图发送鼠标和键盘操作。

原始图像细节模式最高支持 1024 万像素或最长边 6000 像素。视觉提升因此会影响网页操作、文档解析和高分辨率界面定位，而不只是图像问答。

工具搜索改变了代理的扩展方式

过去：全部预载

系统会把所有工具定义提前放进提示词。大型 MCP 服务可能因此占用数万 token。

现在：按需查找

模型先接收轻量工具清单。真正需要某项工具时，再查找并载入完整定义。

结果：成本下降

在 250 项 MCP Atlas 任务中，准确率不变，总 token 用量减少 47%。这使更大的工具生态变得可行。

更强，也更可控

GPT‑5.4 Thinking 会先给出工作计划，并允许用户在回答生成期间调整方向。它还能在长时间推理中保持更完整的上下文。

事实性也有所改善。与 GPT‑5.2 相比，单项事实陈述为假的概率下降 33%，整份回答包含错误的概率下降 18%。这不会消除幻觉，但会提高专业工作流中初稿的可靠性。

安全方面，OpenAI 仍将其视为具有 High cyber capability。部署措施包括监控、可信访问控制，以及面向部分 ZDR 请求的异步阻断。

价格上涨，效率是否抵消成本取决于任务

API 标准价格为每百万输入 token $2.50、缓存输入 $0.25、输出 $15。这些价格高于 GPT‑5.2。OpenAI 的判断是，更高的 token 效率可以减少许多任务的总用量，但文章没有承诺所有工作负载都会更便宜。

Codex 中的 100 万 token 上下文仍属实验功能。超过标准 272K 上下文的请求，会按正常速率的 2 倍计入使用限额。

工具规模不必等于上下文负担按需载入工具定义，让代理可以接触更大的 MCP 生态，同时保留缓存并减少 token。
基准领先不等于所有场景领先GPT‑5.4 在 Terminal‑Bench 2.0 等个别项目上仍低于 GPT‑5.3‑Codex。长上下文准确率也会随长度明显下降。
可引导性是产品层面的重要变化用户可以在模型工作期间修正方向。复杂任务不必总以中止、重写提示词和重新开始为代价。

今天，我们在 ChatGPT、API 和 Codex 中发布 GPT‑5.4。在 ChatGPT 中，它的名称是 GPT‑5.4 Thinking。这是我们目前面向专业工作能力最强、效率最高的前沿模型。我们还在 ChatGPT 和 API 中发布 GPT‑5.4 Pro，供希望在复杂任务中获得最佳性能的用户使用。

GPT‑5.4 将我们近期在推理、编程和代理式工作流方面的重要进展整合进一个前沿模型。它继承了 GPT‑5.3‑Codex 领先业界的编程能力，并改善了模型使用工具和软件环境的方式。它也更擅长处理电子表格、演示文稿和文档等专业任务。最终得到的模型能更准确、高效地完成复杂的实际工作。它用更少的来回沟通，交付用户真正要求的结果。

在 ChatGPT 中，GPT‑5.4 Thinking 现在可以预先说明自己的工作计划。模型工作期间，用户可以调整方向。这样无需增加额外轮次，也能让最终结果更贴近实际需要。GPT‑5.4 Thinking 还增强了深度网络研究能力，尤其适合非常具体的查询。对于需要长时间思考的问题，它也能更好地保持上下文。这些改进让回答质量更高、生成更快，也更贴合当前任务。

在 Codex 和 API 中，GPT‑5.4 是我们发布的首个具备原生先进计算机操作能力的通用模型。它让代理能够操作计算机，并跨应用完成复杂工作流。模型支持最长 100 万 token 的上下文，因此代理可以规划、执行并验证持续时间很长的任务。GPT‑5.4 还通过工具搜索改善了模型面对大型工具和连接器生态时的工作方式。代理可以更高效地找到并使用正确工具，同时不牺牲智能水平。最后，GPT‑5.4 也是我们目前 token 效率最高的推理模型。与 GPT‑5.2 相比，它解决问题所需的 token 显著减少，因此能够降低 token 用量并提升速度。

结合通用推理、编程和专业知识工作的进步，GPT‑5.4 能够支持更可靠的代理、更快的开发流程，以及 ChatGPT、API 和 Codex 中质量更高的输出。

知识工作

GPT‑5.4 建立在 GPT‑5.2 的通用推理能力之上。对于专业人士关心的真实任务，它能给出更稳定、更完善的结果。

GDPval 用于测试代理在 44 种职业中完成明确知识工作的能力。在这项评测中，GPT‑5.4 创下新的最佳成绩。在 83.0% 的对比中，它的表现达到或超过行业专业人士。GPT‑5.2 的对应结果为 70.9%。

GDPval 中的任务覆盖美国 GDP 贡献最高的九大行业，共涉及 44 种职业。模型需要完成真实的工作成果，例如销售演示文稿、会计电子表格、急诊护理排班表、制造图纸或短视频。GPT‑5.4 的推理强度设为 xhigh。GPT‑5.2 设为 heavy，在 ChatGPT 中对应的强度略低。

“GPT‑5.4 是我们测试过的最佳模型。它现在位居 APEX‑Agents benchmark 榜首。这项 benchmark 用于衡量模型完成专业服务工作的表现。它尤其擅长制作需要较长周期才能完成的成果，例如幻灯片、财务模型和法律分析。它不仅取得领先表现，运行速度也更快，成本还低于其他有竞争力的前沿模型。”

我们特别重视提高 GPT‑5.4 创建和编辑电子表格、演示文稿及文档的能力。在一项内部电子表格建模 benchmark 中，任务与初级投资银行分析师可能完成的工作相似。GPT‑5.4 的平均得分为 87.3%，GPT‑5.2 为 68.4%。在一组演示文稿评测提示词上，人类评审有 68.0% 的时候更偏好 GPT‑5.4 的演示文稿。原因包括更好的美感、更丰富的视觉变化，以及更有效地运用图像生成。

生成文档时，推理强度设为 xhigh。

你可以在 ChatGPT 中使用 GPT‑5.4 Thinking 或 Pro 体验这些能力。如果你是 Enterprise 客户，我们建议使用新发布的 ChatGPT for Excel 加载项。它也在今天正式推出。我们还更新了 Codex 和 API 中提供的电子表格技能与演示文稿技能。

为了让 GPT‑5.4 更适合真实工作，我们继续减少幻觉和错误。GPT‑5.4 是我们目前事实性最好的模型。在一组经过身份信息移除、且用户曾标记事实错误的提示词上，与 GPT‑5.2 相比，GPT‑5.4 的单项陈述为假的概率降低 33%。整份回答包含任何错误的概率降低 18%。

“GPT‑5.4 为文档密集型法律工作树立了新标准。它在 BigLaw Bench 评测中得到 91%。与其他模型相比，GPT‑5.4 目前更擅长组织复杂的交易分析，在长篇合同中保持准确，并提供法律从业者需要的高细节水平。”

计算机操作与视觉

GPT‑5.4 是我们首个原生支持计算机操作的通用模型。这对开发者和代理都是重要进展。对于需要构建代理，让其跨网站和软件系统完成真实任务的开发者，它是目前能力最强的可用模型。

我们让 GPT‑5.4 能够在广泛的计算机操作任务中保持良好表现。它很擅长编写代码，通过 Playwright 等库操作计算机。它也能根据截图发出鼠标和键盘指令。开发者可以通过 developer message 引导其行为，让模型适应具体场景。开发者甚至可以指定自定义确认策略，让模型的安全行为适应不同的风险容忍度。

多项覆盖不同环境的计算机操作 benchmark 展示了模型的性能和灵活性。OSWorld‑Verified 衡量模型通过截图及键盘、鼠标操作来使用桌面环境的能力。GPT‑5.4 在这项评测中取得新的最佳成绩，成功率达到 75.0%。这远高于 GPT‑5.2 的 47.3%，也超过 72.4% 的人类表现。1

WebArena‑Verified 用于测试浏览器操作。GPT‑5.4 同时使用 DOM 和截图驱动的交互时，成功率达到领先的 67.3%。GPT‑5.2 为 65.4%。Online‑Mind2Web 同样测试浏览器操作。GPT‑5.4 仅使用截图作为观察信息时，成功率达到 92.8%。ChatGPT Atlas 的 Agent Mode 为 70.9%。

工具让出（tool yield）是指助手暂停执行，等待工具返回结果。如果先并行调用三个工具，随后再并行调用三个工具，让出次数就是两次。与工具调用次数相比，工具让出次数更能反映延迟，因为它体现了并行执行带来的收益。

GPT‑5.4 会解读浏览器界面截图，并通过基于坐标的点击与界面元素交互，从而发送电子邮件并安排日历事件。视频没有加速。

GPT‑5.4 计算机操作能力的提升，建立在更强的通用视觉感知能力之上。MMMU‑Pro 测试模型的视觉理解和推理能力。GPT‑5.4 在不使用工具时达到 81.2%，高于 GPT‑5.2 的 79.5%。视觉感知的提升也带来了更好的文档解析能力。在 OmniDocBench 上，GPT‑5.4 不使用推理强度时的平均错误率为 0.109。该指标通过模型预测与标准答案之间的归一化编辑距离计算。GPT‑5.2 的结果为 0.140。

MMMU‑Pro 的推理强度设为 xhigh。OmniDocBench 的推理强度设为 none，用于反映低成本、低延迟条件下的性能。

我们还在改善模型对高密度、高分辨率图像的理解能力。这类图像需要保留完整细节。从 GPT‑5.4 开始，我们引入 original 图像输入细节级别。它支持最高 1024 万总像素或最长边 6000 像素，以先达到的限制为准。high 图像输入细节级别现在支持最高 256 万总像素或最长边 2048 像素。在 API 用户的早期测试中，original 和 high 细节级别显著提升了定位能力、图像理解和点击准确率。

“在我们覆盖约 3 万个业主协会和房产税门户网站的计算机操作评测中，GPT‑5.4 首次尝试的成功率达到 95%，三次尝试内达到 100%。此前的 CUA 模型约为 73% 至 79%。它完成会话的速度约快三倍，使用的 token 约少 70%。这显著提高了大规模使用时的可靠性和成本效率。”

开发者可以在 API 中通过更新后的 computer 工具使用这些能力。推荐的最佳实践请参阅更新后的文档。

编程

GPT‑5.4 将 GPT‑5.3‑Codex 的编程优势与领先的知识工作和计算机操作能力结合起来。对于需要模型使用工具、反复迭代并减少人工干预的长时间任务，这些能力尤为重要。GPT‑5.4 在 SWE‑Bench Pro 上达到或超过 GPT‑5.3‑Codex，同时在不同推理强度下拥有更低延迟。

我们通过观察模型的线上行为并进行离线模拟来估算延迟。估算包含工具调用耗时，也就是代码执行时间，以及生成 token 和输入 token。真实延迟可能有很大差异，并取决于模拟没有覆盖的许多因素。评测遍历了从 none 到 xhigh 的各种推理强度。

启用后，Codex 中的 /fast 模式可让 GPT‑5.4 的 token 生成速度最高提升至 1.5 倍。它使用相同模型，智能水平也不变，只是运行更快。这让用户能持续推进编程、迭代和调试工作，不轻易打断工作节奏。开发者可以在 API 中使用 Priority processing，以相同的高速访问 GPT‑5.4。

在评测和内部测试中，我们发现 GPT‑5.4 很擅长复杂前端任务。与此前发布的所有模型相比，它生成的结果明显更美观，功能也更完整。

为了展示增强后的计算机操作能力与编程能力如何协同工作，我们还发布了一项名为“Playwright (Interactive)”的实验性 Codex 技能。Codex 可以借此以视觉方式调试网页和 Electron 应用。它甚至可以一边构建应用，一边测试应用。

这是一个由 GPT‑5.4 根据一条简略提示词制作的主题公园模拟游戏。模型使用 Playwright Interactive 在浏览器中进行试玩，并通过图像生成制作等距视角资源。模拟内容包括基于网格的道路铺设、游乐设施与景观建设、游客寻路、排队和游乐设施运行周期。公园的资金、游客数量、幸福度、清洁度和评分会随着布局效果及游客反应而升降。Playwright 自动完成了浏览器试玩。它会建设并扩展公园，放置和移除道路与设施，检查镜头移动，并在多轮游玩中验证游客、队列、设施状态和界面指标是否正确更新。

工具使用

GPT‑5.4 显著改善了模型使用外部工具的方式。代理现在可以在更大的工具生态中运行，更可靠地选择正确工具，并以更低的成本和延迟完成多步骤工作流。

工具搜索

GPT‑5.4 在 API 中引入工具搜索。模型面对大量工具时，可以借此高效工作。

过去，向模型提供工具时，所有工具定义都会提前放入提示词。对于拥有大量工具的系统，每次请求可能因此多出数千甚至数万 token。这不仅提高成本、拖慢响应，还会让上下文塞满模型可能根本用不到的信息。

采用工具搜索后，GPT‑5.4 会先接收一份轻量的可用工具列表，以及用于搜索工具的能力。当模型需要使用某项工具时，它可以查找该工具的定义，并在此时把定义加入对话。

这种方法会大幅减少工具密集型工作流所需的 token，同时保留缓存，让请求更快、更便宜。它还让代理能够可靠地使用规模大得多的工具生态。MCP 服务可能包含数万 token 的工具定义，因此效率提升可能十分可观。

为了展示效率提升，我们从 Scale 的 MCP Atlas benchmark 中选取 250 项任务，并启用全部 36 个 MCP 服务。评测采用两种模式。第一种模式将所有 MCP 函数直接放入模型上下文。第二种模式把所有 MCP 服务放在工具搜索之后。工具搜索配置在保持相同准确率的同时，将总 token 用量降低了 47%。

示例 token 数量来自 MCP‑Atlas 公开数据集中 250 项任务的平均值。

代理式工具调用

GPT‑5.4 还改进了工具调用。它在推理过程中能更准确、更高效地判断何时使用工具，以及应当如何使用工具。这项改进在 API 中尤其明显。与 GPT‑5.2 相比，它在 Toolathlon 上用更少轮次取得了更高准确率。Toolathlon 用于测试 AI 代理能否使用真实工具和 API 完成多步骤任务。例如，代理可能需要读取电子邮件，提取作业附件并上传，对作业评分，再把结果记入电子表格。

工具让出是指助手暂停执行，等待工具返回结果。如果先并行调用三个工具，随后再并行调用三个工具，让出次数就是两次。与工具调用次数相比，工具让出次数更能反映延迟，因为它体现了并行执行带来的收益。

对于注重延迟，并且倾向使用 None 推理强度的场景，GPT‑5.4 也进一步超过了此前的模型。

在 τ2‑bench 中，模型必须使用工具完成一项客户服务任务。模拟用户可能会与模型沟通，也可能对环境状态采取操作。评测中的推理强度设为 None。

改进的网络搜索

GPT‑5.4 更擅长代理式网络搜索。在 BrowseComp 上，它比 GPT‑5.2 绝对提升 17 个百分点。BrowseComp 衡量 AI 代理能否持续浏览网络，找到难以定位的信息。GPT‑5.4 Pro 在这项评测中以 89.3% 创下新的最佳成绩。

在实际使用中，这意味着 GPT‑5.4 Thinking 更擅长回答需要整合多个网络来源的问题。它可以进行更多轮搜索，更持续地寻找最相关的信息源。面对“大海捞针”式的问题时，这项能力尤其有用。随后，它还能把找到的信息综合成清晰、论证充分的回答。

在 BrowseComp 中，我们使用搜索屏蔽列表，排除包含 benchmark 答案的网站，以防止污染并保证评测公平。GPT‑5.4 的测量日期晚于 GPT‑5.2，因此得分变化同时反映了模型、搜索系统和互联网状态的变化。GPT‑5.4 使用的屏蔽列表更长，也经过了更新。模型使用 ChatGPT 搜索工具，它与 API 搜索可能存在少量差异。

“GPT‑5.4 xhigh 是多步骤工具使用领域新的最佳模型。Zapier 运行着业界一些最严格的工具使用 benchmark，覆盖数百项高级真实工作流。此前的模型会中途放弃，而 GPT‑5.4 能把工作完成。它是迄今最有坚持力的模型。”

可引导性

与 Codex 开始工作时说明处理方法的方式相似，ChatGPT 中的 GPT‑5.4 Thinking 现在会在处理较长、较复杂的查询时先给出工作说明。用户还可以在回答生成期间添加指令或调整方向。这样更容易把模型引导到准确的目标上，无需从头开始，也不必增加多轮对话。该功能现已在 chatgpt.com 和 Android 应用中推出，稍后将登陆 iOS 应用。

面对困难任务时，模型还能思考更长时间，并更清楚地记住对话中的早期步骤。因此，它可以处理更长的工作流和更复杂的提示词，同时让回答始终保持连贯和相关。

安全

过去几个月中，我们在准备部署 GPT‑5.4 的同时，继续改进随 GPT‑5.3‑Codex 引入的防护措施。与 GPT‑5.3‑Codex 类似，我们依据 Preparedness Framework，将 GPT‑5.4 视为具有 High cyber capability，并按照 system card 中的说明部署相应保护。措施包括扩展后的网络安全防护体系，例如监控系统、可信访问控制，以及面向 Zero Data Retention（ZDR）环境中高风险请求的异步阻断。我们也会继续投资更广泛的安全生态。

网络安全能力天然具有双重用途。因此，我们在部署时继续采取审慎方法，并持续校准政策和分类器。对于 ZDR 环境中的部分客户，按请求执行阻断仍是网络风险缓解体系的一部分。分类器仍在改进，因此在完善防护期间可能出现误判。这些更新旨在改善防护措施的实际运行效果。具体目标包括减少不必要的拒绝和过度保留的回答，同时维持对滥用行为的有力防护。

我们继续研究思维链（Chain‑of‑Thought，CoT）的可监控性，以便更好地理解模型如何推理，并帮助发现潜在的不当行为。作为这项工作的一部分，我们推出了一项新的开源评测 CoT controllability。它衡量模型能否主动模糊自己的推理过程，从而逃避监控。我们发现，GPT‑5.4 Thinking 控制其 CoT 的能力较低。从安全角度看，这是积极特征。它表明模型缺乏隐藏自身推理的能力，CoT 监控仍然是一种有效的安全工具。

可用范围与价格

GPT‑5.4 从今天起在 ChatGPT 和 Codex 中逐步推出。在 API 中，GPT‑5.4 已以 gpt-5.4 的名称开放。GPT‑5.4 Pro 也已在 API 中以 gpt-5.4-pro 的名称开放，供需要在最复杂任务上获得最佳性能的开发者使用。

在 ChatGPT 中，GPT‑5.4 Thinking 从今天起向 ChatGPT Plus、Team 和 Pro 用户开放，并取代 GPT‑5.2 Thinking。付费用户仍可在模型选择器的 Legacy Models 区域使用 GPT‑5.2 Thinking，保留期为三个月。它将在 2026 年 6 月 5 日退役。Enterprise 和 Edu 方案的管理员可以在设置中启用抢先体验。GPT‑5.4 Pro 向 Pro 和 Enterprise 方案开放。ChatGPT 中 GPT‑5.4 Thinking 的上下文窗口与 GPT‑5.2 Thinking 相同，没有变化。

GPT‑5.4 是我们的首个主线推理模型。它整合了 GPT‑5.3‑codex 的前沿编程能力，并同时在 ChatGPT、API 和 Codex 中推出。我们将其命名为 GPT‑5.4，以体现这次跨越，也让用户在 Codex 中选择模型时更简单。未来，Instant 模型和 Thinking 模型可能会以不同速度演进。

Codex 中的 GPT‑5.4 实验性支持 100 万 token 上下文窗口。开发者可以通过配置 model_context_window 和 model_auto_compact_token_limit 进行尝试。请求一旦超过标准的 272K 上下文窗口，就会按正常速率的两倍计入使用限额。

在 API 中，GPT‑5.4 的每 token 价格高于 GPT‑5.2，以体现其能力提升。同时，更高的 token 效率可以减少许多任务所需的总 token 数。Batch 和 Flex 的价格为标准 API 费率的一半。Priority processing 的价格为标准 API 费率的两倍。
