用旧模型自举新模型：Composer autoinstall 如何把"烂环境"变成训练燃料

作者：Shomil Jain、Joshua Warner、Andrew Zhai · 原文链接
一句话总括：Cursor 让上一代 Composer 模型自动为下一代的 RL 训练搭建可运行环境——这不是"把仓库 clone 下来"那么简单，而是模型主动安装依赖、mock 缺失组件、反复测试直到环境真能跑通，从而把"环境配置"这道训练瓶颈变成了一条会自我增强的复利链路。

为什么重要坏环境不只是慢，而是直接烧掉算力

RL 训练需要"能跑起来"的环境。如果环境一开始就是坏的，模型只会把 token 浪费在调试配置上，而不是学习解决问题；最糟的情况下，一个坏环境会让问题根本无解，于是这一整段算力拿不到任何奖励信号、彻底白烧。

所以环境质量不是工程细节，而是直接决定训练信号好坏的关键变量。autoinstall 正是为了把这个变量稳定下来。

把"搭环境"自动化做扎实，等于直接提高了每一份训练算力的有效产出。

怎么做一个 agent 定义"成功"，另一个 agent 去实现它

autoinstall 分两个阶段，分工明确：

  1. 目标设定：第一个 agent 拿到固定 checkout 的代码库，探索 readme/makefile，尝试 uv、clippy 等语言工具，提出 10 条命令以及"环境配好后应该输出什么"的描述。
  2. 实际搭建：第二个 agent 从这 10 条里拿到选出的 3 条目标命令，反复调用工具把环境搭起来，直到三条都能跑通且输出匹配第一阶段的预期。
  3. 失败即重试：不达标就重启第二阶段，最多 5 次仍搭不好，就直接丢弃这个环境。
用"先定义验收标准、再让另一个模型去满足"的双 agent 结构，把模糊的"配好环境"变成了可自动判定的闭环。

关键能力配不通就自己造：mock 一切缺失的东西

为了让命令真正跑起来，Composer 不会停在"装不上就放弃"。它会主动 mock 缺失文件、生成占位图片、甚至建假的数据库表；遇到测试需要的外部组件，它会用 MinIO 配置替代 S3、用 Docker 容器补上缺失的 sidecar，并生成启动脚本来在 RL 使用前拉起这些长驻进程。

mock 缺失文件 占位图片 假数据库表 MinIO 代替 S3 Docker sidecar 启动脚本
现代编码模型愿意"为了把环境跑通而想尽办法"，这种主动性是 autoinstall 能处理真实复杂项目的前提。

真实案例给区块链项目 Celo 配环境：上网查文档、补依赖、绕过认证

团队用 autoinstall 处理了大型区块链项目 Celo（celo-org/celo-monorepo）。它的挑战在于依赖庞杂、测试还需要 mock 一套认证流程。

第一阶段，由于项目自带文档稀疏，agent 自己用 web 搜索去项目文档站找配置命令，还从文档里挑出一个最小可运行应用作为验证目标。第二阶段，agent 事先并不知道会遇到什么坑——它发现必须额外安装相关仓库 Foundry，于是又上网读 Foundry 的文档；最小应用第一轮没跑起来，第二轮它想到创建一个 mock 用户在本地启动应用，从而满足了要求。

真实项目的难点往往是"事先不可知的依赖与认证",而模型靠探索+网络搜索+造假数据,把这些坑一个个填平。

复利效应新模型更会搭环境，于是成为更强的"搭环境工具"

训练 Composer 2 时,管理 autoinstall 的正是它的前代 Composer 1.5。而 Composer 2 在专门考察"搭建开发环境能力"的 Terminal-Bench 上的得分大幅提升:

下一轮 autoinstall 的更强基座

这意味着 Composer 2 将为未来的 autoinstall 提供更好的基础——更会搭环境的模型,能搭出更好的训练环境,进而训练出更强的模型。团队预期未来旧模型还会介入运行管理、数据预处理、架构调参等更多训练环节。

这是一条自我增强的链路:模型的进步会回流到训练流程本身,使下一轮起点更高。

最值得带走的洞见

  1. 环境配置是 RL 训练的隐形瓶颈。坏环境不只是变慢,而是会让问题无解、让算力拿不到奖励信号白烧。
  2. "先定义成功、再实现成功"的双 agent 结构很关键。一个 agent 提出 10 条命令并描述预期输出,另一个选 3 条去实现,最多重试 5 次——把模糊任务变成可自动判定的闭环。
  3. 真正的难点是事先不可知的依赖与状态。模型靠主动 mock(假文件、假数据库、MinIO、Docker、mock 用户)和网络搜索,才搭得通 Celo 这类真实复杂项目。
  4. "用旧模型自举新模型"已被数据验证有效。Terminal-Bench 从 47.9% 升到 61.7%,新模型反过来成为更强的环境搭建工具。
  5. 这预示训练流程本身正在被模型接管。从搭环境开始,未来运行管理、数据预处理、架构调参都可能交给上一代模型,形成持续复利。
