OpenAI Privacy Filter 正式发布

Introducing OpenAI Privacy Filter

一句话总括：OpenAI 把具有前沿个人数据识别能力的模型压缩到可在本地高吞吐运行的规模，并以开放权重方式提供给开发者。

真正的变化：从格式匹配走向语境判断

传统工具擅长识别电话号码和电子邮件等固定格式，却容易漏掉依赖上下文才能判断的私人信息。

Privacy Filter 使用语言模型的语境理解能力。它能区分应当保留的公开信息，以及与私人个体相关、需要遮蔽的信息。

这改变了隐私过滤的能力边界：判断依据不再只是字符长什么样，还包括信息在当前语境中意味着什么。

能力实践意义

模型如何形成隐私跨度

  1. 建立分类体系
    定义个人身份、联系方式、账户号码和密钥等需要识别的信息。
  2. 改造预训练模型
    把自回归语言模型转换为双向 token 分类器，再进行监督式后训练。
  3. 约束式跨度解码
    用 BIOES 标签和受约束的 Viterbi 程序，把逐 token 预测组合成连贯边界。
数据告诉我们什么：在 PII-Masking-300k 上，原始评估的 F1 为 96%。修正评估中发现的标注问题后，F1 达到 97.43%。这说明模型表现很强，也提醒读者：benchmark 的标注质量会直接影响结论。

少量微调带来的跃升

在领域适配 benchmark 上，加入少量数据便能把 F1 从 54% 提高到 96%，随后逐渐接近饱和。

这意味着通用隐私模型不必覆盖每家机构的全部政策。团队可以在通用能力之上，用自己的术语、格式和决策边界完成适配。

小模型并不等于低能力

模型共有 15 亿个参数，但每次仅激活 5,000 万个参数。它仍支持最长 128,000 token 的输入。

这种设计把前沿能力放进更容易本地部署的体量中。隐私基础设施因此可以更快、更靠近数据源。

边界
它不是匿名化工具，也不是合规认证。

模型可能漏掉罕见标识符，也可能在上下文不足时过度或不足遮蔽。语言、文字系统、命名习惯和领域分布的变化也会影响表现。法律、医疗和金融等高敏感场景仍需要领域评估、针对性微调和人工审核。

今天，我们发布 OpenAI Privacy Filter。这是一款开放权重模型，用于检测并遮蔽文本中的个人身份信息（PII）。

此次发布是 OpenAI 建设更具韧性的软件生态系统这项长期工作的一部分。我们希望为开发者提供实用的基础设施，帮助他们安全地使用 AI。这些基础设施包括工具和模型，让开发者从一开始就能更轻松地落实有力的隐私与安全保护。

Privacy Filter 是一款拥有前沿个人数据检测能力的小型模型。它面向高吞吐量的隐私工作流设计，可以结合上下文识别非结构化文本中的 PII。

模型可以在本地运行。这意味着 PII 无须离开用户的机器，就能得到遮蔽或删除。它还能高效处理长输入，通过快速的单次处理完成遮蔽判断。

在 OpenAI 内部，我们已在自己的隐私保护工作流中使用 Privacy Filter 的微调版本。

我们之所以开发 Privacy Filter，是因为我们相信，借助最新的 AI 能力，隐私保护标准可以超过市场上的现有水平。根据评估期间发现的标注问题完成修正后，今天发布的 Privacy Filter 在 PII-Masking-300k benchmark 上达到了当前最佳水平。

借助此次发布，开发者可以在自己的环境中运行 Privacy Filter，也可以针对自身场景进行微调。开发者还能把更强的隐私保护能力接入训练、索引、日志记录和审查流程。

拥有前沿个人数据检测能力的小型模型

现代 AI 系统的隐私保护不能只靠模式匹配。

传统 PII 检测工具通常依赖确定性规则，以识别电话号码和电子邮件地址等固定格式。这些工具在狭窄场景中可能表现良好，却经常漏掉更隐晦的个人信息，也不擅长理解上下文。

Privacy Filter 具备更深入的语言和上下文理解能力，因此可以处理更细微的判断。它把强大的语言理解能力与面向隐私的标签体系结合起来，能够在非结构化文本中检测范围更广的 PII。这也包括必须结合上下文才能作出正确判断的情况。

它可以更准确地区分两类信息。一类是公开信息，应当保留。另一类与私人个体有关，应当遮蔽或删除。

最终得到的模型足以提供前沿水平的隐私过滤表现。同时，它的规模又足够小，可以在本地运行。

这意味着尚未过滤的数据可以留在设备上，不必先发送到服务器进行去标识化处理，从而降低暴露风险。

模型概览

Privacy Filter 是一款采用跨度解码（span decoding）的双向 token 分类模型。

它以自回归预训练 checkpoint 为起点，随后被改造成使用固定隐私标签分类体系的 token 分类器。模型不会逐 token 生成文本，而是在一次处理中为整个输入序列添加标签，再通过受约束的 Viterbi 程序解码出连贯的文本跨度。

这种架构为生产环境带来了几项实用特性：

  • 快速且高效：所有 token 都在单次前向传播中完成标注。
  • 理解上下文：语言先验让模型能够根据周围语境识别 PII 文本跨度。
  • 长上下文：此次发布的模型最多支持 128,000 个 token 的上下文。
  • 可配置：开发者可以调整运行阈值，并按工作流需要权衡召回率与精确率。

此次发布的模型共有 15 亿个参数，其中激活参数为 5,000 万个。

Privacy Filter 可以预测以下八类文本跨度：

  • private_person
  • private_address
  • private_email
  • private_phone
  • private_url
  • private_date
  • account_number
  • secret

account_number 类别用于遮蔽多种账户号码。其中包括信用卡号和银行账号等银行信息。secret 类别用于遮蔽密码和 API 密钥等秘密信息。

这些标签使用 BIOES 跨度标签进行解码，有助于生成更干净、更连贯的遮蔽边界。

构建过程

我们分几个阶段开发了 Privacy Filter。

首先，我们建立了一套隐私分类体系，用来定义模型应当检测的文本跨度类型。

其中包括个人标识信息、联系方式、地址、私人日期、多种不同的账户号码，以及 API 密钥和密码等秘密信息。账户号码还涵盖信用信息和银行信息。

其次，我们将一个预训练语言模型转换为双向 token 分类器。具体做法是用 token 分类头替换语言建模头，再使用监督分类目标进行后训练。

第三，我们使用公开数据和合成数据的混合数据集进行训练。这些数据旨在同时覆盖真实文本和难度较高的隐私模式。

部分公开数据的标签并不完整。对于这些数据，我们使用模型辅助标注和审核来提高覆盖率。我们还生成了合成示例，以增加格式、上下文和隐私子类型的多样性。

推理时，模型会通过受约束的序列解码，把 token 级别的预测结果转换为连贯的文本跨度。

这种方法保留了预训练模型广泛的语言理解能力，同时让模型专门适应隐私检测任务。

Privacy Filter 的表现

我们在标准 benchmark 上评估了 Privacy Filter。我们还进行了额外的合成评估和聊天式评估，用来测试更困难、也更依赖上下文的情况。

在 PII-Masking-300k benchmark 上，Privacy Filter 的 F1 分数为 96%，其中精确率为 94.04%，召回率为 98.04%。

我们在审核期间发现了数据集标注问题。使用修正后的 benchmark 版本时，F1 分数为 97.43%，其中精确率为 96.79%，召回率为 98.08%。

我们还发现，这款模型能够得到高效适配。即使只使用少量数据进行微调，也能迅速提高模型在特定领域任务中的准确率。

在我们评估的领域适配 benchmark 上，微调使 F1 分数从 54% 提升到 96%，随后逐渐接近饱和。

除了 benchmark 表现，Privacy Filter 还面向嘈杂的真实文本设计，力求提供实用的隐私过滤能力。

这类文本包括长文档、有歧义的指代、混合格式字符串，以及与软件有关的秘密信息。

模型卡还报告了针对代码库秘密信息检测的专项评估，以及覆盖多语言、对抗性和上下文依赖示例的压力测试。

局限性

Privacy Filter 不是匿名化工具，不是合规认证，也不能替代高风险场景中的政策审核。它只是更广泛的隐私设计体系中的一个组件。

模型行为会反映训练所使用的标签分类体系和决策边界。不同机构可能需要不同的检测或遮蔽政策。这些政策可能需要在领域内进行评估，也可能需要进一步微调。

如果语言、文字系统、命名习惯或业务领域与训练数据分布不同，模型表现也可能发生变化。

和所有模型一样，Privacy Filter 也会出错。它可能漏掉不常见的标识符，也可能无法识别含义模糊的私人指代。

上下文有限时，模型可能过度遮蔽实体，也可能遮蔽不足。这个问题在较短的输入序列中尤其明显。

在法律、医疗和金融工作流等高敏感领域，人工审核、领域专项评估和微调仍然十分重要。

获取方式

我们发布 OpenAI Privacy Filter，是为了帮助整个生态系统建立更有力的隐私保护。

该模型已于今天在 Hugging Face 和 GitHub 上发布，采用 Apache 2.0 许可证。

它可用于实验、定制和商业部署，也可以针对不同的数据分布与隐私政策进行微调。

除模型之外，我们还共享了相关文档。文档涵盖模型架构、标签分类体系、解码控制、预期使用场景、评估设置和已知局限。

这些资料可以帮助团队了解模型擅长什么，也能帮助他们判断哪些场景需要谨慎使用。

未来展望

AI 系统的隐私保护是一项持续工作，涉及研究、产品设计、评估和部署等多个环节。

Privacy Filter 体现了我们认为十分重要的一条发展方向：针对真实 AI 系统中的重要而明确的任务，打造具备前沿能力的小型高效模型。

我们选择发布这款模型，是因为我们认为，隐私保护基础设施应当更容易检查、运行、适配和改进。

我们的目标是让模型了解世界，而不是了解私人个体。Privacy Filter 有助于实现这个目标。

我们发布 Privacy Filter 的这个预览版本，是为了收集研究界和隐私领域社群的反馈，并进一步迭代模型表现。
