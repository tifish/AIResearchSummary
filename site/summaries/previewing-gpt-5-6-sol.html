GPT‑5.6 Sol 预览：能力跃升，但这次护栏先行

一句话总括：OpenAI 推出 GPT‑5.6 三档模型并把网络安全能力的明显跃升与史上最厚的分层安全栈捆绑发布——因为模型已强到能发现漏洞和利用原语，OpenAI 选择先做"政府通报 + 少数可信伙伴"的分阶段预览，而不是直接面向所有人开放。

01一个家族，三档定位，新命名法

number 表示"代"，Sol / Terra / Luna 表示可独立演进的"能力层级"

GPT‑5.6 用一套更清晰的命名拆开了"代际"与"档位"：旗舰 Sol 主打最强智能，Terra 面向日常工作（性能与 GPT‑5.5 相当但便宜一半），Luna 以最低成本提供可用能力。

结论：把"代"和"档"解耦，意味着用户今后会在"智能 / 速度 / 成本"三角里做选择，而不是只盯一个版本号——价格策略明显在向"便宜的中端"下压。

02能力：编码、生物、网络安全三线齐进

同时新增 max 推理档与调用子代理的 ultra 模式

Sol 是 OpenAI 迄今最强的模型，官方用三个方向的评测做了预览：

  • 编码：在考验命令行规划、迭代与工具协调的 Terminal‑Bench 2.1 上创下新的 SOTA。
  • 生物：在长程基因组与定量生物分析基准 GeneBench v1 上超过 GPT‑5.5，而且用了更少的 token。
  • 网络安全：在 ExploitBench 上仅用约 1/3 输出 token 即可与 Mythos Preview 竞争；在 UC Berkeley 联合打造的 ExploitGym 上，三款模型随推理增加都明显变强。

新增的 max 推理档给 Sol 更长的深度思考时间，而 ultra 模式通过调用子代理来加速复杂任务，已超出单一智能体的范畴。

结论：进步不只是"更准"，更是"更省"——用更少 token 拿到同等甚至更高表现，意味着前沿能力正在变得更便宜、更可规模化部署。

03关键转折：能力强到必须为它"加锁"

未跨越 Preparedness 框架的"网络关键"阈值，但已逼近

Sol 在网络安全上"更擅长帮人找漏洞、修漏洞，而非可靠地端到端完成攻击"。在 Chromium 与 Firefox 的测试中，它能识别 bug 和利用原语（exploit 的构件），但在测试条件下没有自主产出可用的完整攻击链，因此未跨越"网络关键"阈值。

但 OpenAI 强调：基准阈值无法覆盖模型与其他工具组合后的所有用法。正是这种不确定性加上能力的阶跃，让它选择"更强能力 + 更强护栏 + 分阶段发布"。

洞见：这是发布的真正分水岭——当模型开始具备"防御者受益、攻击者也可能受益"的双用能力时，发布逻辑从"默认开放"转向"先管控、再放开"。

04分层安全栈：没有单一护栏够用

OpenAI 用多层叠加而非单点防御，各模型配置不同：

模型内训练的拒答 生成中的实时分类器 账户级跨会话审查 差异化访问 监控与执法 持续测试

高风险场景下，实时分类器若发现可疑输出，可暂停生成、交由更大的推理模型审阅上下文，违规内容在到达用户前被拦下；账户级审查则用来把"持续恶意行为"与"合法的双用安全工作"区分开。

结论：安全被设计成"层层叠加"——任何一层都可能误伤合法工作（预览期可能更频繁），但这正是预览要测的：在拦住滥用的同时，正常工作能否依旧高效完成。

OpenAI 投入超过 70 万 A100 等效 GPU 小时做自动化红队，专门寻找通用越狱，因为"只对已知攻击有效的防护不够稳健"。这能比人工覆盖更多攻击模式、更早发现失败规律。此外还引入第三方人类专家红队作为补充，并建立快速响应流程来复现、评估、修复新越狱并纳入持续评测。

洞见：把大量算力从"提升能力"分给"攻击自己的护栏"，本身就是一个信号——安全正在被当作需要前沿算力投入的工程问题，而非附加项。

06可用性与定价：先窄后宽，缓存更可预测

预览期 GPT‑5.6 先通过 API 与 Codex 面向少数可信伙伴开放，已提前向美国政府通报能力与计划；OpenAI 明确表示不认为这种政府准入流程应成为长期默认，只是当前通往更广泛可用的最快路径。

变化细节
更可预测的缓存支持显式缓存断点、最短 30 分钟缓存寿命
缓存写入按未缓存输入价的 1.25 倍计费
缓存读取继续享 90% 的缓存输入折扣
硬件加速7 月起 Sol 上线 Cerebras，最高 750 token/秒

结论：定价与缓存机制都在为"长上下文、多轮代理"工作流优化成本，配合 Cerebras 的高速度，指向更重的 agentic（代理式）使用场景。

最值得带走的洞见

  • 能力与安全首次"同步加码"成为发布主轴——不再是先发能力、后补安全，而是把两者捆在一起作为前提。
  • 效率即能力：用 1/3 token 比肩对手、用更少 token 超过上一代，说明前沿竞争已从"能不能做到"转向"多便宜做到"。
  • 分阶段 + 政府通报是新拐点：当模型逼近"网络关键"阈值，发布逻辑从默认开放转向受控放开——但 OpenAI 明确反对让它常态化。
  • 安全被工程化：70 万 GPU 小时自动化红队 + 分层栈 + 快速响应流程，意味着护栏开始消耗前沿级算力。
  • 命名解耦预示长期路线：Sol/Terra/Luna 作为可独立演进的能力层级，暗示未来在同一"代"内会有更灵活的智能—速度—成本组合。
