GPT-5.6 Sol 预览：能力跃升，发布却"先收窄"

下一代旗舰模型 · 编码 / 生物 / 网络安全全面进步 · 史上最严安全栈
一句话总括：GPT-5.6 的真正新闻不是"更强"，而是"更强的同时被刻意收窄发布"——OpenAI 因网络安全能力的跃升，首次在发布前向美国政府报备、先只对少数受信任伙伴开放，标志着前沿模型正进入"能力 + 管控 + 政府介入"同步推进的新阶段。

家族三款模型，按"代际 + 能力档"重新命名

新命名法中，数字（5.6）代表代际，Sol / Terra / Luna 代表可各自迭代的能力档（智能 / 均衡 / 高速低价）。最值得注意的是 Terra：性能对标上一代 GPT‑5.5，价格却便宜一半。

价格-能力曲线整体下移：上一代旗舰级的表现，如今以中端价位即可获得，这才是对大多数开发者最实际的影响。

能力编码、生物、网络安全的三处实测进步

领域 / Benchmark结果
编码 · Terminal-Bench 2.1刷新 SOTA（命令行规划/迭代/工具协调）
生物 · GeneBench v1超过 GPT‑5.5，且用更少 token
网络安全 · ExploitBench仅用约 1/3 输出 token 即与 Mythos Preview 相当

新增两项形态：max 推理档（给模型最长时间深度推理）与 ultra 模式（调用子代理并行处理复杂任务，突破单一代理上限）。

"用更少 token 达到同等水平"是反复出现的主题——前沿竞争已从"能不能做"转向"以多低的成本与效率做到"。

边界能找漏洞，但未能自主造出完整攻击链

在 Chromium / Firefox 测试中，Sol 能识别 bug 与利用原语（exploit 的"积木"），但在受测条件下未能自主产出可用的全链漏洞利用，因此未跨越 Preparedness 框架的"网络关键（Cyber Critical）"阈值。

OpenAI 同时坦言：benchmark 阈值无法涵盖模型被组合、被多步使用的所有方式，这种不确定性正是它选择"强能力 + 强防护 + 分阶段发布"的原因。

定位很关键——模型"帮人找并修漏洞"强于"端到端发起攻击"，OpenAI 押注让防御者先受益，而非攻击者。

安全分层防护 + 70 万 GPU 小时自动化红队

没有单一防护能挡住自适应滥用，因此采用多层叠加：模型级拒答 → 实时分类器（高风险时暂停生成、由更大推理模型复审后决定是否放行）→ 账户级跨会话审查 → 差异化访问与监控。

为应对会进化的攻击者，OpenAI 投入超 70 万 A100 等效 GPU 小时做自动化红队，专门寻找能跨多种场景生效的"通用越狱"，并配合第三方人类专家红队与快速响应修复流程。

安全本身被当作一个需要大规模算力投入的"AI 对抗 AI"工程问题，而不只是事后打补丁——这是治理思路的转变。

发布受限预览：政府介入的"短期"妥协

预览期仅通过 API 与 Codex 向少数受信任伙伴开放，名单已与政府共享；广泛开放计划在"未来几周"。OpenAI 明确表态：不认为这种政府准入流程应成为长期默认，因为它会把最好的工具挡在用户与防御者之外，此举只是为了换取更快的广泛可用，并推动制定可复用的网络安全行政令框架。

另外：7 月将在 Cerebras 上以最高 750 token/秒 运行 Sol；缓存机制更可预测——缓存写入按未缓存输入费率 1.25x 计费，缓存读取仍享 90% 折扣，最短缓存寿命 30 分钟。

这是 AI 发布史上一个信号性时刻：能力越强，"何时、向谁、如何发布"越成为与监管方协商的对象。

最值得带走的洞见

  1. 性价比下移最实在：Terra 以一半价格对标上代旗舰，多数开发者的真实收益来自成本曲线下移，而非榜单数字。
  2. "更少 token"成新战场：无论生物还是网络安全，Sol 都强调用约 1/3 token 达到同等水平——效率正取代纯能力成为竞争焦点。
  3. 能力被刻意"配上刹车"：未跨越网络关键阈值，却仍采用最严防护与分阶段发布，说明 OpenAI 把不确定性本身当作风险来管理。
  4. 安全是算力密集型工程：70 万 GPU 小时自动化红队，意味着"用 AI 守 AI"已是前沿模型的标配成本项。
  5. 政府介入登场，但 OpenAI 划线：首次发布前报备并受限开放，却公开反对其成为长期默认——前沿模型的发布治理框架正在被现场定义。
