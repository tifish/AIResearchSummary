GPT-5.6 Sol 预览：更强模型被放进更严格的发布框架里

一句话总括：这次发布真正的贡献，是把旗舰模型的代码、生物和网络安全长任务能力，与多层安全防护、有限预览、政府协调和更清晰的价格层级一起打包。

3 个层级Sol、Terra、Luna 分别面向旗舰、均衡和低成本场景
700,000+A100 等效 GPU 小时用于自动化红队测试
750 tokens/sSol 计划 7 月在 Cerebras 上达到的最高速度

GPT-5.6 的重点不是一个模型，而是一组可按成本和能力选择的模型

OpenAI 将 GPT-5.6 系列分成 Sol、Terra、Luna：Sol 是旗舰，Terra 面向日常工作，Luna 强调速度和低成本。Terra 被称为性能可与 GPT-5.5 竞争，同时便宜 2 倍。

这改变了读者对 AI 影响的理解：前沿模型不再只是一条“最强模型”路线，而是开始变成可按任务风险、预算和速度组合的产品族。

Sol 的能力提升集中在需要规划、工具协调和长周期推理的任务

GPT-5.6 Sol 新增 max reasoning effort，并通过 ultra mode 使用子代理来加速复杂任务。在 Terminal-Bench 2.1 上，它被描述为命令行工作流的新 SOTA；在 GeneBench v1 上，它强于 GPT-5.5 且使用更少 token。

这些结果说明，AI 的价值正在从“答题更准”转向“能否持续推进复杂工作流”，尤其是编码、基因组分析和定量生物学这类需要多步骤判断的场景。

网络安全能力更强，但 OpenAI 强调它还没有跨过最高风险阈值

Sol 被称为 OpenAI 目前最强的网络安全模型，可改善漏洞研究和利用相关长任务。在 ExploitBench 上，它用约 1/3 输出 token 就能与 Mythos Preview 竞争；ExploitGym 也显示 Sol、Terra、Luna 随推理增强而提升。

但在 Chromium 和 Firefox 相关评估中，Sol 只识别出漏洞和 exploit primitives，没有在测试条件下自主产出可用的 full-chain exploit，因此没有跨过 Preparedness Framework 的 Cyber Critical 阈值。洞见是：安全风险评估正在从“模型会不会写危险代码”转向“模型能否自主完成端到端攻击链”。

防护方案是分层系统，而不是一次拒答就结束

OpenAI 描述的安全栈包括模型内拒答、生成过程中的网络和生物滥用分类器、更大推理模型复核、账户级信号、差异化访问、监控、执法和持续测试。高风险输出可能在生成中被暂停、复核，必要时在到达用户前被拦截。

这说明前沿模型的安全不再只是训练阶段问题，而是运行时治理问题；同样的技术内容在防御研究和攻击滥用中可能很像，因此系统必须看上下文、用户模式和访问级别。

自动化红队成为发布工程的一部分

OpenAI 称其投入超过 700,000 A100-equivalent GPU hours 做自动化红队，目标是寻找能跨多个提示或上下文生效的 universal jailbreak。同时，第三方人类专家红队会在预览期继续进行。

这让读者看到一个趋势：模型越强，安全测试本身也越依赖模型和算力；未来的发布竞争可能不仅看 benchmark，也看谁能更快发现、复现、修补和回归测试新型绕过方式。

有限预览反映出前沿模型发布已受到政策和供应链约束

GPT-5.6 预览期先通过 API 和 Codex 面向少量可信伙伴，参与方信息已与美国政府共享。OpenAI 表示这不是理想的长期默认流程，但认为它有助于在数周内更广泛开放，并配合网络安全 Executive Order 框架形成可重复流程。

这改变了“模型发布就是产品上线”的直觉：高能力模型的发布节奏已经受到政府沟通、风险分层、伙伴测试和更广泛可用性之间的平衡影响。

价格和缓存规则让开发者更容易估算部署成本

GPT-5.6 按每 100 万 token 计价：Sol 为 $5 输入 / $30 输出，Terra 为 $2.50 / $15，Luna 为 $1 / $6。缓存写入按未缓存输入价格的 1.25 倍计费，缓存读取继续享受 90% cached-input discount，最低缓存生命周期为 30 分钟。

Terra$2.50 input · $15 output
Luna$1 input · $6 output

这里的洞见是，模型能力提升正在和成本可预测性绑定；对企业和开发者来说，缓存、速度和分层模型可能和绝对能力同样重要。

最值得带走的洞见

  • GPT-5.6 的核心信号是“能力升级 + 分层发布 + 分层防护”，而不是单一模型性能宣传。
  • 网络安全能力的评估重点正在转向端到端攻击链、自主性和真实工具组合，而不是孤立代码片段。
  • 自动化红队和运行时拦截正在成为前沿模型安全的基础设施。
  • Sol、Terra、Luna 的命名和定价表明，未来模型选择会更像按任务选择计算资源。
  • 有限预览说明高能力 AI 的发布已经进入产品、政策、安全和企业需求共同决定的阶段。
