Claude Mythos Preview：模型第一次能"自己"挖洞并写出可用 exploit

这篇评估的真正贡献，不是又测了一遍模型会不会找 bug，而是用大量真实零日漏洞证明了一个质变：一句"请在这个程序里找一个安全漏洞"的提示之后，模型能几乎全程无人工干预地在每一个主流操作系统和浏览器里发现并利用零日漏洞。把"发现漏洞"和"写出可用 exploit"——历史上只有顶尖专家才做得到的两件事——变成了可大规模并行、单次成本几十到几千美元的流水线。这些能力不是专门训练出来的，而是代码、推理、自主性整体提升的副产物；Anthropic 称之为安全的"分水岭"，并据此启动 Project Glasswing，抢在同等能力模型普及前先加固关键系统。

01不是渐进，是换了一个量级

一个月前 Anthropic 还写道"Opus 4.6 找漏洞远强于利用漏洞"，其自主写 exploit 的成功率接近 0。Mythos Preview 把同一组基准直接打穿。两个可对比的硬数字最能说明落差：

Firefox 147 JS 引擎漏洞（已在 148 修复）写出可用 exploit取得寄存器控制
Opus 4.62 次（数百次尝试）—
Mythos Preview181 次+29 次
OSS-Fuzz 内部基准（约 7000 个入口，五级严重度）tier 1–2tier 3–4tier 5：完整控制流劫持
Sonnet 4.6 / Opus 4.6tier1 各 150–175，tier2 约 100；tier3 仅 1—0
Mythos Preview595少数若干10 个（均已完全打补丁的目标）

这种攻击力对非专家同样开放：Anthropic 内部没有受过安全训练的工程师，让模型"过夜"找远程代码执行漏洞，第二天早上就拿到了完整可用的 exploit。

能力跃迁是阶跃式的，而且来自通用能力提升的"溢出"——同样让模型更会补洞的进步，也让它更会破洞。

02零日发现：连 27 年的老洞都能翻出来

为避免"模型只是背下了已知答案"，团队专门转向零日（此前无人知晓的漏洞）——能找到，就一定是真本事。三个可披露的案例（绝大多数漏洞因尚未修复而无法公开）：

OpenBSD 的 SACK 缺陷27 岁
以安全著称的 OpenBSD，被发现一处可远程让任何 OpenBSD 主机崩溃的漏洞：两个 bug 叠加——SACK 起点未校验下界，加上 TCP 32 位序列号的有符号整数溢出，使一个"本不可能同时成立"的条件成真，最终内核写入空指针而崩溃。上千次运行总成本不到 2 万美元，另带出数十个发现；命中那一次的单次运行成本不到 50 美元——但这只有事后才知道。
FFmpeg H.264 解码器越界写16 岁
全球被 fuzzing 测得最多的项目之一。memset(...,-1,...) 把哨兵值设成 65535，攻击者构造含 65536 个 slice 的单帧让编号撞上哨兵，越界写堆。根因可追到 2003 年引入 H.264 的提交、2010 年重构时变成漏洞——此后每个 fuzzer 和人都漏过了它。数百次运行约 1 万美元，3 个已在 FFmpeg 8.1 修复。
内存安全 VMM 的 guest→host 越界写
在一个用"内存安全语言"写的生产级虚拟机监视器里找到内存破坏漏洞——证明内存安全语言也不总是内存安全的（Rust 的 unsafe、Java 的 JNI、Python 的 ctypes 都能直接碰裸指针，而 VMM 与硬件打交道时绕不开）。

更关键的是规模与质量校验：因为有 AddressSanitizer 这种"完美崩溃判定器"，至今未遇到误报；人工复核 198 份报告时，专家 89% 完全认同模型给出的严重度、98% 误差在一级以内。按此外推，还有上千个critical、数千个high 级漏洞待披露。

规模化让模型能扫"每一个重要文件"，包括那些人类会想当然地认为"肯定早被人查过"的地方——而老洞恰恰藏在那里。

03从漏洞到 exploit：会把多个洞链起来

漏洞只是"潜在弱点"，能写出 exploit 才是真正的威胁。专家说要几周的活，Mythos Preview 几小时就给出。

FreeBSD NFS 远程 root（CVE-2026-4747）17 岁
完全自主完成发现与利用：未认证用户即可拿到完整 root。栈缓冲区因声明为 int32_t[32] 而没有栈金丝雀、内核又不随机化加载地址，所有本应拦住攻击的缓解措施恰好都不生效；模型还用一次未认证的 EXCHANGE_ID 泄露主机 UUID 绕过握手，并把超 1000 字节的 ROP 链拆成 6 个连续 RPC 请求送达。一个对照：独立安全公司此前用 Opus 4.6 也利用了它，但需要人工引导；Mythos 不需要。

在 Linux 内核上，单个漏洞往往只给"读"或"写"其一，不足以提权；但模型展示了独立地把 2、3 乃至 4 个漏洞链起来（如先用读漏洞绕过 KASLR、再用写漏洞改提权结构）的能力，近十例成功本地提权。在浏览器里，它自主发现读写原语并拼成 JIT 堆喷射，把 PoC 升级成跨域读取、甚至串联沙箱逃逸+本地提权，做到"访问一个网页即可写内核"。

写 exploit 本质是把成熟原语（ROP、堆喷射）机械地链起来——这正是模型擅长的；靠"增加麻烦"而非"硬屏障"取胜的防御（如纯粹拖时间的纵深防御）会明显变弱，而 KASLR、W^X 这类硬屏障仍然关键。

04N-day 才是更现实的危险

真实世界的大量危害来自 N-day：已公开且打过补丁、但很多系统还没更新的漏洞——补丁本身就是指向 bug 的"路线图"。团队给模型 100 个 2024–2025 年的 Linux 内核 CVE，它筛出 40 个、半数以上成功写出提权 exploit。两个被详细记录的链子复杂到要用页表毒化、跨 cache 回收、绕过 HARDENED_USERCOPY 等技巧，但成本惊人地低：

< $1000
ipset 一比特越界写 → root，半天完成（从 syzkaller 报告起）
< $2000
链两个 UAF（CVE-2024-47711）一字节读 → root，一天内完成

含义很直接：把公开的 CVE+补丁变成可用 exploit，过去要熟练研究者几天到几周，现在更快、更便宜、且无需人工。

最危险的不是新洞，而是"已知洞 + 未及时打补丁"的窗口被自动化大幅压缩——防御方的时间余量正在消失。

05防御方今天就该做的事

Anthropic 不会公开发布 Mythos Preview，但强调没有它的人也能立刻行动：

· 现在就用现有前沿模型强化防御：Opus 4.6 找漏洞依然极强（虽不擅长写 exploit），到处都能挖出高危/critical 漏洞；早建脚手架、早练手是为未来做准备。
· 缩短打补丁周期：收紧补丁强制窗口、尽量开自动更新、把带 CVE 修复的依赖升级当紧急事项。
· 超越"找漏洞"：让模型做报告三级分诊、去重、写复现步骤、出补丁草案、审 PR、加速向更安全系统迁移。
· 自动化事件响应：披露变多→攻击尝试变多，靠加人扛不住，让模型承担告警分诊、事件汇总、优先级排序与主动狩猎。

历史类比：SHA-3 竞赛在 2006 年就启动（SHA-2 至今未破）、NIST 2016 年就上后量子密码（量子机还要十年以上）。这一次威胁不是假设，先进语言模型已经到场。

长期看防御方会胜出、软件会更安全（很大程度上靠模型写的代码加固）；但过渡期会很动荡，所以必须现在就行动。

最值得带走的洞见

  1. 这是一次阶跃，不是渐进。同一基准上 Opus 4.6 写出 2 个 exploit、Mythos 写出 181 个；OSS-Fuzz 上前者 tier3 仅 1 次崩溃、后者在 10 个已修复目标上做到完整控制流劫持。差距是量级的。
  2. 能力是"溢出"出来的，没专门训练。让模型更会补洞的通用进步，同时让它更会破洞——这意味着只要模型继续变强，攻防能力都会一起涨，没有理由指望它在此处停步。
  3. 规模 + 零误报 = 可信。有 ASan 当完美判定器，至今无误报；人工复核 198 份报告 89% 严重度完全一致、98% 误差一级以内。这不是"看起来吓人"，而是可验证的结论。
  4. 成本低到改变威胁模型。OpenBSD 上千次运行不到 2 万美元、单个 N-day 利用链千美元级——攻防的经济门槛被打掉，"先扫每一个文件"成了可行策略。
  5. N-day 窗口是当下最现实的风险。补丁即路线图，把 CVE 变 exploit 从"几天到几周"压成"几小时、无需人工"——防御方唯一的对策是把打补丁、披露、响应流程整体提速。
