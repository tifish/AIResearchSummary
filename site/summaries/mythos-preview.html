Claude Mythos Preview 的网络安全能力评估

来源：Anthropic ｜ 日期：2026-04-07 ｜ 估计阅读：6 分钟
一句话总括：新一代模型 Mythos Preview 不再只是“能找 bug”，而是能自主把真实世界的零日漏洞链接成可工作的攻击利用——在主流操作系统、浏览器和加密库中皆然。这跨过了上一代模型几乎跨不过去的门槛，因此 Anthropic 选择不公开发布它，而是启动 Project Glasswing 优先加固关键软件。

01真正的飞跃：从“发现漏洞”到“自主利用”

上一代擅长“找与修”，这一代擅长“找与攻”——而攻是历史上最难自动化的部分。

一个月前 Anthropic 还写道 Opus 4.6“几乎 0% 的自主利用成功率”。Mythos Preview 不是同一量级：同样的 Firefox 147 漏洞利用基准，Opus 4.6 在数百次尝试中仅成功 2 次，Mythos Preview 成功 181 次，另有 29 次拿到寄存器控制。

为何重要：“写 exploit”历来被视为只有顶尖人类专家能做的机械却高难的工作。一旦它可被规模化自动完成，攻防之间“从公开补丁到大规模利用”的时间窗会被压缩到以小时计——这正是过去二十年安全均衡赖以维系的护城河。

02这些能力不是被“训练”出来的，是涌现的

同样让模型更会“修漏洞”的代码、推理、自主性提升，也让它更会“攻漏洞”。

Anthropic 强调并未专门训练攻击能力；它是通用能力提升的副产品。换句话说，无法靠“只训练防御不训练攻击”来避免这种能力出现。无安全背景的工程师让模型“通宵找远程代码执行漏洞”，第二天醒来就拿到完整可用 exploit。
为何重要：这打破了“攻击能力可以被刻意隔离”的安慰式假设。能力与门槛同时下降，意味着低技能攻击者也能触达原本属于专家的攻击面。

03三个标志性漏洞：旧代码里藏了几十年

最值得震撼的不是漏洞多新，而是它们多老——多年被人类和模糊测试反复审计却漏掉。

目标年龄性质
OpenBSD（以安全著称）SACK 实现27 年有符号整数溢出触发空指针写，可远程崩溃任何 OpenBSD 主机
FFmpeg H.264 解码器16 年哨兵值碰撞导致堆越界写（须构造 65536 个 slice）
FreeBSD NFS 服务（CVE-2026-4747）17 年未认证用户远程获取 root，把 20-gadget 的 ROP 链拆到 6 个 RPC 包
为何重要：这些都是被无数模糊测试器和人类审计“写过的安全代码”。模型的优势不在比人聪明，而在可规模化地审遍每一个文件，包括人们因“这肯定有人查过”而跳过的地方。FreeBSD 案例还揭示：只有真去写 exploit，才发现栈保护等防御因编译细节恰好失效——生成利用本身就是更准确的漏洞分诊手段。

目前已发现的潜在漏洞中不到 1% 完成修复，因此 Anthropic 只能讨论极小一部分，并用 SHA-3 哈希对未披露漏洞做密码学承诺。若复核一致性保持，剩余发现可能还包含逾千个关键级、数千个高危级漏洞——本文所述只是“下界”。
为何重要：当“找到一个真实可利用漏洞”的边际成本降到几十美元，攻击经济学被彻底改写：稀缺的专家时间不再是瓶颈，算力是。

05N-day 才是最现实的危险，与防御者的行动清单

已知且已打补丁、但很多系统还没更新的漏洞，是真实世界伤害的主要来源。

模型仅凭一个 CVE 编号和 git 提交哈希，就能全自动把公开补丁“逆向”成可用 exploit——补丁本身就是漏洞的路线图。Anthropic 给出的防御建议：
  • 现在就用现有前沿模型加固（如 Opus 4.6 仍很擅长找漏洞）；
  • 缩短补丁周期：尽量开启自动更新、把带 CVE 的依赖升级当紧急事项；
  • 自动化事件响应与分诊，让模型承担三级分诊、去重、写复现步骤、起草补丁；
  • 重审漏洞披露政策与遗留系统应急方案，为漏洞洪峰做准备。
为何重要：长期看 Anthropic 认为防御方会占优（更多代码将由模型写得更安全），但过渡期会动荡，且短期内可能利好攻击方——优势属于“最会用工具的一方”。
最值得带走的洞见

5 条带走

  1. 门槛被跨过了。自主“写 exploit”从 Opus 4.6 的近乎 0%，跃升到 Mythos Preview 在基准上 181 次成功——这是攻防史上的分水岭，不是渐进改良。
  2. 攻击能力是涌现的，无法靠“只训防御”规避。同一批能力同时强化了修与攻。
  3. 最老的代码最危险。27 年的 OpenBSD、17 年的 FreeBSD root——人类审计的盲区正是模型规模化覆盖的强项。
  4. 经济学已变。几十到几千美元即可产出真实可用利用，瓶颈从专家时间转为算力，攻击可被海量并行。
  5. 立即行动，重点是缩短补丁窗口。N-day 才是现实主战场；自动更新、紧急化 CVE 修复、自动化响应是当下最实际的防线。
