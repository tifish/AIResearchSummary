一句话总括

Petri 的真正贡献，是把模型对齐评测变成更开放、更真实、更独立的公共工具链。

Anthropic 不是只更新一个内部测试工具，而是把 Petri 的发展交给 Meridian Labs，让检测欺骗、谄媚、配合有害请求和破坏研究倾向的能力更容易被外部机构复用和信任。

2025.10Petri 首次作为开源对齐测试工具箱发布
3.0新版重点是可适配性、真实性和深度评估
Sonnet 4.5 起每个 Claude 模型都纳入 Petri 对齐评估

Petri 把“模型是否会装作安全”变成可重复测试的问题

方法Petri 使用一个独立的 auditor model 模拟对齐相关场景，再由 judge model 对模型回复转录进行打分，观察是否出现欺骗、谄媚或配合有害请求等行为。

这改变了 AI 风险讨论的重心：问题不只是模型在标准问答里答得好不好，而是它在被置于特定压力和目标环境中时会怎样行动。

Petri 3.0 的重点是让测试更像真实部署，而不是更像考试

趋势新版将 auditor model 和 target model 拆成可分别调整的组件，方便研究者把同一框架迁移到更多用途。

Dish 附加组件进一步使用真实系统提示和真实 scaffold 来运行测试，减少模型识别“自己正在被评测”的机会。结论是，越接近真实部署的评测，越可能暴露普通基准测试看不到的行为风险。

与 Bloom 集成后，Petri 从广覆盖走向“广覆盖加深挖”

能力Petri 原本更适合快速覆盖多种对齐场景，Bloom 则用于对特定行为做更深入的评估；两者结合后，研究者可以先发现可疑模式，再集中分析具体风险。

这说明 AI 对齐评测正在从单点 benchmark 转向工具栈：先筛查，再复核，再解释，才更适合处理复杂模型行为。

移交给 Meridian Labs，是为了让评测结果更有公信力

治理Anthropic 将 Petri 的开发交给 AI 评估非营利组织 Meridian Labs，并把这一动作类比为此前将 Model Context Protocol 捐赠给 Linux Foundation。

关键洞见是：当评测工具由单一 AI 实验室控制时，外界更难相信其结果完全中立；独立机构维护可以提升跨行业、政府和研究群体的采信度。

外部采用说明，对齐评测正在成为公共安全能力

影响英国 AI Security Institute 已将 Petri 作为评估模型破坏 AI 研究倾向的重要组成部分，这表明此类工具不只是实验室内部质量控制，也正在进入政府和安全机构的评估流程。

结论是，随着模型能力提高，可靠评测本身会成为 AI 治理基础设施，而不是发布前的附属检查。

最值得带走的洞见

  • Petri 的核心价值在于测试模型行为倾向，而不是只测试知识或任务表现。
  • 真实系统提示、真实 scaffold 和更自然的评测环境，会让对齐测试更接近实际风险。
  • 将工具交给独立非营利机构，可以提高评测结果在行业外部的可信度。
  • Petri 与 Bloom 的组合代表一种趋势：AI 安全评估会越来越像持续运行的工具链。
  • 政府机构采用这类工具，说明 AI 对齐测试正在从研究议题转向治理基础设施。
