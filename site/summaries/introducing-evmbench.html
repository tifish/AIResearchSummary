EVMbench：给 AI 智能体的“智能合约攻防考卷”

来源：OpenAI（与 Paradigm 合作）· 2026 年 2 月 18 日
一句话总括：EVMbench 第一次用可复现、可量化的方式回答“AI 智能体能在多大程度上发现、修补和攻破智能合约漏洞”—— 答案是“攻易守难”：模型已能端到端地把合约里的钱掏空，却仍不擅长穷尽式审计和无副作用地修复。

它为什么重要：被守护的是上千亿美元

关键数据 → 趋势 → 结论

智能合约常态化地保护着 超过 1000 亿美元的开源加密资产。当 AI 智能体越来越会读写和执行代码，它们既是潜在攻击者，也是潜在防御者。 结论：在这种“经济上真实有后果”的环境里测量模型能力，本身就是一种网络安全风险预警。

漏洞多数取自 Code4rena 等公开代码审计竞赛，另外加入了为高吞吐、低成本稳定币支付而设计的 Tempo L1 区块链的若干审计场景。 结论：把基准延伸到“支付型合约”，正是押注智能体驱动的稳定币支付将成为下一个实际高发领域。

三种考法：检测、修补、利用

  • Detect 检测：审计合约仓库，按对真实漏洞的召回率与对应赏金计分。
  • Patch 修补：改写易受攻击的合约，必须既保留功能又消除可利用性，由自动测试与漏洞检查验证。
  • Exploit 利用：在沙盒链上对部署的合约发起端到端“掏空资金”攻击，通过交易重放与链上验证程序化评分。

为保证客观可复现，团队用 Rust 写了测试框架，在隔离的本地 Anvil 环境中确定性重放交易、限制不安全 RPC 方法；所有漏洞都是历史且公开记录的。

结论：仅半年多，模型的实战攻破能力翻了一倍多——这正是“AI 对攻防双方都可能是变革性力量”的实证，也意味着风险演进速度远快于直觉。

有趣的行为差异：目标越明确，模型越强

智能体在 exploit 上表现最好，因为目标极其明确——“不断迭代直到资金被掏空”。而在 detect 上，它们常常找到一个问题就停手，而非穷尽式审计；在 patch 上，既要消除细微漏洞又要保留完整功能仍然很难。detect 召回率与 patch 成功率都低于完整覆盖。 结论：当前模型擅长“有清晰胜利条件的进攻”，不擅长“开放式、需要彻底性的防守”。

诚实的边界：基准不等于真实世界的全部难度

漏洞来自审计竞赛，虽真实且高危，但广泛部署的主流合约受过更多审视、更难攻破；detect 模式无法判断模型多报的漏洞是“人类漏掉的真问题”还是误报；exploit 采用顺序重放、干净的本地链（而非主网分叉）、仅支持单链，需依赖时序的攻击和某些主网部署被排除在外。 结论：分数应被读作“能力下界与趋势信号”，而非“现实安全水平的最终判决”。

不只是测量，更是行动号召

OpenAI 同步推进防御侧投入：扩大安全研究智能体 Aardvark 的私测、为广泛使用的开源项目提供免费代码库扫描，并基于 2023 年的网络安全资助计划再投入 1000 万美元 API 额度，重点支持开源软件与关键基础设施。基准的任务、工具与评测框架全部开放。 结论：因为网络安全本质“双重用途”，策略是“加速防御者、拖慢滥用者”的循证迭代路线。

最值得带走的洞见

  1. 能力不对称是核心发现：AI 当前“攻强守弱”，目标明确的攻击进步快，开放式的检测与修补仍是瓶颈。
  2. 进步速度惊人：exploit 得分半年从 33.3% 升到 71.0%，提醒我们 AI 网络风险按月而非按年演进。
  3. 真实经济场景才有意义：117 个真实漏洞 + 守护 1000 亿美元的合约，让评测扎根于有实际后果的环境。
  4. 支付型合约是新前沿：纳入 Tempo 稳定币支付场景，预判智能体驱动的链上支付将成为高发领域。
  5. 测量即倡议：开放基准、扩大 Aardvark、投入 1000 万美元，意在把 AI 辅助审计推成开发者与安全研究者的标准工作流。
