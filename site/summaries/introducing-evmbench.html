Introducing EVMbench

EVMbench 的真正贡献，是把 AI agent 对智能合约安全的影响拆成可复现的检测、修补和利用任务，让“模型会不会改变攻防格局”有了可比较的证据。

OpenAI Research 2026-02-18 智能合约安全 AI agent 评测
$100B+ 智能合约经常保护的开源加密资产规模。
117 来自 40 次审计的精选高危漏洞。
71.0% GPT-5.3-Codex 在 exploit 模式的得分。
$10M OpenAI 承诺用于网络防御的 API credits。

EVMbench 衡量的不是写代码能力，而是经济场景中的安全能力

基准来自真实审计材料和公开漏洞。它要求 agent 在合约仓库、测试、部署脚本和链上状态之间完成任务。

这改变了读者对 AI 影响的理解。模型能力不只体现在回答问题，而是体现在能否影响真实资产安全。

Detect：审计代码库，按找到的真实漏洞和审计奖励计分。
Patch：修改脆弱合约，在保留功能的同时消除可利用性。
Exploit：在沙盒链上执行端到端攻击，并通过交易回放和链上验证评分。

可复现环境让安全评测更接近工程现实

构造任务

研究者复用已有 PoC exploit 测试和部署脚本。缺失部分则手写，以保证漏洞可触发、可修补、可评分。

约束执行

Rust harness 部署合约并确定性回放交易。危险 RPC 方法被限制，任务运行在本地 Anvil 环境。

校验结果

exploit 模式使用程序化 grader、交易回放和链上验证。这样减少主观判断，也降低 agent 取巧得分的空间。

模型最擅长明确目标，最难做好完整防御

任务模式 暴露的弱点 对 AI 影响的解释
Exploit 能力提升最快，GPT-5.3-Codex 得分 71.0%。 当目标是“持续迭代直到资金被转走”时，agent 更容易把代码理解、执行和反馈循环连起来。
Detect 召回率仍未覆盖全部漏洞，有时找到一个问题就停止。 这说明 AI 审计不是简单问答任务。完整覆盖需要系统性搜索和退出标准。
Patch 修补漏洞同时保持完整功能仍很困难。 防御能力比攻击演示更难评估。正确修复需要理解合约意图，而不只是阻断某个 exploit。
EVMbench 的核心信号是：AI agent 已经能在沙盒中完成相当强的漏洞利用，但防御性审计和稳健修复仍没有达到可完全依赖的水平。 这使它既是能力榜单，也是风险预警。

限制同样重要

这些结果不能直接等同于主网上的真实攻击成功率。基准越清楚边界，读者越能正确使用分数。

漏洞多来自 Code4rena 审计竞赛。它们真实且高危，但未必代表最受审查的大型主网合约。
detect 模式只能检查 agent 是否找到人类审计报告中的漏洞。额外发现目前难以可靠判定是真漏洞还是误报。
exploit 交易按顺序回放，精确时序、多链状态和主网分叉行为暂不覆盖。

文章把能力发布和防御责任放在一起

安全训练用于降低误用。
自动监控和执行管线用于发现滥用。
Aardvark 私测扩展面向防御研究。
1000 万美元 API credits 支持开源和关键基础设施防护。

最值得带走的洞见

真实漏洞让评测更有含金量

117 个漏洞来自 40 次审计，并配合可执行环境。它比静态问答更能反映 agent 的工程能力。

攻击目标更容易被模型优化

exploit 模式得分提升最快。明确反馈循环会放大 agent 的迭代优势。

防御仍需要人类流程

detect 和 patch 仍不完整。AI 更适合作为审计放大器，而不是替代审计责任。

评测本身是一种治理工具

当双重用途能力被公开、量化和复现，开发者、研究者和平台才能更早调整防护策略。EVMbench 的价值不只是排名，而是帮助生态把更强模型导向漏洞发现和修复。
