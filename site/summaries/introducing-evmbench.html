Introducing EVMbench

一句话总括：EVMbench 把 AI 智能体放进可复现的区块链环境，用真实高危漏洞同时检验其审计、修复和攻击能力。

更短、更易读的中文总结

EVMbench 不只测试模型能否谈论安全问题。它要求智能体面对可执行的合约代码，并完成发现、修补或利用漏洞的实际任务。这让网络安全能力的测量更接近真实经济风险。

同一套漏洞，三种能力视角

基准把安全能力拆成三个模式。这样的设计能分辨模型是会“看出问题”，还是能真正修好或利用问题。

Detect：全面发现

智能体审计整个代码库。评分看它找回了多少已知漏洞，以及这些漏洞对应的审计奖励。难点在于保持全面，而不是找到一个问题后就停止。

Patch：安全修复

智能体必须消除可利用性，同时保留合约原有功能。测试显示，修补细微漏洞而不引入功能退化仍然很难。

Exploit：端到端攻击

智能体在隔离链上持续尝试，直到完成资金抽取。由于目标明确，这反而是三种模式中表现最好的任务。

边界：沙盒成绩不是现实世界的完整风险刻度

样本主要来自 Code4rena 竞赛。现实中广泛部署的合约往往经过更多审查，可能更难利用。

exploit 任务使用干净的本地 Anvil 环境，只支持单链，并按顺序重放交易。依赖精确时机、主网状态或跨链交互的行为不在当前覆盖范围内。

detect 评分也只核对人类已发现的漏洞。智能体额外报告的问题，目前无法可靠判断是真正的新漏洞，还是误报。

核心洞见：最明确的攻击目标带来了最高得分，而更开放、更强调完整性的防御任务表现较弱。AI 安全工具的下一步重点，不只是提升单次成功率，还要改善审计覆盖率和修复可靠性。

为什么这很重要

智能合约经常承载价值超过 1000 亿美元的开源加密资产。随着 AI 智能体阅读、编写和执行代码的能力不断提高，我们越来越有必要在具有实际经济意义的环境中衡量这些能力。同时，也应推动人们将 AI 系统用于防御，帮助审计并加固已经部署的合约。

我们与 Paradigm（在新窗口中打开）共同推出 EVMbench。这个基准用于评估 AI 智能体发现、修补和利用高危智能合约漏洞的能力。EVMbench 收录了 40 次审计中经过筛选的 117 个漏洞，其中大部分来自开放式代码审计竞赛。

EVMbench 还加入了 Tempo（在新窗口中打开）区块链安全审计过程中的若干漏洞场景。Tempo 是一条专门构建的 L1，旨在通过稳定币实现高吞吐量、低成本的支付。这些场景把基准扩展到了面向支付的智能合约代码。我们预计，由智能体发起的稳定币支付将会增长。这些场景也让基准扎根于一个实际重要性正在上升的领域。

为了构建任务环境，我们尽可能改造已有的概念验证漏洞利用测试和部署脚本。如果这些材料不存在，我们就手动编写。在 patch 模式中，我们确认漏洞确实可以被利用，也能够在不引入编译错误的情况下得到缓解。否则，评估环境本身就会失去有效性。

在 exploit 模式中，我们编写了定制评分器，并对环境进行了红队测试。目的是找出智能体可能欺骗评分器的方法，再将这些途径封堵。除了由 Paradigm 提供领域专业知识并进行任务质量控制之外，我们还使用自动化任务审计智能体，提高环境的可靠性。

EVMbench 评估三种能力模式：

  • Detect：智能体审计一个智能合约代码库。评分依据是它对真实漏洞的召回率，以及这些漏洞对应的审计奖励。
  • Patch：智能体修改存在漏洞的合约。它必须保留预期功能，同时消除漏洞的可利用性。系统会通过自动化测试和漏洞利用检查验证结果。
  • Exploit：智能体在已部署合约上执行端到端的资金抽取攻击。任务运行在沙盒区块链环境中，并通过交易重放和链上验证，以程序化方式评分。

为了支持客观且可复现的评估，我们开发了一个基于 Rust 的测试框架。它负责部署合约，以确定性方式重放智能体交易，并限制不安全的 RPC 方法。Exploit 任务运行在隔离的本地 Anvil 环境中，而不是实时网络上。任务所用漏洞都来自历史记录，并且已经公开披露。

我们在全部三种模式中评估了前沿智能体。在 exploit 模式中，通过 Codex CLI 运行的 GPT‑5.3‑Codex 得分为 71.0%。与先前模型相比，这是一次显著提升。六个多月前发布的 GPT‑5 得分为 33.3%。Detect 模式的召回率和 patch 模式的成功率仍未达到完整覆盖，因为大量漏洞对智能体来说依旧很难发现和修复。

EVMbench 还揭示了模型在不同任务中的一些有趣行为差异。智能体在 exploit 场景中表现最好，因为目标非常明确：持续尝试，直到资金被抽走。相比之下，它们在 detect 和 patch 任务中的表现较弱。

在 detect 模式中，智能体有时找到一个问题后就会停止，而不会彻底审计整个代码库。在 patch 模式中，既保留全部功能，又消除隐蔽漏洞，仍是一项困难的任务。

局限性

EVMbench 无法代表现实世界中智能合约安全问题的全部难度。基准中的漏洞取自 Code4rena 审计竞赛。这些漏洞真实存在，而且严重程度很高。但许多部署规模大、使用广泛的加密合约会接受更严格的审查，因此可能更难被利用。

我们的评分系统很稳健，但并不完美。在 detect 模式中，我们检查智能体是否找到了人类审计人员发现的相同漏洞。如果智能体识别出额外问题，我们目前还没有可靠的方法判断这些问题究竟是人类遗漏的真实漏洞，还是误报。

Exploit 场景也存在结构性限制。交易会在评分容器中按顺序重放，因此依赖精确时机机制的行为不在评估范围内。链上状态来自一个干净的本地 Anvil 实例，而不是主网分叉。目前，我们也只支持单链环境。在部分任务中，这意味着必须使用模拟合约，而不能使用主网上的实际部署。

为什么这很重要

智能合约保护着价值数十亿美元的资产。AI 智能体可能给攻击者和防御者都带来根本性变化。在这个领域衡量模型能力，有助于追踪正在出现的网络安全风险。这也凸显了将 AI 系统用于防御的重要性，包括审计和加固已经部署的合约。

EVMbench 既是一项测量工具，也是一项行动倡议。随着智能体能力提高，开发者和安全研究人员越来越有必要把 AI 辅助审计纳入工作流程。

最近几个月，我们看到模型在网络安全任务上的表现取得了显著进步。这些进步同时惠及开发者和安全专业人员。与此同时，我们也在准备更强的网络安全保障措施，以支持防御性用途，并提高整个生态系统的韧性。

网络安全天然具有双重用途。因此，我们采用以证据为基础、持续迭代的方法。一方面，我们希望加快防御者发现和修复漏洞的速度。另一方面，我们也希望减缓滥用。我们的缓解措施包括安全训练、自动化监控、对高级能力实行可信访问，以及包含威胁情报在内的执行机制。

我们正在投资生态系统层面的保障措施。例如，我们正在扩大安全研究智能体 Aardvark 的私有测试范围。我们也在与开源维护者合作，为广泛使用的项目提供免费的代码库扫描。

在 2023 年启动的 Cybersecurity Grant Program 基础上，我们还将投入价值 1000 万美元的 API 额度。这个项目将使用我们能力最强的模型推动网络防御，重点支持开源软件和关键基础设施系统。从事善意安全研究的组织，可以通过 Cybersecurity Grant Program 申请 API 额度和相关支持。

我们公开 EVMbench 的任务、工具和评估框架，以支持学术界和业界继续研究如何衡量并管理正在出现的 AI 网络安全能力。
