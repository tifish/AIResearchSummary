一句话总括

EVMbench 用真实智能合约漏洞衡量 AI agent 的攻防能力，而不是只看它们是否会解释代码。

这篇文章的核心贡献，是把 AI 安全评测放进有经济意义、可复现、可执行的区块链环境中：模型要么找出漏洞，要么修好漏洞，要么在沙盒链上实际完成攻击。

$100B+ 智能合约保护的开源加密资产规模
117 来自 40 次审计的精选高危漏洞
71.0% GPT-5.3-Codex 在 exploit 模式中的得分

评测对象从“理解漏洞”变成“在链上验证能力”

EVMbench 包含 detect、patch、exploit 三种任务：detect 看 agent 能否审计代码并召回真实漏洞，patch 看它能否在不破坏功能的前提下修补漏洞，exploit 看它能否在本地沙盒链上完成端到端资金抽取。

这改变了读者对 AI 影响的理解：AI 安全能力不能只靠自然语言解释来判断，必须放到可执行环境中检验结果是否真的成立。

漏洞集足够贴近现实，但不是现实安全的全部难度

基准包含 117 个漏洞，来自 40 次审计，多数源自公开代码审计竞赛，并加入了 Tempo 区块链安全审计中的支付相关智能合约场景。

这些样本让 benchmark 有现实经济语境，但文章也承认，广泛部署的核心加密合约通常经历更严苛审查，可能比这些任务更难攻破；结论是，EVMbench 是能力测量工具，不是现实世界风险的完整上限。

最强模型在明确攻击目标上进步最快

在 exploit 模式中，GPT-5.3-Codex 通过 Codex CLI 得分 71.0%，而约六个多月前发布的 GPT-5 得分为 33.3%。

这个差距说明 agent 在“目标清晰、反馈可执行”的环境里提升很快；洞见是，AI 的网络安全影响会先在可自动验证、可迭代试错的任务中显著放大。

发现和修补漏洞仍明显难于利用漏洞

文章指出，detect 和 patch 的表现仍低于完整覆盖：detect 中 agent 可能找到一个问题就停止，patch 中则要在消除漏洞的同时保持合约原有功能。

这说明防御任务比攻击任务更需要完整性、保守性和上下文判断；结论是，AI 可以增强审计流程，但还不能替代严谨的人类安全工程。

可复现沙盒降低了评测风险，也限制了覆盖范围

EVMbench 使用 Rust harness 部署合约、确定性重放交易，并限制不安全 RPC；exploit 任务运行在本地 Anvil 环境中，不接触真实网络。

这种设计让评测更客观、更安全，但也排除了依赖精确时序、多链交互或真实主网状态的行为；洞见是，高质量安全 benchmark 必须在可控性和真实性之间权衡。

OpenAI 把基准发布和防御生态建设绑定在一起

文章将 EVMbench 放在双重用途背景下讨论：AI 会帮助攻击者，也会帮助防御者，因此需要安全训练、自动监控、可信访问、威胁情报和执行管线。

OpenAI 还提到扩展 Aardvark 安全研究 agent 的私有 beta，并通过 2023 年启动的 Cybersecurity Grant Program 承诺 $10M API credits 支持开源软件和关键基础设施防御；结论是，能力评测只有和防御部署一起推进，才有实际治理意义。

最值得带走的洞见

  • AI agent 在可执行、反馈明确的攻击任务中提升很快，安全团队需要提前适应这种速度。
  • 发现全部漏洞和安全修补仍是更难的问题，不能把 exploit 成功率等同于完整审计能力。
  • 智能合约安全是衡量 AI 网络能力的高价值场景，因为失败会直接对应真实资产风险。
  • 可复现沙盒 benchmark 能更清楚地区分模型能力、环境漏洞和评测作弊。
  • 防御性使用 AI、受控开放高级能力和生态支持，将决定这类技术的净安全效果。
