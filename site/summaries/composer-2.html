Composer 2：把"前沿编码智能"和"低价"装进同一个模型

来源：Cursor 团队 · 2026-03-19 · cursor.com/blog/composer-2

一句话总括：Cursor 的真正贡献不在于"又一个编码模型"，而在于它用 首次持续预训练打造强基座 + 强化学习训练长程任务 的组合，把能完成数百步操作的前沿编码能力，做到了远低于同档模型的价格。

定价：性价比是这次发布的核心卖点

$0.50 / $2.50
标准版 输入 / 输出（每百万 token）
$1.50 / $7.50
快速版（同等智能，已设为默认）

同样的智能水平给出两档价格：标准版极低，快速版虽贵一些但仍低于其他"快速模型"，并被直接设成默认选项。个人套餐里 Composer 还独立计入一个附带较多免费额度的用量池。

价格不再是高质量模型的天花板——当"快"和"贵"被解耦，厂商可以把性价比本身当作产品策略来卖。

能力来源：先重建基座，再用强化学习教它干长活

质量跃升来自 Cursor 的第一次持续预训练（continued pretraining），它提供了一个更强的基座，让后续强化学习"有得可放大"。在此之上，模型针对长程编码任务做强化学习，因而能解决需要数百个连续动作的复杂任务。

编码模型的竞争正从"单步答得准"转向"能否独立跑完几百步的真实工程任务"，而这要靠基座+RL 的链路，而非单纯堆参数。

基准：在严格、可复现的设置下报告提升

Terminal-Bench 2.0
Laude Institute 维护的终端 agent 评测
SWE-bench Multilingual
多语言软件工程基准
5×
每个模型-agent 组合跑 5 次取平均

评测用官方 Harbor 框架、默认设置；对其他模型，取"官方榜单分"与"自有基础设施实测分"中的较高值进行对比，避免被指压低对手。TPS 数据取自 2026-03-18 的 Cursor 流量快照，并对 Anthropic 约小 15% 的 token 做了归一化。

把评测口径（框架、迭代次数、对手取最高分、token 归一化）写清楚，本身就是可信度的一部分——读基准时该先看这些条件，再看分数。

最值得带走的洞见

  • Composer 2 的看点是"前沿智能 + 低价"的组合定位，性价比被当作核心竞争力。
  • 标准版 $0.50/$2.50、快速版 $1.50/$7.50（每百万 token，输入/输出），且快速版设为默认。
  • 能力来自"持续预训练打基座 + 强化学习训长程任务"，可完成数百步操作的复杂任务。
  • 提升在 Terminal-Bench 2.0、SWE-bench Multilingual 上以严格、可复现的设置报告（Harbor 框架、5 次取平均、对手取最高分）。
  • 趋势信号：编码模型竞争从"单步准确"转向"长程自主完成真实工程任务"，同时把价格门槛持续压低。
