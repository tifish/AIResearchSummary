推出 Composer 2

Introducing Composer 2

一句话总括：Composer 2 把前沿水平的编程能力、长流程智能体训练和更低的 token 价格组合在一起，目标是让复杂编码任务更具成本效益。

来源：Cursor 作者：Cursor Team 分类：Research

先看数字：Composer 2 的核心卖点是能力与成本的组合

$0.50/M 标准版每百万输入 token 的价格
$2.50/M 标准版每百万输出 token 的价格
数百次操作 复杂长周期任务可能需要的动作规模
5 次 每个模型与智能体组合的评测迭代次数

能力提升不只来自强化学习

Cursor 首次加入持续预训练，为后续的长周期强化学习提供了更强的基础模型。

训练路径

团队先完成首次持续预训练，再针对长周期编程任务开展强化学习。这个顺序意味着能力提升并非只靠最后阶段的奖励优化。

为何重要

Composer 2 可以处理需要数百次操作的挑战性任务。AI 编程能力因此不再只体现为单次补全质量，也体现为能否稳定推进较长的执行链。

从基础模型到长流程编码智能体

1

持续预训练

先建立更强的基础能力，为后续训练提供更高的起点。

2

强化学习

训练重点转向需要持续规划和执行的长周期编程任务。

3

数百次动作

最终目标是完成多步骤挑战，而不是只解决孤立的小问题。

结论：这条训练路线把模型能力的衡量重点，从局部代码生成推向了持续执行和任务完成。

!

基准结果需要结合评测口径阅读

原文称 Composer 2 在 Terminal-Bench 2.0 和 SWE-bench Multilingual 等评测中明显进步，但正文没有列出具体分数。Terminal-Bench 2.0 的结果来自官方 Harbor 框架，每个模型与智能体组合运行 5 次，随后报告平均值。其他模型则采用官方榜单与 Cursor 内部运行结果中的较高值。

评测框架并不完全相同：Anthropic 使用 Claude Code harness，OpenAI 使用 Simple Codex harness，Cursor 则使用 Terminal-Bench 2.0 指定的 Harbor。跨模型比较时，不能只看最终柱状图。

快速版

快速版保持相同智能水平。每百万输入和输出 token 的价格分别是 1.50 美元和 7.50 美元。

Cursor 将快速版设为默认选项。它面向更看重响应速度，同时仍希望控制成本的用户。

速度比较经过归一化

速度数据取自 2026 年 3 月 18 日的 Cursor 流量快照。由于 Anthropic token 约小 15%，TPS 和非 Anthropic 模型的输出价格经过了对应调整。

最值得带走的洞见

01 · 成本是产品定位的一部分

标准版输入价格为每百万 token 0.50 美元，输出价格为 2.50 美元。Composer 2 想证明，高水平编程能力不一定必须伴随最高一档价格。

02 · 长流程能力成为重点

能够执行数百次操作，比一次生成一段正确代码更接近真实的软件开发工作。

03 · 分数不能脱离框架

不同模型使用不同 harness。评测次数、取值方法和运行环境都会影响横向比较。

04 · 快速版换取延迟优势

快速版价格是标准版的三倍，但保留相同智能水平。它体现的是速度与费用之间的选择，而不是能力等级的变化。
