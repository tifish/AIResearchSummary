实践中衡量 AI agent 自主性

Measuring AI agent autonomy in practice

Anthropic 的真正贡献不是证明 agent 已经高度自主，而是给出了一套观察真实部署的测量框架：看用户给了多少自由度，模型何时主动停下，产品如何改变监督方式，以及风险是否正在向高影响领域扩散。

约 45 秒中位 turn duration 仍很短，说明多数使用还不是长时间放手。
40%+750 次会话经验用户使用 full auto-approve 的比例超过 40%。

自主性在尾部增长，而不是在日常任务中爆发

大多数 Claude Code turn 很短。中位数约为 45 秒，低于第 99 百分位的区间也基本稳定。

真正变化发生在最长的任务。2025 年 10 月到 2026 年 1 月，第 99.9 百分位 turn duration 从不到 25 分钟升至超过 45 分钟。

这改变了一个常见理解：agent 自主性不是所有使用场景一起跃升，而是先在最有经验、最复杂、最愿意放手的使用中拉开距离。

能力评测不能替代部署测量

METR 估计 Claude Opus 4.5 能以 50% 成功率完成相当于人类近 5 小时的任务。

Claude Code 的实际第 99.9 百分位约 42 分钟，中位数更短。两者不是矛盾。

评测衡量模型在理想条件下能做什么。真实部署衡量用户、产品和模型共同允许它做多久。

用户监督从逐步批准转向持续监控

经验越多，用户越少逐个批准动作，越多让 Claude 先做，再在需要时干预。

1

新用户更保守

少于 50 次会话的新用户，约 20% 会话使用 full auto-approve。默认设置也鼓励他们逐步批准。

2

经验带来放手

到 750 次会话时，full auto-approve 超过 40%。这说明信任是逐步积累出来的。

3

放手不等于不管

新用户约 5% 的 turn 会打断 Claude，经验用户约 9%。监督变成了监控和及时纠偏。

风险目前可控，但边界已经出现

公共 API 的多数 agent 行动低风险、可逆。 80% 工具调用似乎至少有一种保护措施，73% 似乎有人类参与。

但 Anthropic 也观察到医疗、金融、网络安全等高影响领域的早期使用。很多高风险样本可能是评估或红队测试，却仍说明边界正在扩张。

这让读者看到一个政策重点：不能只看平均风险。少量高自主、高风险部署也可能带来严重后果。

保护措施：权限限制、人类批准或其他约束在多数工具调用中可见。
人类参与：估计 73% 有某种 human-in-the-loop，但该数字可能偏高。
不可逆行动：仅 0.8% 似乎不可逆，例如向客户发送邮件。

不同角色应该关注什么

对象 文章给出的信号 应得到的结论
模型开发者 Claude Code 在复杂任务中主动澄清的频率高于人类打断。 让模型识别自身不确定性，是和权限系统同等重要的安全能力。
产品开发者 经验用户更常自动批准，也更常打断。 好监督不是强制每步审批，而是让用户看得见、插得进、改得动。
政策制定者 高风险领域尚未规模化，但正在出现。 现在不宜规定单一交互模式，更应要求可监测、可追踪、可干预。
自主性不是模型的固定属性。它来自模型是否会停下、用户是否愿意放手，以及产品是否让人能在关键时刻介入。

最值得带走的洞见

真实部署比榜单更关键。 预部署评测告诉我们模型能做什么，部署数据告诉我们人们实际让它做什么。
风险不能只看平均值。 大部分使用低风险，并不排除少数高影响场景需要更强监测。
监督正在换形态。 经验用户不是退出监督，而是从审批者变成监控者。
软件工程是先行样本。 它占 agentic 活动近 50%，但其他行业的验证成本更高，信任曲线可能不同。
下一步是监测基础设施。 要理解 agent，就要能在保护隐私的前提下连接会话、识别风险、观察干预。
