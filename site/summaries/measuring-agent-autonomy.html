一句话总括

现实中的 AI 代理自主性正在变强，但关键变量是监督结构，而不只是模型能力。

Anthropic 的贡献在于用 Claude Code 和公共 API 的真实使用数据，把“代理能做什么”拆成了“用户给它多少空间、模型何时主动停下、产品如何让人介入”这三个可观察问题。

最长尾任务显示自主运行时间明显拉长

Claude Code 的 99.9 分位单轮时长从 2025 年 10 月到 2026 年 1 月由 不到 25 分钟 增至 超过 45 分钟，而中位数仍约 45 秒。这说明变化主要发生在最复杂、最雄心勃勃的用法上。

用户经验让监督从逐步批准转向按需干预

新用户约 20% 会话全自动批准，到约 750 次会话经验后超过 40%；同时打断率从约 5% 升至 9%。这不是放弃监督，而是把监督从“每步盖章”改成“持续观察并在关键处介入”。

代理自主性是部署结果，不是模型的单一属性

文章把代理定义为能使用工具采取行动的 AI 系统，并分别用 Claude Code 的完整会话和公共 API 的单个工具调用观察现实行为。Claude Code 能看到任务从开始到停止的全过程，API 则覆盖数千个客户场景。

这个方法改变了读者对 AI 影响的理解：评估代理不能只看基准测试中的最大能力，还要看真实用户愿意给它多少权限、它是否会主动请求澄清、以及产品是否能支持人类及时介入。

长任务变长并不等于能力突然跃迁

最长尾时长的增长跨模型发布过程较平滑，没有随着每次模型发布出现突变；内部使用中，高难任务成功率从 8 月到 12 月翻倍，同时每会话人工干预从 5.4 次降到 3.3 次。

结论是存在“部署滞后”：模型在理想环境下能处理的自主性，高于用户和产品在现实工作流中实际释放的自主性。

有经验的用户不是更少监督，而是换了一种监督

Claude Code 默认要求用户逐步批准操作，因此新用户更常停留在审批式监督；经验增加后，用户更常开启自动批准，并在代理偏离目标时打断。公共 API 也有类似趋势：极简任务约 87% 工具调用有人类参与，高复杂任务为 67%。

这提醒产品和政策设计者，要求每一步都人工批准可能制造摩擦，却未必带来更好安全性；更重要的是让人看得见代理在做什么，并能快速纠偏。

模型主动停下也是一种安全机制

在更复杂任务中，Claude Code 主动请求澄清的频率上升，并且在最复杂任务上比人类打断更常见。文章承认这不代表每次停下都正确，但它表明模型自身的不确定性感知可以成为监督的一部分。

洞见在于，代理安全不只依赖外部权限墙和人工审批，也依赖模型在不确定、目标含糊或风险升高时主动把控制权交回给人。

当前总体低风险，但高风险边界已经出现

公共 API 中多数工具调用低风险且可逆：约 80% 看似至少有一种防护，73% 有某种人在环，只有 0.8% 看似不可逆。软件工程占近 50% 的代理活动，医疗、金融和网络安全等更高风险领域仍处于早期试验。

结论是平均风险数字让人相对安心，但治理不能只看平均值；真正值得监测的是高自主、高风险的小规模前沿用例如何扩张。

这项研究的限制同样重要

数据只来自 Anthropic，Claude Code 主要是软件工程场景，API 分析只能看单个工具调用，风险和自主性分类由 Claude 推断，且时间窗口集中在 2025 年末到 2026 年初。

因此，文章更像是一套早期测量框架，而不是关于所有 AI 代理的最终结论；它的政策价值在于指出应该持续监测哪些信号。

最值得带走的洞见

  • 现实中的代理自主性正在增长，尤其体现在最长尾、最复杂任务上。
  • 有效监督不等于逐步批准；资深用户更常让代理先做，再在关键节点介入。
  • 模型主动请求澄清是重要安全能力，应该与权限控制和人类监督一起设计。
  • 当前大多数代理行为低风险，但医疗、金融、网络安全等领域的前沿用例需要部署后监测。
  • 政策不宜过早规定固定交互模式，应优先要求可见性、可干预性和隐私保护下的真实使用监测。
