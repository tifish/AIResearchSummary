实践中的 AI 智能体自主权测量

基于 Claude Code 与公共 API 数百万次真实人机交互的隐私保护分析
这篇研究真正的贡献，不是预测智能体能做什么，而是第一次用第一方真实流量测量"人们实际允许智能体做多少"——结论是：模型已经具备的自主能力，远超它在现实部署中被赋予的发挥空间。这种"部署冗余"意味着，安全的关键不在限制能力，而在理解人、模型、产品三方如何共同塑造自主权。

1智能体的自主运行时长正在拉长，但这不全是"能力变强"

25→45分钟
最长会话（99.9 分位）3 个月翻倍
~45 秒
单次运行中位数（长期稳定）

2025 年 10 月到 2026 年 1 月，尾部最雄心勃勃的任务运行时长几乎翻倍。关键在于：这条曲线在各次模型发布之间是平滑增长的，而非随新模型上线出现跳跃。如果纯由能力驱动，应该看到阶梯式突变。

平滑增长说明自主权是被逐渐"信任出来"的——用户建立信任、产品持续优化共同推动，而非单一能力解锁。这改变了"模型一升级就更危险"的直觉。

2越老练的用户，越敢放手，却也越频繁地打断

20%→40%+
全自动批准会话占比（新手→750 次会话）
5%→9%
每轮中断率（新手→资深）

这看似矛盾：放手更多、却打断更勤。其实是监督策略的转变——新手逐项审批，因此很少需要中途打断；资深用户让 Claude 自主跑，只在出错或需要纠偏时介入。更高的中断率反而反映了"知道何时该出手"的主动监控。公共 API 上也印证：最简单任务 87% 的工具调用有人参与，最复杂任务降至 67%。

"有效监督"不等于"审批每一步"，而是处于能在关键时刻介入的位置。逐步审批在长任务上结构性地不可行。

3Claude 主动喊停的次数，比人类打断它还多

2×+
最复杂任务上，Claude 主动澄清频率高于人类中断

任务越难，Claude 越频繁地停下来询问澄清，且超过人类主动中断的频率。这说明模型对自身不确定性有一定校准能力——它会在本可继续时主动限制自己的自主权，去征求人的意见。（需谨慎：它也可能问得过多或停错时机。）

"智能体主动停下"本身是一种被低估的监督机制。训练模型识别并表达自身不确定性，是与权限系统、人工审批互补的安全属性。

绝大多数操作是低风险、可逆的。但研究已观察到风险与自主权前沿的新用法——安全相关操作、金融交易、医疗信息（其中许多疑似是评估/红队测试）。当前智能体高度集中于软件工程，而医疗、金融、网络安全等更高风险领域正在萌芽。软件之所以先行，是因为其输出容易测试和复核、易于建立信任。

平均值令人安心，却会掩盖前沿部署。随着采用扩散到难以验证输出的高风险领域，风险—自主权前沿必将扩张——这正是需要持续监测的地方。

5对照与佐证：能力与实践之间的差距

Anthropic 内部数据显示，8 月至 12 月最难任务成功率翻倍，同时每次会话人类干预从 5.4 次降至 3.3 次——干预更少、结果更好。对照 METR 评估：Opus 4.5 能以 50% 成功率完成需人类近 5 小时的任务，而实际尾部运行仅约 42 分钟。两者口径不同（METR 测任务难度，本研究测真实耗时），但合在一起指向同一结论。

能力评估测"理想环境下能做什么"，真实测量测"现实中实际发生什么"。二者缺一不可，且共同证明：实践中授予模型的发挥空间，落后于其真实能力。

最值得带走的洞见

  1. 存在"部署冗余"。模型能承担的自主程度，超过现实中它被允许行使的——安全瓶颈在信任与产品设计，而非纯能力。
  2. 自主权是"共同建构"的。它不是模型的固有属性，而由模型行为、用户监督策略、产品设计三方共同涌现，因此无法仅靠部署前评估刻画。
  3. 监督的本质是"能介入"，不是"审批一切"。资深用户从逐项批准转向主动监控+按需纠偏；强制审批每一步只增摩擦、未必增安全。
  4. 让模型会"喊停"是关键安全属性。Claude 主动澄清比人类打断更频繁，模型识别自身不确定性应成为行业训练目标。
  5. 部署后监测基础设施仍处早期且不可或缺。许多发现无法靠部署前测试获得；跨厂商以隐私保护方式把零散请求重建为"智能体会话"，是亟需的共同课题。
