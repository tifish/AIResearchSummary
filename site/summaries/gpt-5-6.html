GPT-5.6: Frontier intelligence that scales with your ambition

效率不再只是“小模型”的卖点

GPT-5.6 Sol 在 Agents’ Last Exam 上达到 53.6。它比 Claude Fable 5 高出 13.1 分。即使只使用中等推理强度，Sol 仍领先 11.4 分，估算成本约为对手的四分之一。

Terra 和 Luna 也超过了 Fable 5，估算成本约为后者的十六分之一。这里的核心变化不是单纯提高峰值分数，而是让高水平智能能够覆盖更多日常任务。

按需购买更多思考

max 会投入比 xhigh 更多的推理时间，用于探索、检查和修订。

ultra 默认协调 4 个代理并行工作。它消耗更多令牌，但能在高难度任务上同时提高结果质量并缩短等待时间。

这意味着用户可以先采用高效默认设置，再把额外计算集中投入真正困难的问题。

从回答问题到完成端到端工作

  1. 理解任务。 模型从文档、Slack、Notion、Microsoft 365 和 Google Drive 等来源整理混乱信息。
  2. 组织工具。 Programmatic Tool Calling 允许模型编写轻量程序，筛选中间结果并动态决定下一步。
  3. 检查成品。 更强的计算机操作和设计判断让模型能够查看渲染结果，发现视觉或功能问题。
  4. 并行推进。 ultra 和 Responses API 的多代理能力可以拆分工作流，再汇总各代理的成果。

真正的提升来自工作闭环：模型不只生成内容，还能检查、修改、验证并交付可直接使用的成果。

在编码、知识工作和研究场景中，GPT-5.6 的关键指标不只是“答得更好”，而是用更少的轮次和令牌完成更多可验证的工作。

能力扩展伴随着更保守的安全边界

能力判断

GPT-5.6 在生物和网络安全领域都更强，但 OpenAI 表示它尚未跨越任一领域的 Critical 阈值。网络安全测试显示，它更擅长发现和修复漏洞，还不能稳定地自主完成针对加固目标的端到端攻击。

防护代价

Sol 的网络安全防护会拦截约为此前模型 10 倍的潜在有害活动。这可能干扰正常工作，因此 ChatGPT 和 Codex 提供在能力较低模型上重试的选项。

模型采用训练内防护、实时检查、推理监控、持续监测和账户级执行等多层机制。约 70 万个 A100 GPU 等效小时的黑盒自动红队测试，也说明安全成本正随能力同步增长。

供应范围与定价

在有限预览结束后，我们正式推出 GPT-5.6 模型家族。其中包括新的旗舰模型 Sol、面向日常工作的均衡型模型 Terra，以及成本效率最高的模型 Luna。

GPT-5.6 Sol 为智能水平与效率树立了新标准。它在编码、知识工作、网络安全和科学领域取得了最先进的成绩。同时，它以更少的令牌和更低的估算成本，超过此前的模型及其他前沿模型。

最终结果是更高的单位成本性能。用户可以用相同预算完成更多任务，也可以用更低的总成本获得相近结果。

我们还引入了一种加速高难度工作的新方式。ultra 是能力最高的设置。它会协调多个代理，在并行工作流中共同完成任务，从而更快处理复杂问题。

更强的计算机操作能力和设计判断，也让 GPT-5.6 Sol 成为我们迄今打磨最完善的协作模型。它可以检查、改进并交付可直接使用的成果。

我们在训练 GPT-5.6 时，重点是让每个令牌都能产生更多有用工作。在 Agents’ Last Exam 这项覆盖 55 个领域、用于评估长时间专业工作流的测试中，GPT-5.6 Sol 创下 53.6 的新高，比 Claude Fable 5（自适应推理）高 13.1 分。

即使采用中等推理强度，它仍比 Fable 5 高 11.4 分，估算成本约为后者的四分之一。

这种效率也延伸到了更小的模型。小模型对于让智能变得更普及、更实惠至关重要。GPT-5.6 Terra 和 GPT-5.6 Luna 都超过了 Fable 5，估算成本约为后者的十六分之一。

Artificial Analysis Intelligence Index 是一项综合衡量智能水平的指标，覆盖代理工作、编码、科学推理和通用能力。在最大推理强度下，GPT-5.6 Sol 与 Fable 5 的差距不到 1 分，但完成任务的时间减少了 61%，估算成本约为一半。

Agents’ Last Exam：覆盖多个专业领域的长周期代理工作流评测。

GPT-5.6 发布时配备了我们迄今最稳健的防护措施。这些措施旨在抵御蓄意且会不断调整策略的滥用，同时避免对正当工作施加宽泛限制。

在正式开放之前，我们对模型和防护措施开展了迄今周期最长、范围最广的评估。评估结合了人工红队测试和大规模自动化测试。

预览期间，我们与专家机构及可信合作伙伴密切合作，对防御措施进行压力测试，并在更广泛发布前强化防护。

最终形成的系统把模型训练中内置的保护，与实时检查、监控以及根据可信度和风险调整的访问权限结合起来。

默认高效，按需发挥最高性能

GPT-5.6 Sol 是我们迄今最强的编码模型。在 Artificial Analysis Coding Agent Index 上，采用最大推理强度的 GPT-5.6 Sol 取得 80 分，刷新最高纪录，比 Fable 5 高 2.8 分。

与此同时，它使用的输出令牌不到一半，耗时不到一半，成本约低三分之一。

这种优势也覆盖整个模型家族。Terra 的表现略高于 Fable 5，Luna 则超过 Opus 4.8。两者的耗时都约为对手的三分之一，输出令牌约为一半，估算成本约为四分之一。

GPT-5.6 还在 Terminal-Bench 2.1 和 DeepSWE 上取得新的最佳成绩。这两项评测分别测试复杂命令行工作流，以及在真实代码库中进行长周期工程工作的能力。

Artificial Analysis Coding Agent Index：一项独立的编码代理性能指数，覆盖实现任务、终端操作和真实代码库。

GPT-5.6 可以编写并运行轻量程序。这些程序能够协调工具、处理中间结果、监控进度，并随着工作推进决定下一步操作。

因此，工具密集型任务可以使用更少的令牌、更少的模型往返和更少的人工指导来推进。

开发者不再需要为每一步编写固定脚本，也不必把每个工具响应都重新传回模型。Responses API 中的 Programmatic Tool Calling 可以筛选大量中间数据，只保留重要内容，并在执行过程中调整工作流。

对于值得投入更多时间和算力的问题，GPT-5.6 可以突破高效默认设置的限制。

max 为 GPT-5.6 提供比 xhigh 更长的时间，用于推理、探索备选方案、运行检查并修订处理方式。

ultra 更进一步。它默认协调 4 个代理并行工作。它会使用更多令牌，但能在高难度任务中取得更强结果，并缩短获得结果所需的时间。

推动网络安全与科学前沿

GPT-5.6 是我们迄今最强的网络安全模型。它以明显更少的令牌达到了前沿水平。

ExploitBench 2 衡量模型从接触存在漏洞的代码，到实现任意代码执行的进展。在相近的输出令牌预算下，GPT-5.6 得分为 73.5%，GPT-5.5 为 47.9%。

ExploitGym 3 要求代理把现实世界中的漏洞转化为可运行的漏洞利用程序。在两小时上限内，GPT-5.6 几乎把 GPT-5.5 的最高通过率提高了一倍，从 15.1% 增至 24.9%。时间增加到六小时后，它达到 33.7%。

SEC-Bench Pro 测试模型为复杂软件生成概念验证的能力。GPT-5.6 得分为 71.2%，GPT-5.5 为 45.8%，而且延迟也有所改善。1

GPT-5.6 可以支持安全代码审查、修补、威胁建模和蓝队行动等重要防御任务。

符合条件的个人和组织可以通过 OpenAI Daybreak 的 Trusted Access for Cyber 计划，在已授权环境中获得更精细的防护和更多防御能力。这些工作包括漏洞分类与验证、恶意软件分析、检测工程和补丁验证。

个人可以验证身份并申请可信访问，组织也可以为团队提出申请。

个人成员需要在 9 月 1 日前启用 Advanced Account Security，并使用硬件支持的通行密钥，才能继续访问网络安全能力最强的前沿模型。未完成设置的成员将恢复为默认访问权限。

还没有硬件支持通行密钥的用户，可以从我们的合作伙伴 Yubico 获得优惠价格。我们也在采取更多措施，限制高风险实体以及高风险司法辖区的访问。

ExploitBench：测试模型构建能力逐步增强的 V8 漏洞利用程序。GPT-5.6 相比 GPT-5.5 有大幅提升。由于无法可靠估算这项 benchmark 的延迟，因此未展示延迟图。

GPT-5.6 Sol 在科学研究领域也取得了广泛进步。在生命科学评测中，GPT-5.6 相比 GPT-5.5 实现了帕累托改进，覆盖真实世界生物学、生命科学研究工作流和化学。

GeneBench Pro：涵盖长周期基因组学和定量生物学分析。GPT-5.6 用更少的令牌和时间取得了更好结果。Claude Fable 5 未被纳入比较，因为它不会回答高级生物学问题，并拒绝了这项评测中的大多数问题。

GPT-5.6 加速 OpenAI

GPT-5.6 是我们迄今最适合加速 AI 研究的模型。

在 OpenAI 内部，研究人员会在整个开发循环中使用它，包括诊断故障、优化训练系统、运行实验和解读结果。

GPT-5.6 内部测试期间，我们已经观察到这种加速效果和更高的采用率。每位活跃研究人员平均每天产生的输出令牌，比 GPT-5.5 曾达到的最高水平高出一倍以上。

这种工作方式正在迅速成为常态。

过去六个月，用于内部编码推理的研究算力增长了 100 倍，内部代理令牌使用量增长约 22 倍。

这些采用指标本身不能衡量研究进展。但它们显示，AI 辅助正在研究工作中快速增长，也在销售、市场、用户运营和财务等其他团队中迅速普及。

为了直接衡量这种能力，我们开发了一套内部评测。它以真实 AI 研究任务为基础，包括调试研究系统、优化内核和训练方案、运行机器学习实验，以及改进另一个模型。

综合 RSI 能力：在一组衡量递归自我改进进展的评测中，GPT-5.6 Sol 比 GPT-5.5 提高 16.2 分，全面加速了内部研究。

让安全与防护随能力共同扩展

随着模型能力提高，我们也会强化安全体系。目标是让高级智能继续被广泛用于有益工作，同时对风险最高的用途进行更严格的审查。

针对 GPT-5.6，我们构建了迄今最稳健的安全系统。系统会根据各模型的能力进行校准，并投入了比以往更多的计算资源。

GPT-5.6 模型在生物学和网络安全领域都比此前模型更强，但在这两个类别中均未跨越 Critical 阈值。

在网络安全方面，测试表明 GPT-5.6 更擅长发现和修复漏洞。它还无法稳定自主地对加固目标实施端到端攻击。这给防御方留下了在弱点被利用前加固系统的机会。

在生物学方面，测试显示 GPT-5.6 可以支持正当研究，但不具备创造、改造或合成高度危险的新型威胁所需的端到端能力。

这两个领域本身都具有双重用途。

在网络安全中，能够帮助攻击者利用漏洞的能力，也能帮助防御者发现漏洞、复现问题并构建可靠修复方案。

因此，过度拦截本身也会带来安全风险。它可能阻止防御者测试系统和部署补丁，而恶意行为者仍可使用其他模型。其中包括能力不断增强的开源模型，以及已经成熟的工具。

有效的防护必须考虑请求的上下文和可能后果。它需要保留正当防御工作，同时在证据表明存在严重伤害风险时施加更严格的控制。

GPT-5.6 的防护采用分层设计，以提高准确性和冗余度。它们也能随着新攻击出现而快速调整。

模型训练中内置的保护，会与实时检查、持续监测和账户级执行共同工作。即使某一层未能按预期发挥作用，系统仍能保持安全。

在许多系统中，是否拦截完全由分类器标记决定。这些系统依赖智能水平较低、也较难调整的模型来阻止伤害。

我们的方法增加了一个推理监控器。它会审查对话，并判断是否可能造成伤害。

这种设计旨在支持防御工作，同时阻止严重滥用。最敏感的能力则通过 Trusted Access 保留给已验证用户。

部分保护措施使用测试时推理，因此我们可以迅速更新它们来修补缺口，而不必从头重新训练分类器。

在继续加强系统抵御自适应攻击的能力时，我们采取了更保守的方式。

与此前模型相比，GPT-5.6 Sol 的网络安全防护会拦截约 10 倍的潜在有害活动。

这些措施可能给正当使用带来阻碍。因此，我们在 ChatGPT 和 Codex 中提供了一个选项，让用户可以方便地在能力较低的模型上重新尝试提示。

我们会继续降低防护措施对正常使用的影响，同时维持严格的稳健性标准。

这反映了我们的迭代部署方式。我们会先采取保守设置，再根据真实使用中获得的信息持续改进。

正式开放之前，我们开展了迄今强度最高的安全评估。其中包括大量红队测试、由外部专家参与的能力与防护稳健性测试，以及约 70 万个 NVIDIA A100 Tensor Core GPU 等效小时的黑盒自动化红队测试。

这些工作让我们可以系统检查可能的薄弱点，发现越狱方法，并在发布前强化系统。

世界上不存在完美的安全机制。随着模型能力持续增强，我们的安全工作也必须继续。

新的弱点仍会出现，也会出现绕过现有防护的新越狱方式。每一代新模型还会带来新的攻击和滥用途径。

我们针对这种现实构建系统，采用分层防护、持续监测、快速修复和整个防御社区的协作。

针对 GPT-5.6，我们在现有安全和生物学漏洞赏金计划之外，又增加了新的快速修复流程和迄今力度最大的监测措施。

研究人员提交的发现、监测结果和现实世界中的滥用案例，将持续转化为新的评测和更强的防护。

如需了解更多防护信息，请阅读更新后的 GPT-5.6 system card。

供应范围与定价

GPT-5.6 分为三个模型层级。Sol 是旗舰模型；Terra 成本更低，性能可与 GPT-5.5 竞争；Luna 速度最快，价格也最低。

数字代表模型代际。Sol、Terra 和 Luna 则是长期保留的能力层级，可以按照各自节奏继续升级。

GPT-5.6 从今天起在 ChatGPT、Codex 和 OpenAI API 中提供。全球发布现已开始，并将在未来 24 小时内逐步扩大到全面可用。

Chat

Plus、Pro、Business 和 Enterprise 用户可以通过中等及更高的推理强度使用 GPT-5.6 Sol。Pro 和 Enterprise 用户还可以选择 GPT-5.6 Sol Pro，以便在复杂任务中获得最高质量的结果。

ChatGPT Work 和 Codex

Free 和 Go 用户可以使用 GPT-5.6 Terra。

Plus、Pro、Business 和 Enterprise 用户可以在 GPT-5.6 Sol、Terra 和 Luna 之间选择，并为每个模型设置推理强度。

在 ChatGPT Work 和 Codex 中，所有能够使用 GPT-5.6 的用户都可以使用 max，并可在设置中将其开启。

在 ChatGPT Work 中，Pro 和 Enterprise 用户可以使用 ultra。在 Codex 中，Plus 及更高等级方案可以使用 ultra。

API

开发者可以通过 OpenAI API 使用 Sol、Terra 和 Luna。

在 Responses API 中，Programmatic Tool Calling 允许 GPT-5.6 在内存中编写并运行程序，用于协调工具和处理中间结果。因此，这项能力与零数据保留（Zero Data Retention，ZDR）兼容。

multi-agent 功能最初以测试版提供。它允许 GPT-5.6 在单次请求中运行多个并发子代理，并汇总它们的工作。
