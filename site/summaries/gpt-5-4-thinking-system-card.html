GPT‑5.4 Thinking 系统卡

来源 OpenAI ｜ 分类 Publication ｜ 2026-03-05

一句话总括：GPT‑5.4 Thinking 是 GPT‑5 系列最新的推理模型，而这张系统卡真正的看点不是又一次跑分，而是它成为首个在网络安全能力上被评为"High"并据此实施缓解的通用模型——把过去只出现在专用编码模型上的高风险门槛，第一次落到了面向大众的通用模型身上。
说明：本页所依据的抓取正文目前只包含系统卡的引言，尚未包含具体的安全评测数值、红队结果或各能力分项评分。下文仅就引言中可核实的信息展开，不对未披露的指标做推断。

它是什么GPT‑5 系列里最新的"会思考"的那一个

GPT‑5.4 Thinking（代号 gpt-5.4-thinking）是 GPT‑5 系列最新的推理模型。它的整体安全缓解方法论与该系列前代保持一致——也就是说，框架没有推倒重来，变化集中在某一类风险的处置上。

结论：把它读作"系列内的常规迭代 + 一处关键升级"，而不是全新体系。

真正的新意第一个把网络安全缓解做到"High"等级的通用模型

这是引言里最重要的一句：5.4 Thinking 是首个为网络安全（Cybersecurity）"High"能力等级实施缓解措施的通用模型。在 OpenAI 的准备度（Preparedness）框架中，"High"是触发额外安全缓解的高风险门槛；此前能逼近这一门槛的多是专用编码/智能体模型，如今它出现在一款通用模型上，意味着"模型有能力实质性辅助网络攻击"这件事，已经被当作一款大众产品必须正面应对的现实风险，而不是远期假设。

结论：节点性信号不在分数高低，而在"门槛被一款通用模型跨过"本身。

做法来源网络安全缓解不是凭空造，而是接续 GPT‑5.3 Codex

5.4 Thinking 的网络安全安全做法建立在 GPT‑5.3 Codex 最新方案之上，并同时覆盖 ChatGPT 与 API 两条交付路径。也就是说，专用编码模型上先行验证的缓解经验，被迁移、推广到了通用模型与全部主要使用入口。

GPT‑5.3 Codex 的网络安全缓解→ 迁移到 GPT‑5.4 Thinking（通用模型）→ 同时落地 ChatGPT 与 API

结论：高风险能力的安全做法正沿"专用模型 → 通用模型 → 全入口"扩散。

读卡须知基线是 5.2，不是 5.3——别找错对照

引言特意提醒：不存在名为"GPT‑5.3 Thinking"的模型，因此评估 5.4 Thinking 时主要对照的基线是 GPT‑5.2 Thinking。理解任何后续披露的对比数字时，参照系应落在 5.2 上，否则容易把代际跨度算错。

结论：看这张卡的对比，记住分母是 5.2 Thinking。

最值得带走的洞见

  1. 门槛被通用模型跨过，才是头条。网络安全风险首次在一款面向大众的通用模型上被评为"High"并触发缓解，意味着"AI 实质性辅助网络攻击"已从远期假设变成产品级现实问题。
  2. 安全方法在按梯度扩散。缓解做法从专用的 GPT‑5.3 Codex 迁移到通用的 5.4 Thinking，并同时覆盖 ChatGPT 与 API——专用模型正成为通用模型安全能力的"试验田"。
  3. 框架没变，处置在变。整体缓解思路沿用系列前代，真正升级的是对某一类高风险能力的针对性应对，说明厂商在用既有框架"加压"而非重建。
  4. 对照基线是 5.2 而非 5.3。因为不存在"GPT‑5.3 Thinking"，任何 5.4 的对比都应以 GPT‑5.2 Thinking 为参照，读数时别把代际跨度算错。
  5. 这只是引言，留出判断余地。本次正文未含具体评测数值与红队结果；"High"等级的实际含义，仍需以完整系统卡的分项数据为准。
