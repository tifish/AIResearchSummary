GPT‑5.4 Thinking 系统卡：第一个为"高网络安全能力"做缓解的通用模型

来源：OpenAI ｜ 分类：Publication ｜ 原文链接
一句话总括：GPT‑5.4 Thinking 真正的看点不是分数，而是治理——它是 OpenAI 第一个在通用模型上针对网络安全"High（高）"能力等级落地安全缓解措施的模型，把原本用于专用代码模型的安全分级机制，推进到了面向所有人的主力产品。

它是 GPT‑5 系列最新的推理模型

GPT‑5.4 Thinking（文中也写作 gpt-5.4-thinking）属于 GPT‑5 系列的 reasoning（推理）模型分支。它的整体安全缓解思路，与该系列的前代模型保持一致。

洞见：渐进式发布意味着安全框架是被复用而非重写的——真正值得关注的，是这一版在框架里"新增"了什么。

首次为"高网络安全能力"实施缓解，是本版的核心增量

这是第一个就网络安全 能力等级 = High 实施缓解措施的通用模型。其做法建立在 GPT‑5.3 Codex 已经在 ChatGPT 和 API 中采用的最新网络安全方法之上。

洞见：这说明前沿模型"按能力分级触发管控"不再只停留在专用代码模型，而是开始扩散到通用产品——能力越强，附带的安全义务越重，二者正被绑定。

对比基线是 GPT‑5.2 Thinking，而非 5.3

该系列中不存在名为 GPT‑5.3 Thinking 的模型，因此本系统卡用于对照的主要基线是 GPT‑5.2 Thinking。

洞见：命名跳号提醒读者：版本号是产品节奏，不是能力刻度——解读厂商进展时，要看清"和谁比"，而不是默认号大就一定全面更强。

最值得带走的洞见

  1. 分水岭在治理，不在跑分：GPT‑5.4 Thinking 的标志性变化是首次把网络安全"High"能力的缓解落到通用模型上。
  2. 安全实践在跨产品迁移：其方法承袭自 GPT‑5.3 Codex 在 ChatGPT 与 API 中的做法，专用模型的经验正被复用到主力产品。
  3. 能力与责任被绑定：当模型在某一危险领域达到"高"能力，对应的缓解措施随之触发——这是前沿模型安全分级机制的实际运行。
  4. 读懂版本号：没有 5.3 Thinking，基线是 5.2 Thinking；版本号反映发布节奏而非能力高低。
  5. 带着问题读原文：引言确立了"做了什么缓解"，但"缓解效果如何"需要原文中的评测数据来回答。
