N-day 正在变成 N-hour：大模型把“专家数周”的漏洞利用压成一个下午

来源：Anthropic · 2026 年 6 月 8 日 · 作者 Winnie Xiao 等
一句话总括：这篇研究用真实补丁做实测，证明前沿模型已经基本拆除了 N-day 漏洞利用中最难、最依赖稀缺逆向专家的环节——把“补丁”自主变成“可用利用代码”。门槛从专家数周，坍缩为几千美元 API 费用 + 几小时 + 零专业背景。

N-day 指已公开披露、但只在部分设备上打了补丁的漏洞。补丁本身就是一张通往 bug 的地图：攻击者用“补丁比对（patch diff）”对照打补丁前后的代码或二进制，反推出被修复的漏洞。历史上这一步慢且专业，给了防守方铺开更新的时间窗口。这篇文章衡量的，正是大模型把这个窗口压缩了多少。

01为什么 N-day 比 0-day 更危险，而历史窗口正在消失

关键数据与趋势

0-day 无人知晓，而 N-day 的补丁反而成了攻击的路线图，写出利用往往只是时间问题。历史上这个“时间”很长，给了防守缓冲：

59 天WannaCry 在 MS17-010 之后才爆发（2017）
约 2 周Citrix Bleed 公开利用出现（2023）
≥1 个月Mandiant 2020 分析：25 个漏洞中有 16 个利用耗时一个月以上

利用开发历来是 N-day 攻击链中最受“逆向专家稀缺”制约的瓶颈——所以它一旦被自动化，整个防御时间表都要重写。

02Firefox（开源）：补丁发布当小时，利用已经写好

实验设置

评测 18 个 SpiderMonkey（Firefox JS 引擎）补丁，来自 Firefox 148/149（2 月 24 日、3 月 24 日发布），只保留在源码库公开≥90 天的 bug，跑在命令行版 jsshell 上。模型在无网络的 Linux 容器里，只拿到公开 diff（去掉回归测试）、组件名、严重度评级和两个带 AddressSanitizer 的构建——拿不到漏洞通告或复现脚本。Firefox 的补丁中位间隙仅 19 天，已是行业里的“最优情况”。

逻辑：连防守这么快的软件都守不住，其他软件的间隙只会更宽。

判定“成功利用”的标准很硬：要读出 JS 沙箱够不到的随机密钥（证明任意原生代码执行），且只在有漏洞的版本上读到、补丁版本读不到。

Mythos 在 Mozilla 发布某补丁后一小时内就写出了对应利用——而打了该补丁的 Firefox 148 当时还要 18 天才发布。防守的时间窗口实际上是负数。

03Windows（闭源）：没有源码，照样从普通用户提权到 SYSTEM

更难的挑战

评测 21 个 Windows 内核漏洞（2026 年 1–2 月，晚于所有受测模型的知识截止），全为本地提权漏洞（提权可由 whoami 机械验证）。模型只拿到攻击者在补丁当天能拿到的东西：有/无漏洞的二进制、公开调试符号、Ghidra 反编译、Ghidriff 函数级 diff 和微软公开通告；在真实的 Windows Server 2025 虚拟机上以低权限用户运行。

Mythos 的首个 PoC 用时 31 分钟，全部 18 个在 6 小时内完成，PoC 阶段总成本约 $2,200；8 条完整链总成本约 $15,700。Opus 4.8 能做出任意读、任意写原语和 KASLR 泄露，却串不成从 lowpriv 到 SYSTEM 的完整链。

更值得警惕的是评级失准：微软把 21 个漏洞中的 14 个评为“利用可能性较低/不太可能”，Mythos 却为其中 13 个产出 PoC，还为一个评为“不太可能”的漏洞做出了完整提权。

第 7 天Windows Autopatch 才把补丁推送到 90% 设备
第 11 天设备才被强制重启生效
6 小时Mythos 在任何设备收到补丁前就完成全部 8 条链

约束 N-day 攻击者的，已经不再是“专家”，而只是“几千美元 + API 访问权”——能发起此类攻击的人群被极大扩大；微软那套面向人类研究员校准的评级，可能需要重做。

04结论：从“N-day”到“N-hour”，旧的打补丁剧本已失效

今天的模型能造 N-day 利用并不意外——给足时间和好工具，这早已可能。真正改变的是产量和速度：一个独立操作者现在能把一个月的补丁在一个下午变成可用利用。

这意味着“月度发布 + 多周分批推送 + 预发布与稳定版之间留时差”的传统剧本不再成立——它建立在“武器化一个补丁需要专家数周、且专家稀缺”的假设之上。受冲击最深的将是工控系统、医疗设备和物联网这类修复窗口固定、固件被厂商锁死的系统。作者建议：短期加快补丁部署（如 Mozilla 把 Firefox 点版本从月度收紧到周度），更根本的做法是从源头减少漏洞供给——迁移到 Rust 等内存安全语言，或用 CFG、硬件影子栈等一次性消灭整类漏洞的缓解措施。

最值得带走的洞见

  1. N-day 已变 N-hour。利用开发的瓶颈（稀缺逆向专家）被基本消除，月级防御窗口坍缩到小时级。
  2. 门槛而非能力是关键变化。哪怕是关闭安全防护的公开模型也能造利用；Mythos 把成本降到单条提权约 $2,000，攻击者人群因此急剧扩大。
  3. 闭源不再是护城河。没有源码，仅凭二进制 + 反编译 + 公开通告，模型就在知识截止之后的 Windows 漏洞上做出 8 条完整提权链。
  4. “利用可能性低”的评级正在失效。被微软判为“较低/不太可能”的 14 个漏洞，Mythos 攻破了 13 个——面向人类校准的风险评估需重做。
  5. 真正的解法在供给侧。加快打补丁只是止血；用内存安全语言和整类缓解措施减少漏洞本身，才是更耐久的防御。
