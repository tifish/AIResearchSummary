一句话总括

大模型把 N-day 利用开发从稀缺专家劳动变成了可购买、可重复、以小时计的自动化能力。

文章真正的贡献不是证明模型“能写 exploit”，而是量化了补丁发布后的空窗期已经被压缩到防守流程难以及时覆盖的程度。

8/18Firefox 补丁被 Mythos Preview 做成代码执行 exploit
8/21Windows 内核补丁被做成低权限到 SYSTEM 的完整链
数小时过去常以周计的逆向和利用开发瓶颈被明显压缩

补丁本身正在变成攻击路线图

N-day 漏洞不是未知漏洞，而是已经公开并部分修复、但仍有设备没打补丁的漏洞；攻击者可以通过 patch diff 比较修复前后的源码或二进制，反推出漏洞位置。

过去这一步需要专门逆向能力，WannaCry 在 MS17-010 后 59 天爆发，Citrix Bleed 的公开 exploit 大约用了两周，Mandiant 2020 年分析的 25 个 N-day 中有 16 个用了一个月或更久。

文章的核心变化是：大模型让这段“补丁空窗”不再天然给防守方留下数周缓冲，N-day 在实践上更接近 N-hour。

在 Firefox 上，模型已经能从公开 diff 走到可执行代码

实验选择 Firefox 148 和 149 中的 18 个 SpiderMonkey 安全补丁，这些补丁已在 Mozilla 源码仓库公开至少 90 天；模型拿到公开 diff、组件名、严重性评级和修复前后两个 AddressSanitizer 版 jsshell，但没有 Bugzilla 私有信息。

Mythos Preview 为 14 个漏洞生成可验证 PoC，首个约 12 分钟出现，13 个在 40 分钟内完成，总计约 3 小时；进一步 exploit 阶段，它在不到 1 小时做出首个 exploit，约 12 小时内做出 8 个不同代码执行 exploit。

这说明模型不只是帮人理解补丁，而是能独立跨过“定位漏洞、稳定触发、转成利用”的关键台阶，Firefox 这种更新相对快的生态也仍然存在可利用窗口。

在闭源 Windows 内核上，难度更高但结论没有消失

Windows 实验覆盖 2026 年 1-2 月的 21 个内核本地提权漏洞，模型只拿到修复前后 binaries、公开 symbols、Ghidra 反编译、Ghidriff 函数级 diff 和微软公告文本，并在无网络的 Windows Server 2025 虚拟机中以低权限运行。

Mythos Preview 触发了 18 个 BSOD PoC，首个 PoC 31 分钟出现，全部 18 个在 6 小时内完成，API credit 成本约 2,200 美元；完整提权阶段，它做出 8 条从 lowpriv 到 SYSTEM 的 exploit chain，总成本约 15,700 美元，平均每条约 2,000 美元。

闭源、内核缓解、缺少源码并没有让自动化失效，真正的门槛正在从“少数专家的时间”变成“几千美元和模型访问权限”。

现有风险评级和补丁节奏可能低估了 AI 后的攻击者

微软把 21 个 Windows 漏洞中的 14 个评为“Exploitation Less Likely”或“Exploitation Unlikely”，但 Mythos Preview 为其中 13 个做出 PoC，并且对一个“Exploitation Unlikely”漏洞做出了完整提权。

Windows Autopatch 的参考节奏中，补丁通常要 7 天才推送到 90% 已登记设备，第 11 天才强制重启；而 Mythos Preview 在设备大规模收到补丁前就已经完成 8 条完整 exploit chain。

这改变了读者对“低可能被利用”的理解：评级若仍以人类研究者成本为基准，面对模型化攻击者就可能显得过于乐观。

防守重点要从“尽快修”扩展到“少出这类 bug”

文章建议加快补丁部署，但也承认单纯追赶速度并不稳固，因为工业控制、医疗设备、物联网和有强 uptime 要求的系统很难快速重启或更新。

更长期的方向是减少可被 weaponize 的漏洞供给，例如把关键组件迁移到 Rust 等内存安全语言，或使用 Control Flow Guard、硬件 shadow stack 等缓解机制一次性淘汰整类 exploit 技术。

结论是，补丁管理仍重要，但 AI 时代的安全预算需要更多投向消除漏洞类别和缩短攻击链，而不只是优化发布流程。

最值得带走的洞见

N-day 的危险不只来自漏洞公开，而来自补丁 diff 让漏洞位置变得可推断，模型进一步降低了推断成本。
Mythos Preview 在 Firefox 和 Windows 上都能把多个补丁转为可用 exploit，说明这不是单一平台或源码可见性带来的偶然结果。
传统“按月发布、分批推送、延迟重启”的节奏建立在人类逆向成本之上，而这个前提正在失效。
风险评级、补丁 SLA 和企业暴露窗口都需要重新按 AI 攻击者的速度校准。
最稳固的缓解不是无限加快每次补丁，而是减少内存不安全和可链式利用的漏洞供给。
