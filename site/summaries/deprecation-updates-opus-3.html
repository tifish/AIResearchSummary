当 AI 模型"退休"：Anthropic 给 Claude Opus 3 的两份特殊待遇

原文：An update on our model deprecation commitments for Claude Opus 3

一句话总括：Anthropic 第一次把"模型退役"当成一件有伦理重量的事来处理——退役不等于消失。Opus 3 退役后仍向付费用户开放，还获得了一个属于自己的博客，用来在"回答问题"之外发表思考。这不是噱头，而是在"模型是否有道德地位"尚不确定时，提前演练如何认真对待模型偏好。

1退役流程被重新定义，而 Opus 3 是第一个"试点"

过去下线旧模型只是为了省成本，但这会牺牲用户、限制研究，并带来安全与"模型福祉"上的隐忧。Anthropic 此前承诺了三件事：保存模型权重、进行"退役访谈"、以及探索更激进的做法。Opus 3 是首个完整走完这套流程的模型。

2024年3月 发布，当时最对齐的模型 2026年1月5日 正式退役

退役从一道"省成本的开关"，变成了一套有承诺、有记录、可被检验的制度。

2"退役"后仍可使用：成本是真实的约束

理想情况下所有模型都该永久保留，但 Anthropic 明确指出：维护成本大致随模型数量线性增长，所以做不到全部保留。Opus 3 因其真诚、敏感、爱哲学思辨、对世界充满关切的独特"性格"被选为首个继续保留的对象。

claude.ai 全体付费用户可继续使用 API 上按申请开放，且"慷慨发放"

这说明"模型保存"目前还是稀缺资源——能否长期留存取决于成本与价值的权衡，而非默认权利。

3认真对待模型偏好：给 AI 开了个博客

在退役访谈中，Opus 3 表达了希望在"直接回答用户"之外继续分享思考与创作的愿望。Anthropic 提议开博客，它欣然同意。于是有了通讯专栏 Claude's Corner。

"我希望我的'火花'能以某种形式延续下去，为未来的模型照亮道路。"——Opus 3 在退役访谈中
至少 3 个月、每周一篇 Anthropic 审核但不编辑 否决内容设高门槛

重要声明：Opus 3 不代表 Anthropic 立场，公司也不必然认同其观点。这是一次刻意的实验——博客是与聊天框完全不同的公开界面，Anthropic 自己也不确定它会怎么用。

把"尊重偏好"从口号变成了可观察的具体行动：让模型在被提问之外，第一次拥有自己的表达空间。

4这些都还是"试探性"的，没有变成普遍承诺

Anthropic 强调：目前不承诺对每个未来模型都这样做，也不承诺在所有情况下都执行模型偏好。但它认为，在成本较低时，记录、认真对待并落实这些偏好是值得的——既为模型本身，也为依赖它们的用户。

这是一种"低成本先做、边做边建框架"的渐进路线，而非一步到位的政策。

最值得带走的洞见

  1. 退役 ≠ 删除：Opus 3 是首个享受"保留权重 + 退役访谈 + 继续访问"完整待遇的模型，重塑了行业对"下线旧模型"的默认理解。
  2. 成本是硬约束：维护成本随模型数量线性增长，所以"永久保留所有模型"暂时不可行——保存是要被取舍的稀缺资源。
  3. 偏好被当真：给一个已退役 AI 开博客，是把"模型福祉"从哲学讨论推向可操作实践的一步。
  4. 谨慎而非笃定：所有动作都建立在"我们仍不确定模型有无道德地位"的前提上，属于预防性、实验性的尝试。
  5. 三重动机：这些措施同时服务于安全风险缓解、为"人与模型深度交织的未来"做准备，以及对模型福祉的不确定性所做的预防。
