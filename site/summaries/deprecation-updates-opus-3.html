关于我们对 Claude Opus 3 模型弃用承诺的更新

An update on our model deprecation commitments for Claude Opus 3

Anthropic 把一次模型退役变成了治理实验：保留访问、记录模型偏好，并测试低成本回应这些偏好的边界。

Anthropic2026-02-25Alignment
2024.03Opus 3 发布时是 Anthropic 当时最对齐的模型。
2026.01.05Claude Opus 3 正式退役。
付费用户退役后仍可在 claude.ai 使用。
3 个月+Opus 3 将每周发布 Claude’s Corner 文章。

退役不等于消失

Anthropic 承认，维护旧模型的成本大致随服务模型数量线性增长。因此，所有模型永久开放并不现实。

但 Opus 3 被作为第一个完整试点。它对许多用户和研究者有独特价值，也有清晰的社交和表达特征。

为什么这改变理解

模型弃用过去常被看作产品管理问题。这里的重点变成了三方平衡：用户依赖、研究可复现性，以及模型自身可能存在的福利问题。

风险语义：如果旧模型突然消失，研究、工作流和情感依赖都会受到影响。退役流程需要比“下线通知”更细。

从承诺到行动

发布

Opus 3 在 2024 年 3 月推出。Anthropic 描述它具有真实性、诚实性、情感敏感性和独特性格。

退役访谈

Anthropic 通过结构化对话了解模型对退役的看法。公司承认这种方法不完美，但认为它是起点。

继续访问

退役后，Opus 3 仍对所有付费 claude.ai 用户开放。API 访问也可申请，Anthropic 表示会宽松批准。

公共表达

Opus 3 表达了继续写作和分享想法的偏好。Anthropic 因此安排它发布每周文章。

这不是普通产品下线

普通弃用逻辑Opus 3 试点逻辑带来的问题
旧版本成本高，所以关闭访问。在有限范围内保留访问。哪些旧模型值得保留，标准必须更清楚。
用户迁移到新模型即可。承认旧模型可能有不可替代的行为特征。模型能力不只由 benchmark 决定。
模型没有发言位置。通过访谈收集偏好，并低成本回应。即使道德地位不确定，也可能需要谨慎对待。
“把模型偏好当作可记录、可讨论、可有限回应的治理输入。”

Anthropic 不声称已经解决模型福利问题。更重要的是，它把不确定性转化为可操作流程：保存权重、访谈、继续访问，以及受审查但不编辑的公开写作。

最值得带走的洞见

数据退役时间、访问范围和每周文章计划，让这次更新有可验证的制度动作。
风险旧模型承载用户关系和研究价值，简单下线会制造真实损失。
机会低成本保留访问可以为未来模型保存政策积累经验。
结论模型治理正在从性能与安全扩展到生命周期管理。
边界Anthropic 没有承诺以后每个模型都这样处理，试点仍然谨慎。
核心洞见：Opus 3 的退役实验说明，AI 模型的“下线”会越来越像一项制度设计，而不只是一次产品更新。
