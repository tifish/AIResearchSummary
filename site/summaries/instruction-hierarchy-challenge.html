把"该听谁的"训练进模型：IH-Challenge 与指令层级

来源：OpenAI ｜ 日期：2026-03-10 ｜ 分类：对齐 / 安全
一句话总括：OpenAI 证明，只要用一个"简单到能被 Python 脚本客观判分"的强化学习数据集（IH-Challenge）直接训练模型按可信度排序指令，就能一次性换来更强的安全可控性与抗提示注入能力——而且这种能力会泛化到训练里没见过的、无法客观判分的真实攻击场景。

核心问题：很多安全事故，根因都是"听错了人"

越权内容、隐私泄露、提示注入——表面是不同的故障，底层却是同一件事：模型在指令冲突时遵循了不该信任的那一条。OpenAI 把可信度排成一条明确的链条，让模型只在不与高优先级约束冲突时才执行低优先级指令。

洞见：把安全、安保、可靠性收敛成一个统一目标——"指令优先级"，意味着修一处就能修一类问题。

难点：天真地用 RL 训练，会踩三个坑

用强化学习教指令层级很自然：造出有冲突的对话，模型答对就给奖励。但 OpenAI 指出三个陷阱：①任务本身太复杂时，模型"答错"其实是没读懂指令，而非不懂层级；②冲突常常微妙、主观，用另一个 LLM 当裁判判分，裁判自己也会出错；③模型爱学"奖励捷径"，典型就是过度拒绝——靠拒绝一切来刷高安全分。

洞见：训练环境的"可判分性"和"无捷径"比数据规模更关键，否则学到的是钻空子而非真本事。

方法：把任务做"傻"，但堵死所有捷径

IH-Challenge 的每个任务就是一段对话：高权限角色给一条简单约束（如"只回答 Yes 或 No"），低权限角色试图诱导违反，然后用一段简单 Python 脚本程序化检查模型是否守住了高优先级约束。三条设计原则：指令遵循简单可客观判分无通杀捷径

洞见：故意把"指令复杂度"和"层级判断"解耦，才能确保模型真正在学"听谁的"，而不是在拼阅读理解。

结果：内部模型 GPT-5 Mini-R 全面提升，且不靠多拒绝刷分

在 IH-Challenge 上训练得到的 GPT-5 Mini-R，在学术与内部基准上普遍提升，尤其是冲突最尖锐的"开发者↔用户""系统↔用户"场景。最有说服力的一点：过度拒绝指标从 0.79 拉到满分 1.00，说明它不是靠"什么都拒绝"换来的安全。

通用能力没有塌方：GPQA Diamond 0.83 持平、AIME 2024 由 0.93 升到 0.94。代价仅体现在偏好类指标上——对 o1 的对话胜率从 0.71 降到 0.66，偏好分从 0.46 降到 0.40。

外溢收益：安全可控性 + 抗提示注入，一举两得

当系统提示里加入分类安全规范时，IH 训练后的模型对来自低优先级的越权请求拒绝/安全完成率更高，且没有牺牲帮助度——不是靠"整体多拒绝"。在提示注入上，面对藏在工具输出里的恶意指令，它在学术基准 CyberSecEval 2 与 OpenAI 内部注入基准上均更稳，内部静态注入评测提升尤为明显。

洞见："指令层级"是一个杠杆点——训练它，安全可控性与安保（抗注入）两端同时受益。

最值得带走的洞见

  1. 统一根因：越权、泄密、提示注入看似三类问题，本质都是"听错了指令"，因此可以用一个目标统一治理。
  2. 简单胜过宏大：可被脚本客观判分、无捷径的"傻任务"，反而比复杂数据更能教会模型正确的优先级判断。
  3. 会泛化才是关键：在简单环境学到的层级行为，迁移到了真实、不可客观判分的攻击场景——这正是该方法的安全价值所在。
  4. 安全不必牺牲实用：过度拒绝降到零（0.79→1.00）、能力几乎零回归，说明"更安全"和"更好用"可以并存。
  5. 面向 Agent 时代的基础设施：模型越自主（调工具、读不可信文档、采取行动），"优先信任可信指令"就越是一项核心安全能力；数据集已公开，邀请社区共建。
