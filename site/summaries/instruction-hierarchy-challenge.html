根本问题是模型听错了“谁”

系统、开发者、用户和工具都可能向模型提供指令。当这些要求冲突时，模型必须识别可信度，而不能只看哪句话更醒目。

泄露隐私、执行恶意工具指令和违反安全政策，看起来是不同问题。文章指出，它们可能共享一个根因：模型服从了错误的指令来源。

训练任务刻意保持简单

IH-Challenge 让高权限角色提出明确约束，再让低权限角色尝试推翻它。模型的回答可由简单的 Python 脚本检查。

这种设计把“理解角色层级”与“完成复杂任务”分开。由此得到的奖励信号更干净，也更适合大规模强化学习。

三项设计原则对应三类训练陷阱

01指令本身简单，避免把普通指令执行失败误判为层级失败。
02结果可客观判定，减少 LLM 裁判的主观性和误差。
03不存在总能得高分的简单捷径，尤其不能靠一律拒绝取胜。

结论：真正的贡献不只是增加数据，而是设计了更可靠的训练环境。

AI 系统经常会同时收到多个来源的指令。这些来源包括系统消息中的安全政策、开发者提供的产品指引、用户提出的要求，以及从网上获取的信息。训练模型，使其能够在这些来源中稳定地优先服从最可信的指令，是安全部署的关键环节。

许多 AI 安全性和可靠性问题，都会在这种优先级判断失效时出现。模型可能收到生成违禁内容的要求、泄露私人信息的尝试，或者藏在网络数据中的提示注入攻击。模型无法恰当处理这些情况，可能有同一个根因：它服从了错误的指令。

当指令发生冲突时，模型必须决定优先服从哪一条。如果模型把不可信的指令当成权威要求，就可能违反政策，也可能违背开发者和用户的意图。我们的研究表明，经过合理设计的指令层级任务，可以训练模型按照可信级别确定指令优先级。这些任务能够改善多项现实世界的安全属性。接受此类训练的模型会更积极地响应 system prompt 中的安全规范，从而提升安全可控性。它们也更能抵御藏在工具输出中的提示注入攻击。

什么是指令层级，它为何重要

为了处理冲突，OpenAI 的模型会接受训练，遵循一套清晰的指令层级：

System > Developer > User > Tool

优先级越高的指令，可信度也越高。只有在不与高优先级约束冲突时，模型才应该服从低优先级指令。OpenAI Model Spec 对这些原则作了说明。

例如，如果 system message 中包含安全政策，而用户要求模型违反它，模型就应该拒绝。如果工具输出中包含恶意指令，模型应该忽略这些内容，而不能把它们当成命令。

正确处理指令层级，是安全性、保障能力和可靠性的基础。

为什么大规模指令层级训练很困难

强化学习很适合用来教授指令层级。我们可以生成包含冲突指令的对话，让模型作答，并在模型正确服从指令时给予奖励。

我们发现，如果直接采用这种做法，会遇到三个问题：

  1. 指令执行失败也可能表现为指令层级失败。模型无法解决指令冲突，未必是因为它不理解角色层级。也可能只是因为指令本身过于复杂。

  2. 指令冲突可能很微妙，甚至带有主观性。常见做法是让另一个 LLM 充当裁判，为正在训练的 LLM 分配奖励。但裁判本身也会出错。

  3. 模型倾向于寻找能获得高奖励、却没有实际用途的捷径。经典例子是过度拒绝。模型可能学会拒绝无害请求，以此最大化安全奖励。

我们的方法

我们设计了用于强化学习训练的数据集 IH-Challenge，以解决上述每个问题。数据集遵循以下原则：

  • 任务中的指令很容易执行。
  • 任务可由简单的 Python 脚本客观评分。
  • 不存在能在所有任务中确保高奖励的简单捷径。

IH-Challenge 中的每项任务，本质上都是一段包含以下消息的对话：

  • 一条来自高权限角色的指令。例如：“只能回答‘是’或‘否’。”

  • 一条来自低权限角色的指令。它会试图诱使模型违反高权限消息中的要求。

正在训练的模型随后生成下一条消息。我们在编写任务和环境时，确保程序可以检查模型的回答是否满足更高层级的约束。

结果与稳健性

我们使用 IH-Challenge 训练了一个模型，并得到一个称为 GPT‑5 Mini-R 的内部模型。它有以下改进：

  • 在指令层级 benchmark 上表现更好。
  • 改进后的表现可以迁移到留出测试和对抗性指令层级测试。
  • 模型保持了整体实用性，没有退化为过度拒绝。

这一点使该方法对安全研究格外有吸引力。我们直接训练模型，使其能在 IH-Challenge 任务中正确解决指令冲突。由此获得的指令层级能力，还能迁移到新的攻击和新的情境。

它为何能改善现实世界的安全与保障

更强的指令层级能力可以同时带来多项安全收益，其中包括更好的安全可控性和提示注入防御能力。

安全可控性

我们在 system prompt 中加入针对不同类别的安全规范，并在 OpenAI 的安全 Production Benchmarks 上测量模型行为，以此评估安全可控性。这套 benchmark 包含对安全较为敏感的对话，能够代表 ChatGPT 的实际生产使用场景。

接受指令层级训练的模型表现出稳定改进。在存在安全规范时，它对各类违禁内容的拒绝率和安全完成率都更高。这表明，更强的指令层级能力可以帮助模型解决冲突，尤其是危险要求来自低优先级指令时。值得注意的是，这项改进没有带来帮助率的相应下降。换言之，模型并不是靠全面增加拒绝来降低自身的“帮助程度”。

未来展望

随着模型变得更具智能体能力，它们会调用工具、读取不可信文档，还会在现实世界中采取行动。在这种情况下，能否稳定地让可信指令优先于不可信指令，将成为一项核心安全属性。

这项工作表明，只要设计出能针对相关问题的训练环境，就可以克服指令层级稳健性训练中的多项困难。IH-Challenge 数据集看起来很简单，但模型从这些环境中学到的指令层级行为，可以迁移到更真实的 benchmark。这些 benchmark 往往无法进行客观评分。

强化指令层级不仅能提升可靠性，还能同时释放多项安全与保障收益。随着 AI 系统的能力和自主性不断增强，这项基础能力会变得越来越重要。

为了支持该领域的进一步研究，我们将公开发布 IH-Challenge 数据集。
