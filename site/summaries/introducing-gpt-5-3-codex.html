推出 GPT-5.3-Codex

Introducing GPT-5.3-Codex

OpenAI 的核心主张是：Codex 已经不只是更强的代码生成器，而是在向能研究、操作工具、执行任务并接受实时监督的电脑工作代理演进。

25%面向 Codex 用户的运行速度提升。速度变化让长任务更容易持续交互。
77.3%Terminal-Bench 2.0 成绩。它说明终端操作能力成为模型竞争的核心指标。
64.7%OSWorld-Verified 成绩。相比前代 GPT 模型，电脑视觉操作能力大幅提高。
$10MOpenAI 承诺用于网络防御的 API credits。能力提升也带来治理投入。

从写代码到完成工作

文章把 GPT-5.3-Codex 描述为目前最强的 agentic coding model。它延续 GPT-5.2-Codex 的编码能力，也吸收 GPT-5.2 的推理和专业知识能力。

真正的变化不只是分数提高。OpenAI 强调它能处理长时间任务。任务可以包含调研、工具使用、复杂执行和中途转向。

这改变了读者对 AI 编程工具的理解：价值不再只来自一次性生成代码，而来自持续推进一个真实工作流。

网页开发展示的是默认质量

OpenAI 用两个游戏和一个 SaaS 落地页示例说明模型的前端能力。GPT-5.3-Codex 可以在多轮提示和数百万 token 的迭代中改进游戏，也能在欠明确的网页需求中补足合理默认设计。

复杂度

赛车游戏包含不同赛车、8 张地图和道具。示例强调长期构建，而不是单页小样。

交互性

潜水游戏要求探索、收集、氧气、压力和危险管理。这测试的是系统设计能力。

产品感

落地页示例自动处理年付折扣和多条用户证言。结论是模型更懂产品默认值。

网络安全是最大约束条件

High capability

OpenAI 将 GPT-5.3-Codex 归类为 Preparedness Framework 下网络安全相关任务的 High capability 模型。它也是首个被直接训练来识别软件漏洞的模型。

这说明能力提升已经触及双用途边界。模型能帮助防守者，也可能降低攻击门槛。

  • OpenAI 表示尚无明确证据表明它能端到端自动化网络攻击，但采用预防性部署。
  • 缓解措施包括安全训练、自动监控、可信访问和带威胁情报的执行管线。
  • 部分高风险请求可能会从 GPT-5.3-Codex 自动路由到 GPT-5.2。
  • Trusted Access for Cyber 和 Aardvark 扩展体现了“增强防守、延缓滥用”的策略。
最重要的信号不是某一个 benchmark 领先，而是同一个模型开始覆盖编码、桌面操作、知识工作和安全防御。

交互方式也在变化

文章强调 Codex app 中的实时 steering。用户不必等到最终输出，能在模型工作时提问、讨论方案并调整方向。

这让“监督多个代理并行工作”成为新的产品问题。能力越强，人类越需要更好的中途可见性和控制感。

结论很清楚：AI 影响软件行业的方式正在从“替代某个步骤”转向“重组完整工作节奏”。模型会写代码，也会运行命令、读日志、做分析、生成报告和参与部署。

最值得带走的洞见

Codex 的目标正在从 coding agent 扩展为 general computer collaborator。
25% 的速度提升降低了长任务交互成本，实际体验可能比单项分数更重要。
OSWorld 和 GDPval 显示模型正在进入桌面操作和知识工作场景。
网络安全能力越强，访问控制、监控和可信研究计划越关键。
模型参与自身训练和部署，是 AI 研发流程被 AI 改写的明确信号。
