GPT‑5.3‑Codex：从“会写代码的智能体”到“能操作电脑的通用协作者”

来源：OpenAI ｜ 分类：Product ｜ 所有评测均为 xhigh 推理强度
一句话总括：GPT‑5.3‑Codex 把前沿编程能力与通用推理/专业知识合并进同一个模型、速度还快了 25%，真正的突破不在“代码写得更好”，而在于它跨过了从“辅助编码”到“端到端操作电脑完成知识工作”的临界点，并首次深度参与了自身的训练与部署。

一个模型，两种能力合流

过去 OpenAI 把“最强编码”（5.2‑Codex）和“最强推理与专业知识”（5.2）分在两条线上。5.3‑Codex 第一次把两者装进同一个模型，并且运行速度提升 25%，使它能承担涉及研究、工具调用和复杂执行的长任务——可连续自主迭代数天、跨越数百万 token。

更关键的交互变化：你可以像对同事一样，在它工作过程中实时提问、讨论方案、纠偏，而不丢失上下文。

当模型能力足够强后，瓶颈从“智能体能做什么”转移到“人能多轻松地引导和监督并行工作的多个智能体”——5.3‑Codex 把赌注押在“可操控性”上，而非单纯刷分。

基准成绩：编码小赢，电脑操作大跨步

编码类指标相比上一代是小幅领先，但真正拉开差距的是“电脑使用”和“终端”能力——这正是“通用智能体”的关键证据。

OSWorld‑Verified 中人类得分约 72%，5.3‑Codex 的 64.7% 已逼近人类，而上一代仅 38.2%。同时它用比以往任何模型更少的 token 取得这些成绩。

SWE‑Bench Pro 跨四种语言、抗污染、更贴近工业现实，比只测 Python 的 SWE‑bench Verified 更可信；而 OSWorld 几乎翻倍的跃升说明“看屏幕、用桌面软件干活”的能力出现了质变，而非渐进改良。

超越编码：覆盖整个软件生命周期与办公知识工作

模型被设计来支持调试、部署、监控、写 PRD、改文案、用户研究、测试、指标，乃至做幻灯片、用表格分析数据。在衡量 44 个职业 知识工作任务的 GDPval 上，它与 5.2 持平（70.9%）。在网页开发上，它对“含糊需求”给出更合理的默认实现——例如把年付自动换算成打折后的月价、生成含三条评价的自动轮播，使页面默认更接近可上线状态。

“写代码”正退化为一种底层工具，模型真正交付的是完成的工作产物——这扩大了“谁能用它来构建”以及“能构建什么”的边界。

第一个“参与创造自己”的模型

Codex 团队用 5.3‑Codex 的早期版本来调试自身训练、管理自身部署、诊断评测结果：监控训练运行、追踪训练过程中的模式、分析交互质量、提出修复、构建给研究员用的分析应用；工程团队用它定位上下文渲染 bug、排查低缓存命中率，并在发布期间动态扩缩 GPU 集群以稳住延迟。一位研究员让它写正则分类器统计澄清频率与每轮进展，几分钟内跑遍全部会话日志出报告；一位数据科学家让它在三分钟内总结上千数据点的关键洞察。

许多 OpenAI 研究员形容，自己今天的工作与两个月前已“根本不同”。

模型加速自身研发，是 AI 影响最值得警觉也最具说服力的信号——递归式的能力放大已经从论述变成了发布说明里的具体工作流。

网络安全：首个“高能力”模型，配套最强安全栈

5.3‑Codex 是 OpenAI 在其“准备框架（Preparedness Framework）”下首个被列为网络安全“High capability” 的模型，也是首个被直接训练用于发现软件漏洞 的模型。OpenAI 称尚无确凿证据表明它能端到端自动化网络攻击，但采取预防性措施：部署迄今最全面的安全栈（安全训练、自动监控、高级能力的可信访问、含威胁情报的执行管线），并把检测到高风险的请求自动改路由到 5.2。

配套生态举措：扩大安全研究智能体 Aardvark 私测、为 Next.js 等开源项目提供免费代码扫描（已据此发现并披露漏洞）、启动“Trusted Access for Cyber”试点，并在 2023 年 100 万美元资助计划基础上追加 1000 万美元 API 额度 用于加速网络防御，尤其面向开源与关键基础设施。

网络安全天然双重用途——OpenAI 的应对方式是“加速防御者、拖慢滥用者”的证据驱动迭代，这也是首次有厂商因模型攻防能力而被迫上专门的安全防护框架。

可用性与背后的算力

已在付费 ChatGPT 计划的 App、CLI、IDE 扩展和 Web 全面可用，API 即将开放。该模型在 NVIDIA GB200 NVL72 系统上协同设计、训练并部署，速度提升 25% 来自基础设施与推理栈优化。

最值得带走的洞见

  1. 临界点而非升级：编码分数只是小赢，但 OSWorld 从 38%→65%、Terminal‑Bench 从 64%→77% 的跨步，标志着智能体迈向“能操作整台电脑”的通用化。
  2. 能力合流是核心设计：把“最强编码”与“最强推理”塞进同一个模型，意味着不再需要在“会写代码”和“懂业务”之间二选一。
  3. 递归自我加速已落地：模型用于调试自身训练、管理部署、分析评测——AI 研发被 AI 显著加速不再是预测，而是发布事实。
  4. 安全能力首次触顶警戒线：首个“高能力”网络安全模型 + 首个被训练找漏洞的模型，迫使配套最全安全栈、风险请求降级路由与 1000 万美元防御投入。
  5. 更少 token、更强可操控：用更少 token 取得更高成绩，加上实时可介入引导，真正的产品价值正从“输出质量”转向“人机协作效率”。
