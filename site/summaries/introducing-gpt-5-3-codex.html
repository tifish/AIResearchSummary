推出 GPT-5.3-Codex

Introducing GPT-5.3-Codex

一句话总括：GPT-5.3-Codex 把前沿编程、专业推理和计算机操作能力整合进一个可实时引导的代理，使 Codex 从代码助手迈向端到端的工作协作者。

01 · 能力边界

重点不只是代码写得更好，而是能把复杂工作持续做完

GPT-5.3-Codex 结合了 GPT-5.2-Codex 的编程能力和 GPT-5.2 的推理与专业知识。它能执行涉及研究、工具使用和复杂操作的长期任务。

用户还可以在模型工作期间提问、讨论方案并改变方向。上下文不会因此丢失。人与代理的关系由“提交任务后等待”转向持续协作。

02 · 基准结果

编程领先，计算机操作能力提升更明显

77.3%Terminal-Bench 2.0比 GPT-5.2-Codex 的 64.0% 高 13.3 个百分点。

提升最大的部分不是传统代码基准，而是终端操作和视觉桌面任务。这表明模型正在获得跨工具执行完整流程的能力。

03 · 自我参与开发

GPT-5.3-Codex 也是首个深度参与自身开发的 OpenAI 模型

训练阶段

早期版本监控并调试训练过程，还分析交互质量、追踪行为变化并提出修复方案。

这说明代理已经能参与模型研发中持续时间长、信息量大的分析工作。

部署阶段

工程团队用它定位上下文渲染错误和缓存命中率偏低的根因，并优化运行框架。

它还会动态扩缩 GPU 集群，以应对流量激增并稳定延迟。

评估阶段

它构建分类器和数据管道，分析全部会话日志，并在不到三分钟内概括数千个数据点。

价值不只来自单次生成，而是来自对研发循环的整体加速。

04 · 风险

网络安全能力首次达到 High capability

GPT-5.3-Codex 是 OpenAI 首个在 Preparedness Framework 下被归为网络安全高能力的模型，也是首个接受软件漏洞识别专项训练的模型。

OpenAI 尚无确切证据证明它能端到端自动实施网络攻击。但这一评级意味着能力提升已经触发更高等级的防护要求。

05 · 防御机会

限制滥用，同时扩大防御用途

风险较高的请求可能被自动转交 GPT-5.2。高级能力则通过 Trusted Access for Cyber 提供。

OpenAI 还将提供 1000 万美元 API 额度，重点支持开源软件和关键基础设施防御。能力治理由统一封锁转向分级访问和持续验证。

当代理能看懂界面、调用工具、持续工作并接受实时纠偏时，衡量它的重点就不再只是“能否回答”，而是“能否可靠地完成整段工作流程”。

我们正在推出一款新模型，进一步释放 Codex 的能力：GPT-5.3-Codex。它是迄今能力最强的代理式编程模型。

该模型既推进了 GPT-5.2-Codex 的前沿编程能力，也吸收了 GPT-5.2 的推理能力和专业知识。现在，这些能力被整合在同一个模型中，运行速度还提高了 25%。

这使它能够承担涉及研究、工具使用和复杂操作的长期任务。它就像一位同事。你可以在 GPT-5.3-Codex 工作时与它交流并调整方向，而且不会丢失上下文。

GPT-5.3-Codex 是首个在自身创建过程中发挥重要作用的 OpenAI 模型。Codex 团队使用它的早期版本调试模型自身的训练、管理部署，并诊断测试结果和评估数据。Codex 对自身开发的加速幅度令我们的团队深感惊讶。

有了 GPT-5.3-Codex，Codex 不再只是一个可以编写和审查代码的代理。它几乎可以完成开发者和专业人士能在计算机上完成的任何工作。

前沿代理能力

GPT-5.3-Codex 在 SWE-Bench Pro 和 Terminal-Bench 上创下行业新高，在 OSWorld 和 GDPval 上也表现强劲。我们使用这四项 benchmark 衡量编程能力、代理能力和现实任务能力。

编程

GPT-5.3-Codex 在 SWE-Bench Pro 上取得了当前最佳成绩。这是一项严格评估真实软件工程能力的测试。

SWE-bench Verified 只测试 Python，而 SWE-Bench Pro 涵盖四种语言。它更能抵抗数据污染，也更困难、更多样，并且与产业实践更相关。

它在 Terminal-Bench 2.0 上也大幅超过此前的最佳成绩。这项测试衡量 Codex 这类编程代理所需的终端操作能力。值得注意的是，GPT-5.3-Codex 使用的 token 少于以往任何模型，让用户能完成更多工作。

Web 开发

前沿编程能力、更好的审美表现和上下文压缩能力结合后，模型可以完成令人印象深刻的作品。它能从零开始，经过数天持续工作，构建功能丰富的复杂游戏和应用。

为了测试模型的 Web 开发能力和长期代理能力，我们让 GPT-5.3-Codex 制作两款游戏。一款是 Codex 应用发布时那款赛车游戏的第二版，另一款是潜水游戏。

模型使用“开发 Web 游戏”技能，并接受预先选好的通用后续提示，例如“修复这个问题”或“改进游戏”。GPT-5.3-Codex 在数百万 token 的工作过程中自主迭代了这些游戏。你可以观看预告片并亲自试玩，了解 Codex 能做什么。

与 GPT-5.2-Codex 相比，当你要求 GPT-5.3-Codex 制作日常网站时，它也能更好地理解你的意图。

对于简单或描述不完整的提示，它现在会默认构建功能更丰富、默认设置更合理的网站。这为你提供了更好的起点，便于进一步实现自己的想法。

编程之外

软件工程师、设计师、产品经理和数据科学家的工作远不只是生成代码。

GPT-5.3-Codex 旨在支持软件生命周期中的所有工作，包括调试、部署、监控、撰写产品需求文档（PRD）、编辑文案、用户研究、测试和指标分析等。

它的代理能力还超越了软件领域。无论你要制作幻灯片，还是分析电子表格中的数据，它都能帮助你完成想做的工作。

在采用与我们此前 GDPval 结果相似的自定义技能后，GPT-5.3-Codex 在 GDPval 衡量的专业知识工作中也有强劲表现，成绩与 GPT-5.2 相当。

GDPval 是 OpenAI 于 2025 年发布的一项评估。它衡量模型处理 44 种职业中定义明确的知识工作任务时的表现。这些任务包括制作演示文稿、电子表格和其他工作成果。

GPT-5.3-Codex 输出

OSWorld 是一项代理式计算机操作 benchmark。代理必须在可视化桌面计算机环境中完成生产力任务。GPT-5.3-Codex 展现出的计算机操作能力远强于此前的 GPT 模型。

在 OSWorld-Verified 中，模型需要借助视觉能力完成各种计算机任务。人类得分约为 72%。

综合编程、前端、计算机操作和现实任务的结果可以看出，GPT-5.3-Codex 并非只是在单项任务上表现更好。

它标志着一次明显的跃迁：我们正朝着单一的通用代理迈进。这种代理可以在真实技术工作的完整范围内进行推理、构建成果并执行操作。

可交互的协作者

随着模型能力增强，问题的重心也在改变。关键不再只是代理能做什么，而是人们能否方便地与代理互动、指挥代理，并监督多个并行工作的代理。

Codex 应用让代理的管理和指挥变得更容易。加入 GPT-5.3-Codex 后，这种协作也更具交互性。

使用新模型时，Codex 会频繁提供进度更新，让你了解工作中的重要决定和进展。你不必一直等到最终结果出现。

你可以实时互动，提出问题、讨论方案，并把工作引向需要的解决方式。GPT-5.3-Codex 会说明自己正在做什么，响应反馈，并从开始到结束持续让你掌握情况。

要允许用户在模型工作时调整方向，请在应用中打开“Settings > General > Follow-up behavior”。

我们如何使用 Codex 训练和部署 GPT-5.3-Codex

Codex 最近的快速进步建立在 OpenAI 多项研究成果之上。这些研究项目遍布整个公司，持续了数月乃至数年。

Codex 正在加快这些研究。OpenAI 的许多研究人员和工程师表示，他们今天的工作方式已经与两个月前有了根本区别。

即使是 GPT-5.3-Codex 的早期版本，也已经展现出非凡能力。我们的团队因此能使用早期版本改善训练，并支持后续版本的部署。

Codex 适用于极为广泛的任务，因此很难完整列举它帮助团队的所有方式。

例如，研究团队使用 Codex 监控并调试本次发布所需的训练过程。它带来的加速不只体现在调试基础设施问题上。

它帮助团队追踪整个训练过程中出现的模式，深入分析交互质量，提出修复方案，并为人类研究人员构建功能丰富的应用。这些应用让研究人员可以准确理解模型行为与此前模型有何不同。

工程团队使用 Codex 优化并调整 GPT-5.3-Codex 的运行框架。当我们发现一些影响用户的异常边缘情况时，团队成员使用 Codex 找出了上下文渲染错误，并定位缓存命中率偏低的根本原因。

在发布期间，GPT-5.3-Codex 仍在继续协助团队。它会动态调整 GPU 集群规模，以适应突然增加的流量并保持延迟稳定。

在 Alpha 测试期间，一位研究人员希望了解 GPT-5.3-Codex 每轮对话额外完成了多少工作，以及这对生产力产生了什么影响。

GPT-5.3-Codex 设计了几个简单的正则表达式分类器，用来估算澄清问题出现的频率、用户正面和负面反馈的频率，以及任务取得进展的情况。

随后，它以可扩展的方式对全部会话日志运行这些分类器，并生成了一份包含结论的报告。

使用 Codex 构建产品的人变得更满意。代理能更好地理解他们的意图，每轮对话可以取得更多进展，需要提出的澄清问题也更少。

GPT-5.3-Codex 与前代模型存在很大差异，因此 Alpha 测试数据中出现了许多异常且有违直觉的结果。

团队中的一位数据科学家与 GPT-5.3-Codex 合作，构建了新的数据管道，并以比标准仪表板工具丰富得多的方式呈现结果。

研究人员还与 Codex 一同分析结果。Codex 在不到 三分钟的时间里，简洁概括了数千个数据点中的关键洞见。

单独来看，这些任务都是 Codex 帮助研究人员和产品开发者的有趣示例。综合来看，我们发现这些新能力显著加快了研究、工程和产品团队的工作。

守护网络安全前沿

过去几个月里，我们看到模型在网络安全任务上的表现有了显著提升。这些进步既能帮助开发者，也能帮助安全专业人员。

与此同时，我们一直在准备更强的网络安全防护措施，以支持防御性用途，并增强整个生态系统的韧性。

GPT-5.3-Codex 是第一个依据 Preparedness Framework 被我们归类为网络安全相关任务 High capability 的模型。它也是第一个由我们直接训练、用于识别软件漏洞的模型。

我们尚无确切证据表明它能够端到端自动实施网络攻击。不过，我们采取了预防性策略，并为其部署了迄今最全面的网络安全防护体系。

这些缓解措施包括安全训练、自动监控、面向高级能力的受信访问机制，以及包含威胁情报的执行管线。

网络安全具有天然的双重用途。因此，我们采取基于证据并持续迭代的方法。目标是让防御者更快发现并修复漏洞，同时减缓滥用。

作为这项工作的一部分，我们正在推出 Trusted Access for Cyber。这是一项旨在加速网络防御研究的试点计划。

为了防止滥用，如果系统检测到某些请求具有较高网络安全风险，这些请求可能会从 GPT-5.3-Codex 自动转交给 GPT-5.2 处理。我们会继续完善这些防护措施。

从事安全研究的开发者，如果认为自己的请求被错误分类，可以通过 Trusted Access for Cyber 计划申请完整访问权限，也可以使用 /feedback 命令报告问题。

我们还在投资生态系统级防护措施。其中包括扩大安全研究代理 Aardvark 的私有 Beta 测试。它是 Codex Security 产品和工具套件推出的首项服务。

我们也在与开源维护者合作，为广泛使用的项目提供免费代码库扫描。例如 Next.js。一位安全研究人员曾使用 Codex 找到相关漏洞，并在上周进行了披露。

在 2023 年启动的 100 万美元网络安全资助计划基础上，我们还将投入 1000 万美元 API 额度，利用能力最强的模型加速网络防御。这些额度尤其面向开源软件和关键基础设施系统。

从事善意安全研究的组织可以通过 Cybersecurity Grant Program 申请 API 额度和支持。

可用范围与详细信息

GPT-5.3-Codex 已面向 ChatGPT 付费方案开放。凡是可以使用 Codex 的地方，都可以使用该模型，包括应用、CLI、IDE 扩展和 Web。

我们正在努力尽快以安全的方式开放 API 访问。

随着本次更新，我们还通过基础设施和推理技术栈的改进，让 GPT-5.3-Codex 面向 Codex 用户的运行速度提高了 25%。这会带来更快的交互和结果。

GPT-5.3-Codex 围绕 NVIDIA GB200 NVL72 系统共同设计，使用这些系统训练，并由这些系统提供服务。我们感谢 NVIDIA 的合作。

下一步

有了 GPT-5.3-Codex，Codex 正从编写代码迈向把代码当作工具。它可以借此操作计算机，并端到端完成工作。

在推进编程代理能力前沿的同时，我们也在开启更广泛的知识工作。从构建和部署软件，到研究、分析和执行复杂任务，都包括在内。

Codex 最初专注于成为最好的编程代理。如今，这一方向已经发展为通用计算机协作者的基础。它既扩大了能够进行创造的人群，也拓展了使用 Codex 可以实现的事情。
