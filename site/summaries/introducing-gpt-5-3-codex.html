GPT-5.3-Codex：从"会写代码"到"会操作整台电脑"

来源 OpenAI ｜ 2026-02-05 ｜ 分类 Product

一句话总括：GPT‑5.3‑Codex 真正的贡献不是"编码分数又涨一档"，而是把前沿编码能力与 GPT‑5.2 的推理／专业知识合进同一个模型，让智能体跨过"读写代码"的边界、走向"在电脑上端到端完成工作"——而最具信号意义的两件事是：它是 OpenAI 首个被用来帮助训练和部署自己的模型，也是首个被判定为网络安全"高能力"等级、需要带着专门护栏发布的模型。

能力地形跃升极不均匀：计算机操作几乎翻倍，纯编码几乎原地

把附录基准摊开看，"编码模型升级"这一叙事其实被数据反驳了一半：纯软件工程的 SWE‑Bench Pro 只从 56.4% 微涨到 56.8%，几乎持平；真正翻天覆地的是智能体操作能力——终端任务 Terminal‑Bench 2.0 从 64.0% 跳到 77.3%，桌面计算机操作 OSWorld‑Verified 从 38.2% 几乎翻倍到 64.7%（人类约 72%，已逼近）。换言之，这一代的增量主要花在"会用工具、会用电脑"，而非"更会写函数"。

结论：判断这次升级，别只看 SWE 分数——它在编码上已接近天花板，真正的故事是智能体从"代码区"扩张到"整台电脑"。

自我创造首个被用来训练和部署自己的模型

OpenAI 称 GPT‑5.3‑Codex 是首个"参与创造自己"的模型：研究团队用它的早期版本去监控和调试本次发布的训练运行、追踪训练过程中的行为模式、对交互质量做深度分析并提出修复；工程团队用它定位上下文渲染 bug、排查低缓存命中率，并在发布期间动态扩缩 GPU 集群以稳住延迟。一个被反复引用的细节是：一位研究员让它用几个正则分类器去批量扫描全部会话日志、统计澄清问题频率与每轮进度，几分钟内产出结论——许多 OpenAI 工程师形容自己今天的工作"和两个月前根本不同"。

结论：当模型开始进入自己的研发回路、把研究迭代本身加速时，能力增长的复利效应（而非单点分数）才是这篇博客真正想传达的信号。

超越编码从单题任务，走向"用电脑端到端干活"

为验证长程智能体能力，团队让它用"开发网页游戏"技能、配合"修 bug""把游戏做得更好"这类泛化跟进指令，在数百万 token 跨度上自主迭代出两款可玩游戏（赛车与潜水）。在日常网站上，它对模糊需求给出更合理的默认实现（如把年付折算成打折后的月价、自动生成多条评价的轮播）。能力还外溢到软件之外：写 PRD、做幻灯片、用表格分析数据——在衡量 44 个职业真实知识工作的 GDPval 上达到 70.9% 的胜平率，与 GPT‑5.2 持平（即专业知识没退步，但也未在此项再进一步）。同时它做这些用的 token 比以往任何模型都更少。

结论：模型把"会写代码"当成操作电脑的一种工具，而非终点；门槛下降的不只是"谁能编程"，而是"用一个智能体能完成多少类工作"。

交互方式瓶颈从"能力"转向"人怎么指挥一群智能体"

OpenAI 明确点出：当模型够强后，差距从"智能体能做什么"转向"人类能多容易地同时引导和监督多个并行工作的智能体"。GPT‑5.3‑Codex 在 Codex app 里更"可介入"——工作中频繁汇报关键决策与进度，用户可实时提问、讨论方案、中途纠偏而不丢上下文（在设置的 Follow‑up behavior 中开启）。叠加整体快 25%（受益于推理栈优化，并在 NVIDIA GB200 NVL72 上协同设计与服务），交互闭环明显收紧。

结论：下一个竞争维度可能不是基准分，而是"监督带宽"——让一个人有效驾驭多个智能体的产品形态，正在成为护城河。

网络安全首个"高能力"等级模型，带着自动降级护栏发布

这是最值得警惕的一节：GPT‑5.3‑Codex 是 OpenAI 在 Preparedness Framework 下首个被分类为网络安全"高能力"（High）、也是首个被直接训练去识别软件漏洞的模型（CTF 攻防 77.6%）。官方称尚无确凿证据表明它能端到端自动化网络攻击，但采取预防性措施，部署了迄今最完整的安全栈——其中最具体的机制是：被系统判定为高网络风险的请求会自动从 GPT‑5.3‑Codex 路由到能力较弱的 GPT‑5.2。配套举措包括 Trusted Access for Cyber 试点、安全研究智能体 Aardvark 扩大私测、为 Next.js 等开源项目提供免费扫描，以及在 2023 年 100 万美元基金之上追加 1000 万美元 API 额度支持网络防御。目前在付费 ChatGPT 计划的 app／CLI／IDE／web 可用，API 即将开放。

结论："按风险把请求降级到弱模型"是一种新的发布姿态——能力越强，护栏就越要写进推理路径本身，而不只是事后审查。

最值得带走的洞见

  1. 增量花在"用电脑"而非"写代码"。OSWorld 38.2%→64.7%、Terminal-Bench 64.0%→77.3%，而 SWE-Bench Pro 几乎持平——别用纯编码分数衡量这次升级。
  2. 模型进入了自己的研发回路。首个被用来调试自身训练、管理部署、分析评测的模型，预示能力增长开始有复利。
  3. "更快 25% + 更少 token"和能力同等重要。长程智能体（数百万 token 自主迭代）落地，效率本身已是能力的一部分。
  4. 监督带宽成为新战场。官方把"人如何指挥多个并行智能体"摆到与模型能力同等位置，可介入、实时汇报成为产品重点。
  5. "高网络能力"首次触发推理级护栏。首个被判 High cyber 的模型，用"高风险请求自动降级到 GPT-5.2"把安全写进路由逻辑——这是能力前沿与安全约束直接耦合的范例。
