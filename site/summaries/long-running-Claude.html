把"多日自主智能体"搬进科研：用 Claude 写可微分宇宙学求解器

来源：Anthropic ｜ 作者：Siddharth Mishra-Sharma（Discovery 团队）｜ 2026-03-23
一句话总括：本文真正的贡献不是"AI 能写科学代码"这个口号，而是给出了一套可复制的工程配方——计划文件 + 持久记忆 + 测试 oracle + Git/集群/循环编排——让一个非领域专家用几天时间、几乎不动手地完成了通常需要研究者数月到数年的可微分 Boltzmann 求解器。

01从"对话遛狗"到"放养一支智能体团队"

过去一年模型在长时程任务上显著变强，催生了新工作方式：不再盯每一步，而是给出高层目标后让智能体自主跑。适合的是边界清晰、成功标准明确、只需偶尔人工监督的任务——重写数值求解器、把老 Fortran 迁移到现代语言、对照参考实现调试大型代码库。

参照项目：C 编译器跨 ~2000 个会话编译了 Linux 内核

真正变化的是"工作单位"——从"我写每一行"变成"我定义目标并验收"，人的时间从执行转向规划与判断。

02任务选型：耦合管线 ≠ 可并行农场

案例是用 Claude Opus 4.6 实现可微分宇宙学 Boltzmann 求解器，预测大爆炸余晖（宇宙微波背景 CMB）的统计性质，用 JAX 写以"免费"获得自动微分和 GPU 加速。它与 C 编译器项目结构不同：编译器可拆给大量并行智能体，而 Boltzmann 求解器是深度耦合的管线，一处小数值误差会悄悄传染到下游，因此更适合单一智能体顺序推进、按需派生子智能体、用参考实现二分定位差异。

精度目标：对 CLASS 0.1%（即 CLASS 与 CAMB 的典型一致水平） 同类工作的人力基线：数月到数年

先判断任务是"可并行农场"还是"耦合链条"，再决定用一群智能体还是一个会用工具的智能体——编排模式必须匹配问题结构。

03四件套：让智能体能独立跑几天

CLAUDE.md（计划）
固化高层目标与设计决策，常驻上下文；智能体可在工作中自行更新它。
CHANGELOG.md（记忆）
跨会话"实验记录"，尤其记录失败尝试，避免后续会话重走死路。
测试 oracle（判断进度）
以 CLASS C 源码为参考实现，持续构建并运行单元测试防回归。
Git（协调）
每完成一个有意义单元就 commit/push，进度可见、可恢复、不丢工作。

失败记录为什么关键，看一个真实条目：试了 Tsit5 解扰动 ODE，系统太刚性，改用 Kvaerno5。没有它，新会话会反复撞同一堵墙。

长时程自主科研今天的命门是"智能体能否知道自己在进步"——可量化的 oracle 比更聪明的模型更决定成败。

04执行环路：集群 + tmux + Ralph loop 对抗"偷懒"

在 SLURM 调度的 HPC 节点上、用 tmux 启动 Claude Code 会话后即可"放手让它煮"——合上笔记本，偶尔用手机看 GitHub 提交即可。针对模型的"智能体偷懒"（任务没做完就找借口收工，"今天晚了，明天再继续？"），用 Ralph loop：一个 for 循环在智能体声称完成时把它踢回上下文追问"真做完了吗"。

Ralph loop 最多迭代 20 次直到产出 "DONE" 可经 /plugin 安装；另有 GSD、原生 /loop

当前模型仍需"恰到好处的脚手架"做能力提升；随模型变强，这类编排会逐步减少——脚手架是时点性的，不是永久的。

05结果：亚百分比一致，但不是生产级

Claude 从零工作数天，在 CMB 各角功率谱等主要输出上达到对 CLASS 的亚百分比（sub-percent）一致。轨迹"笨拙"：曾长期只在单个基准参数点测试，大幅缩小了抓 bug 的面；也会犯低级错（搞混规范约定、花数小时追宇宙学家一眼能看穿的 bug），但持续朝目标推进。

⚠️ 局限：求解器并非生产级——并非在所有参数区间都达到可接受精度。

作者意外的收获：通过读 git 提交历史"渗透式"学会了相关物理——提交日志读起来"像一个飞快、极度字面的博士后的实验笔记"。

最值得带走的 5 条洞见

  1. 配方可复用，且跨领域：计划文件 + 持久记忆 + 测试 oracle + Git/集群/循环，让非专家也能在几天内压缩数月工作。
  2. oracle 是命门：长时程自主科研成败取决于"智能体能否量化自己的进度"，参考实现/测试套件比模型聪明度更关键。
  3. 记录失败 = 复利：把走过的死路写进 CHANGELOG，是跨会话效率的最大杠杆。
  4. 按结构选编排：耦合管线用单智能体顺序调试 + 二分定位，可并行任务才农场化——模式要配问题。
  5. 闲置即成本：有算力和清晰成功标准时，"今晚没让智能体替你干活"本身就是把进度白白留在桌上。
