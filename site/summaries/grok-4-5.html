一句话总括

介绍 Grok 4.5

Introducing Grok 4.5

Cursor 把新模型的核心能力放在真实工具环境里的长时任务上，而不是只展示代码生成能力。

数万亿 Cursor 数据 token 被用于训练，覆盖代码库、工具交互和开发者代理行为。
首周 2 倍 个人和团队订阅可获得 Grok 4.5 用量翻倍。
$2 / $6 基础版每百万输入 token 2 美元，每百万输出 token 6 美元。
$4 / $18 快速版每百万输入 token 4 美元，每百万输出 token 18 美元。
模型基础

它不是只为写代码而训练

Grok 4.5 是 Cursor 与 SpaceXAI 联合训练的 MoE 模型。训练数据仍深度包含 Cursor 的代码和工具使用轨迹。

变化在于数据配比更宽。文章提到 STEM 任务、研究论文和其他知识工作被纳入训练。读者应把它理解为从编程助手向通用电脑任务代理的扩展。

能力边界

重点是持续调查、使用工具和验证

文章强调长时任务，而不是一次性回答。模型需要在软件工程、数据科学、金融和法律等任务中调用工具、修正错误并验证结果。

这改变了评估 AI 的角度。真正重要的不是单轮语言能力，而是能否在复杂环境中把工作推进到可检查的结果。

训练流程

强化学习环境本身成为核心资产

1

定义问题

工程师先指定任务和验证方式。这样模型训练不只依赖主观偏好，而是围绕可检验结果展开。

2

代理构建环境

分布式代理系统批量构造、测试和打磨任务环境。文章称部分环境若由人工完成，可能需要数百名工程师数月。

3

保持难度

任务必须足够难，难到前沿模型也会失败。否则模型进步后，旧任务就不再提供有效学习信号。

4

训练下一代

Cursor 用上一代模型加速下一代模型的数据环境建设。这说明模型迭代正在进入自我放大的工程循环。

可用性与成本

发布策略把模型直接放进 Cursor 工作流

项目 文章给出的信息 读者应如何理解
平台 桌面、Web、iOS、CLI 和 SDK 均可使用 Cursor 希望它覆盖从编辑器到自动化接口的完整工作链路。
基础版 $2/M 输入 token，$6/M 输出 token 更适合常规长任务，成本结构偏向输出用量。
快速版 $4/M 输入 token，$18/M 输出 token 速度溢价明显，输出成本是基础版的 3 倍。
Composer 2.5 继续提供，属于不同权重级别 Cursor 没有用新模型替代旧模型，而是保留不同规模的选择。
评估提醒
文章没有提供可核验的具体 benchmark 分数。它还说明 Grok 4.5 在 CursorBench 上存在训练数据污染优势，因此被排除。
这点很关键：如果评测集进入训练数据，分数就不再能可靠代表泛化能力。Cursor 把问题公开写出，反而提示读者要更谨慎看待内部评测。
最值得带走的洞见

这次发布真正透露的是模型产品化方向

模型能力正在从回答问题转向完成工作

Grok 4.5 的叙事核心是长时任务、工具调用、错误恢复和验证。这比单次生成代码更接近真实办公场景。

Cursor 数据成为训练优势

数万亿 token 的真实工具交互让模型学习开发者和代理如何在环境中行动。这类数据比普通文本更贴近任务执行。

知识工作边界被重新定义

文章把数据科学、金融和法律放进同一能力框架。AI 影响不再局限于 IDE，而是扩展到所有可由电脑工具完成的复杂工作。

评测污染仍是硬问题

CursorBench 被排除说明内部基准很容易被训练数据影响。读者需要区分真实泛化和训练记忆。

价格透露使用场景

快速版输出价格明显更高。高价值、时间敏感的任务可能适合它，普通批量任务则需要更谨慎控制成本。
