介绍 Grok 4.5

Introducing Grok 4.5

一句话总括：Cursor 与 SpaceXAI 把编程代理的训练经验扩展到了通用知识工作，并用更难、更真实的任务环境训练模型持续调查、调用工具、纠错和验证。

数万亿来自 Cursor 的训练 token
首周 2 倍个人及团队方案的模型使用额度
$2 / $6基础版每百万输入／输出 token
$4 / $18快速版每百万输入／输出 token

从编程专家走向跨领域代理

Composer 2.5 的定位是编程专家。Grok 4.5 则扩大了数据配比，加入高质量 STEM 任务、研究论文和其他知识工作。

结论：Cursor 想让模型处理的，不只是代码，而是电脑上的长时程复杂工作。

基础仍来自真实的软件工作

训练包含数万亿个 Cursor 数据 token。这些数据覆盖用户与代码库、软件工具的广泛交互。

模型既能学习现有软件，也能观察开发者如何工作，以及代理如何与环境互动。

为什么这会改变理解

通用能力并不是通过放弃编程数据获得的。Cursor 的路线是保留真实工具交互，再扩大知识领域。由此形成的模型更接近跨任务执行代理，而不是单纯的问答系统。

难题强化学习训练的是完整执行闭环

训练环境横跨软件工程和更广泛的知识工作。任务被刻意设计到连前沿模型也可能失败的程度。

调查问题

模型先收集信息并理解真实环境，而不是直接猜答案。

使用工具

任务要求模型通过工具采取行动，并持续推进长时程工作。

从错误恢复

失败是训练过程的一部分。模型需要识别偏差并调整路径。

验证结果

环境定义了验证方式，让训练目标落在可检查的结果上。

部分环境若完全由人工建设，需要数百名工程师耗时数月。Cursor 用分布式代理群构建、测试和改进环境，也就是用上一代模型加速下一代模型。

发布即覆盖多端

Grok 4.5 已进入 Cursor 的桌面端、Web、iOS、CLI 和 SDK。

个人与团队方案首周享有双倍使用额度。Composer 2.5 仍会继续提供。

结论：Cursor 将两种不同参数规模和权重类别的模型并行保留，而不是让新模型直接替代旧模型。

评测披露比排行榜名次更重要

评测口径

SWE-Bench Pro 和 Terminal-Bench 中，第三方模型采用自行报告的成绩。SWE-Bench multilingual 的 GPT 5.5 成绩则来自 Cursor 内部运行。

这些成绩的来源并不完全一致，因此横向比较需要保留谨慎。

数据污染

较早的 Cursor 代码库快照被意外纳入训练，使 Grok 4.5 在 CursorBench 上获得优势。Cursor 表示无法确定具体影响，因此排除了该项结果。

相关数据已从未来模型的训练集中删除，CursorBench 也在进行更大规模的更新。这一披露说明，模型评测必须同时检查数据来源和测试集独立性。

最值得带走的洞见

01 · 能力边界正在外扩Grok 4.5 以软件工程为基础，但目标已扩展到数据科学、金融、法律及其他电脑工作。
02 · 真实交互数据是关键资产数万亿个 Cursor 数据 token 让模型接触到开发者与代理实际使用工具的过程。
03 · 难题环境需要持续升级模型进步后，旧任务会失去训练价值。环境本身也必须不断变难。
04 · 污染会破坏评测解释力CursorBench 的训练数据重叠意味着高分不能直接等同于更强的泛化能力。
05 · 模型组合不会立刻收敛Grok 4.5 与 Composer 2.5 将并行提供，分别服务于不同规模和能力需求。

今天，我们与 SpaceXAI 一同发布 Grok 4.5。这是我们最智能的模型，也是我们打造的第一个不只面向软件工程的模型。

Grok 4.5 能够处理困难且耗时很长的任务。这些任务需要模型创造性地使用工具解决问题。无论是软件工程、数据科学、金融、法律工作，还是你在电脑上进行的其他工作，它都可以应对。

Cursor 的个人和团队订阅方案都包含可观的模型用量。发布首周的使用额度还会翻倍。考虑到该模型具备的网络安全能力，我们也增加了新的安全防护措施。

坚实的基础

Grok 4.5 是一个由我们与 SpaceXAI 联合训练的混合专家模型（mixture-of-experts model）。

训练使用了数万亿个来自 Cursor 数据的 token。这些数据涵盖用户与代码库及软件工具之间的广泛交互。模型通过这套数据，既能学习现有软件，也能学习开发者与代理之间的互动。它由此掌握开发者如何工作，以及代理如何与各自的环境交互。

我们把上一代模型 Composer 2.5 训练成了编程专家。到了 Grok 4.5，我们有意让训练数据的构成更加广泛。训练数据包括高质量的 STEM 任务、研究论文和其他知识工作，使模型能够熟练处理多个领域的任务。

在高难度问题上进行强化学习

我们在贴近现实的环境中，让模型针对高难度问题进行强化学习。这些环境既覆盖软件工程，也涵盖更广泛的知识工作。模型会在其中学习调查问题、使用工具、从错误中恢复，并验证结果。

许多问题必须足够困难，难到连前沿模型也会失败。随着模型不断进步，现有任务将无法再教给它们新的东西。过去需要大量推理的问题，也会逐渐变成常规任务。

我们开发了一套分布式代理系统，用于大规模构建这些环境。工程师会指定问题，并说明如何验证解决方案。随后，大批代理负责构建、测试和改进每一个环境。其中一些环境如果由人工完成，需要数百名工程师组成团队，并耗费数月时间。我们正是通过这种方式，利用上一代模型加速下一代模型的进展。

开始使用 Grok 4.5

Grok 4.5 已于今天在 Cursor 中上线，覆盖桌面端、Web、iOS、CLI 和我们的 SDK。

个人与团队方案都把可观的模型用量纳入了我们的第一方模型池。发布首周的使用额度会翻倍。基础模型的价格为每百万输入 token 2 美元、每百万输出 token 6 美元。快速版的价格为每百万输入 token 4 美元、每百万输出 token 18 美元。

Grok 4.5 和 Composer 2.5 属于两种不同的模型权重类别。我们很高兴能够同时支持这两种规模和权重。Composer 2.5 将继续提供。未来，我们还会继续发布同等规模的新模型。

SWE-Bench Pro 和 Terminal-Bench 展示的是第三方模型自行报告的成绩。在 SWE-Bench multilingual 中，GPT 5.5 的成绩来自我们的内部运行结果。

Grok 4.5 在 CursorBench 上具有优势，因为 Cursor 代码库的一个早期快照被意外纳入了训练。具体影响目前尚不明确。我们已经从未来模型的训练数据中删除了这些内容。同时，我们正在对 CursorBench 进行一次更大规模的更新。因此，这里没有列出该项成绩。
