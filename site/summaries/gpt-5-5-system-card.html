GPT-5.5 System Card

一句话总括：这份系统卡的核心贡献，是把 GPT-5.5 的真实工作能力、安全评估、红队测试和 API 部署防护放在同一张风险地图上。

能力提升不只是回答更好，而是更会完成工作

OpenAI 将 GPT-5.5 描述为面向复杂真实工作的模型。它覆盖写代码、在线研究、信息分析、创建文档和电子表格，以及跨工具完成任务。

文中强调的变化不是单点 benchmark。重点是模型更早理解任务，更少需要用户引导，更有效地使用工具，并会检查自己的工作。

这改变了读者对 AI 影响的理解：风险不只来自输出内容，也来自模型能否持续行动并完成多步骤任务。

安全评估被放在发布前流程中

GPT-5.5 经过完整的发布前安全评估和 Preparedness Framework。文章还提到针对高级网络安全和生物能力做了定向红队测试。

这说明模型能力越接近真实工作流，安全评估越需要覆盖工具使用、任务持续性和潜在误用场景。

从模型能力到部署防护的流程

1. 复杂任务

模型被设计用于代码、研究、分析、文档和表格等真实工作。

2. 发布前评估

OpenAI 使用完整安全评估和 Preparedness Framework 检查风险。

3. 定向红队

高级网络安全和生物能力被列为重点测试方向。

4. API 防护

4 月 24 日的更新补充了 GPT-5.5 与 GPT-5.5 Pro 在 API 中的部署防护。

GPT-5.5 与 GPT-5.5 Pro 的关系

主题 文中说法 读者应带走的理解
底层模型 GPT-5.5 Pro 使用相同的底层模型。 普通版本的安全结果通常可作为 Pro 版本的强代理。
计算设置 Pro 版本使用并行 test-time compute。 更多测试时计算可能改变能力边界，因此也可能改变风险姿态。
单独评估 在相关风险或防护策略可能受影响时，OpenAI 会单独评估 Pro。 同一模型在不同运行设置下，不能总被视为同一安全对象。
评估环境 除特别说明外，系统卡结果来自离线评估。 离线结果有参考价值，但部署后的真实使用仍需要防护和反馈。
关键转折

当模型更会使用工具并能持续完成任务时，安全问题从“它会说什么”扩展为“它会怎样行动”。

最值得带走的洞见

能力边界正在变成工作流边界

GPT-5.5 的重点是复杂真实工作，而不是孤立问答。这会放大工具使用和任务延续带来的影响。

安全评估必须跟着能力形态变化

高级网络安全和生物能力被单独红队测试，说明高能力模型需要更有针对性的风险探测。

早期真实用例反馈很重要

近 200 个早期访问合作伙伴提供反馈。这让评估不只停留在实验室场景。

同一底层模型也可能需要不同防护

GPT-5.5 Pro 的并行 test-time compute 可能影响风险。部署设置本身也应进入安全判断。
