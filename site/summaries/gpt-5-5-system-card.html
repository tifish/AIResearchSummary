OpenAI · System Card

GPT‑5.5 系统卡

来源 OpenAI ｜ 分类 Safety ｜ 2026-04-23

一句话总括：GPT‑5.5 真正的看点不是又一次能力跑分，而是 OpenAI 把一次面向"真实工作"的 agentic 能力跃迁，与它"迄今最强的一套安全保障"捆绑发布——能力越往"自己跑完整个任务"走，防滥用就被同步加码到与之匹配的高度。
说明：本页所依据的抓取正文目前只包含系统卡的引言，尚未包含具体的安全评测数值、红队结果或 Preparedness 各能力分项评分。下文仅就引言中可核实的信息展开，不对未披露的指标做推断。

它是什么一款被设计来"自己把活干完"的工作模型

GPT‑5.5 面向的是复杂的真实工作：写代码、联网做研究、分析信息、生成文档与表格，并在多个工具之间穿梭把事情办成。引言对它的描述几乎全是行为层面的变化——比早期模型更早理解任务、更少需要人工引导、更高效地使用工具、会自检，并且不做完不停。这是一张典型的 agentic（自主执行）能力画像，而不是单点问答能力的提升。

结论：把它读作"能独立跑完一条任务链的助手"，能力的重心从"答得准"转向"做得完"。

真正的新意随模型一起发布的是"迄今最强的一套安全保障"

引言里分量最重的一句是：GPT‑5.5 是带着 OpenAI 到目前为止最强的一套安全保障（strongest set of safeguards to date）发布的，目标是在压制滥用的同时，尽量不伤及正当、有益的高级能力使用。它通过了全套预部署安全评测与 Preparedness 框架，其中包含针对两类高危领域——先进网络安全与生物能力——的定向红队。换句话说，能力越强、越能自主行动，配套的"刹车"就被显式地往上加。

结论：节点性信号不在某个分数，而在"能力与防滥用被绑成一对"同步推出。

发布前做了什么近 200 家伙伴先把它放进真实场景跑过

在公开发布前，OpenAI 从近 200 家早期访问合作伙伴处收集了真实用例反馈。这意味着安全姿态不只来自实验室里的离线评测，也吸收了模型在真实工作流中的表现——对一款主打"替你把活干完"的模型，真实场景的暴露面恰恰是风险最先显形的地方。

读卡须知Pro 不是另一个模型，而是同一底座的"加算力"设置

引言交代了一个关键区分：GPT‑5.5 Pro 与 GPT‑5.5 是同一个底座模型，区别只在 Pro 启用了并行测试时计算（parallel test time compute）这一设置。因此 OpenAI 通常把 5.5 的安全结果直接当作 Pro 的强代理；只有当判断该设置可能实质性改变相关风险或所需的保障姿态时，才会单独评估 Pro。此外，系统卡于 2026-04-24 更新，补充了 5.5 与 5.5 Pro 在 API 中部署的保障信息。

GPT‑5.5（同一底座模型）+ 并行测试时计算设置= GPT‑5.5 Pro

结论：读 Pro 的安全结论时，默认它"约等于" 5.5，但凡涉及算力放大的风险要看单独评估。

最值得带走的洞见

  1. 头条是"绑定发布"。GPT‑5.5 把一次 agentic 能力跃迁与"迄今最强安全保障"打包推出——能力越往"自主跑完任务"走，防滥用就被显式加码到匹配高度。
  2. 能力画像全是行为词。更早理解任务、更少需引导、更会用工具、会自检、不做完不停——重心从"答得准"转向"做得完"，这正是 agentic 模型的风险面所在。
  3. 红队收紧到两类高危能力。定向红队聚焦网络安全与生物，说明 OpenAI 把"可能实质辅助攻击/造害"的能力当作必须正面处置的现实风险。
  4. 近 200 家伙伴的真实反馈被前置。安全姿态不只靠离线评测，也吸收了真实工作流中的暴露面，把上线后才显形的风险尽量提前。
  5. Pro 默认"约等于" 5.5，但留了单独评估的口子。Pro 只是同一底座叠加并行测试时计算；凡该设置可能实质改变风险时，结论要以单独评估为准——这只是引言，分项数值仍待完整系统卡。
