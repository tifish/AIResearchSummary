Cursor 如何评估模型质量：当公共榜单失效，就自己造一把尺

来源 Cursor ｜ 作者 Naman Jain ｜ 2026 年 3 月 11 日 ｜ 评测方法论编码 Agent
一句话总括：当前沿模型已把公共编码 benchmark 刷到饱和、无法再区分优劣时，Cursor 用真实开发会话构建了内部评测套件 CursorBench，并用线上实验交叉验证——核心贡献不是"又一个榜单"，而是一套让评测重新对齐真实开发体验的线上-线下闭环方法。

公共 benchmark 在前沿模型上已经"测不出区别"

问题不在分数高低，而在它衡量的东西错了

Cursor 指出公共离线评测有三个根本缺陷：

① 对齐度差——多数 SWE benchmark 仍只考"修 bug"，Terminal-Bench 偏向"从棋盘找最佳一步"这类谜题，与开发者真正交给 agent 的工作不符。

② 评分僵化——任务默认答案唯一，但开发者的请求大多欠定义、允许多种正确解；benchmark 要么惩罚其他正确写法，要么硬加合成约束，两者都失真。

③ 数据污染——SWE-bench Verified / Pro / Multilingual 的题目都来自公共仓库，已进入训练数据，分数被虚高。

为什么重要：这意味着在前沿水平上，榜单已无法分辨对开发者效用截然不同的模型，靠它选型会被误导。

≈60%
SWE-bench Verified 中未解问题被发现"测试本身有缺陷"
已停报
OpenAI 因模型能凭记忆复现 gold patch，停止报告该榜单成绩

CursorBench：用"提交代码"反向追溯出真实任务

数据从真实使用来，而不是从公共仓库来

Cursor 用 Cursor Blame 把已提交的代码追溯回"产生它的 agent 请求"，天然得到一对"开发者查询 ↔ 标准答案"。任务多取自内部代码库和受控来源，降低被模型在训练中见过的风险，并每隔几个月刷新一次以跟上使用方式的变化。

它的任务描述故意写得很短，刻意模拟开发者对 agent 那种含糊、欠定义的说话方式，再用 agentic grader（智能评分器）来稳定打分。评测维度也不止"对不对"，还包括代码质量、效率和交互行为。

规模增长背后，是纳入了更接近真实工程的难题：monorepo 多工作区、排查生产日志、跑长时间实验等。结论：行数虽是粗糙的难度指标，但它反映了 CursorBench 在主动加入更硬的任务。

更难的题，才能把模型重新拉开档次

饱和的榜单看不出差距，CursorBench 能

正因为任务更复杂、规格更模糊，CursorBench 在前沿水平上重新制造出模型间的区分度。一个反直觉的例子：在某些场景下，Haiku 可以追平甚至超过 GPT-5——说明"大模型一定更强"在具体开发场景里并不成立。结论：能区分开发者真实感知差异的评测，才有选型价值。

线上实验兜底：离线高分 ≠ 用着好用

用受控实验把"改进"归因到真实效果

Cursor 同时在真实流量上做受控分析，跟踪一组"交互质量 + 输出质量"的高层代理指标，看它们是否一致变好，而不是只优化单一分数。这能抓住一类离线评测漏掉的退化：评分器看着对、开发者用着更差。

例如在迭代语义搜索时，他们做了一次"完全移除语义搜索工具"的消融实验，从而精确定位它最关键的场景——比如在大型代码库上做基于仓库的问答。结论：CursorBench 的排名也更贴合线上指标里开发者的真实体验。

下一步：为"长时运行、自带电脑的 agent"重做评测

现在的题仍局限在一次会话内

CursorBench-3 虽比公共题更长，但仍在单次会话内完成。Cursor 预判未来一年大部分开发将转向在自己机器上长时间自主运行的 agent，并据此改造评测。难点有三：让评分更便宜、解决"会调用外部服务"任务的可复现性、缩小离线评估与真实体验的差距。
注：2026 年 5 月套件已升级到含更难问题的 CursorBench 3.1，因题目分布改变，分数只能在同一版本内横向比较。

最值得带走的 5 条洞见

  1. 公共编码榜单在前沿模型时代已系统性失效——对齐、评分、污染三重缺陷叠加，OpenAI 弃用 SWE-bench Verified（近 60% 未解题测试有缺陷）就是信号。
  2. 好评测的关键是"数据来自真实使用"——Cursor Blame 把提交代码追溯到原始请求，既保证答案真实，又规避训练污染。
  3. 故意欠定义、任务更大，反而更真实——短描述 + 翻倍的代码/文件规模，让评测贴近开发者真正的工作方式，并重新拉开模型差距。
  4. "更大的模型不一定更好"——Haiku 在某些场景追平/超过 GPT-5，提醒选型必须按真实任务而非参数规模。
  5. 离线分数必须用线上实验校验——单靠评分器会奖励"看着对、用着差"的输出；线上-线下闭环才是可靠地基。
