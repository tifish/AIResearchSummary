一句话总括

与 Mozilla 合作提升 Firefox 的安全性

Partnering with Mozilla to improve Firefox’s security

Anthropic 的核心贡献不是证明 Claude 会“谈论安全”，而是展示它能在 Firefox 这样成熟、复杂、真实使用的软件中快速发现高严重性漏洞，并推动修复流程进入产品发布。

真实代码库让结果更有分量

团队先让 Claude 复现历史 Firefox CVE，再转向当前版本中尚未报告的新漏洞。这个切换很重要。

历史漏洞可能出现在训练数据里。当前版本的新漏洞则更能衡量模型是否真的具备发现能力。

Firefox 是高价值目标。它每天处理不可信网页内容，也服务数亿用户。因此，这里的发现比普通开源项目更能说明 AI 对现实安全工作的影响。

第一处信号来得很快

Claude Opus 4.6 在约 20 分钟探索后，报告了 JavaScript 引擎中的 Use After Free。Anthropic 研究员在独立虚拟机中验证后，又交叉验证并提交到 Bugzilla。

这改变了读者对 AI 安全能力的判断：模型不只是辅助阅读代码，它已经能生成需要维护者认真处理的漏洞线索。

合作流程从单点验证转为批量分诊

1

从 JavaScript 引擎开始。这是浏览器中攻击面很大的部分，因为它会处理来自网页的外部代码。

2

扩展到更大范围。团队最终扫描近 6,000 个 C++ 文件，并提交 112 份唯一报告。

3

Mozilla 调整提交方式。在技术沟通后，Mozilla 鼓励 Anthropic 批量提交发现，而不是逐个完全验证安全影响。

4

修复进入发布链路。多数问题已在 Firefox 148.0 修复，其余问题计划在后续版本处理。

发现漏洞和利用漏洞不是同一件事

能力 文章中的表现 为什么重要
发现漏洞 两周发现 22 个漏洞，其中 14 个为高严重性。 防御者可以更快扩大审计覆盖面，尤其是在维护者资源有限时。
生成 exploit 数百次测试中，只有 2 个案例转化为可用的原始 exploit。 当前模型更偏向防御侧能力，但自动攻击能力已经出现早期信号。
真实攻击可用性 这些 exploit 只在移除部分浏览器防护的测试环境中工作。 Firefox 的 sandbox 等纵深防御仍然有效，但未来能力差距可能缩小。

真正的政策含义是时间窗口

当前模型更擅长找漏洞和辅助修复，而不是稳定利用漏洞。这个差距给防御者留下了窗口。 文章的警告是：随着模型进步，这个窗口不一定会持续很久。

报告要可复现

Firefox 团队重视最小测试用例、详细 proof-of-concept 和候选补丁。

这些材料让维护者能验证报告，而不是只相信模型判断。

补丁要能自检

Anthropic 强调 task verifier。它让 agent 能确认漏洞是否被移除，也能检查功能是否回归。

这让 AI 生成补丁更接近可审查的工程产物。

披露要有流程

AI 会提高漏洞报告速度。维护者和研究者都需要协调披露机制来消化这种速度。

流程本身会成为 AI 安全时代的防线。

最值得带走的洞见

AI 安全能力已进入真实维护流程

这不是只在 benchmark 上成立的结果。Firefox 的复杂度、用户规模和安全成熟度，让这次合作更接近未来软件维护的真实样貌。

防御者暂时占优

Claude 明显更擅长发现和修复漏洞，而不是构造稳定 exploit。

批量发现会改变维护压力

AI 能让报告数量快速上升。维护者需要更好的分诊、验证和优先级机制。

证据比模型结论更重要

最小复现、proof-of-concept 和候选补丁，是让 AI 报告可被信任的关键。

安全窗口需要被主动利用

在自动利用能力成熟前，开发者应尽快用 AI 加速漏洞发现、补丁验证和协调披露。
