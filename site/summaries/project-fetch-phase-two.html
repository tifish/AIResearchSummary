Anthropic · Frontier Red Team · 2026-06-18

Project Fetch：第二阶段

一句话总括：这次实验的贡献不是证明大模型已经会机器人，而是证明通用模型正在快速学会接入、编程和使用现成物理工具。

从辅助人类到自己完成，能力跨越发生得很快

第一阶段的 Project Fetch 发生在 2025 年 8 月。Anthropic 让非机器人专家员工操作一只商用四足机器人，并比较有 Claude 与没有 Claude 的团队表现。

当时 Claude Opus 4.1 不能独立完成任务。它甚至会卡在连接机器人这一步。到第二阶段，Claude Opus 4.7 已经可以在很少人类介入下完成多项任务。

这改变了读者对 AI 影响的理解。AI 不只是让非专家更快上手工具，它正在把原本人机结对完成的工作吸收到模型自己的执行流程里。

速度优势很大，但实验边界也很清楚

研究者在 Claude Code 中运行 Opus 4.7。每项任务做 3 次试验，并把 adaptive thinking 的 effort 设为最高。

人类研究者只负责插入运行 Claude Code 的笔记本，输入初始提示，批准命令，并允许模型进入下一项任务。

3 次Opus 4.7 试验次数
10 倍以上对至少一个人类团队完成过的任务
约 20 倍相对最快人类团队的整体速度优势

这些数字说明，模型在“接入工具、读文档、写程序、调传感器”这类任务上已经很强。它把物理世界的一部分问题转化成了软件和系统集成问题。

最强的部分是接口选择和代码生成效率

人类团队曾花时间在多种传感器接口方案之间摇摆。Opus 4.7 能更快判断最佳路径。它写出的许多代码第一次就能工作。

在两个 2025 年人类团队都完成的 4 个任务上，Opus 4.7 平均比无 Claude 团队快 37 倍以上，比有 Claude 团队快 18 倍以上。

37 倍以上相对 Team Claude-less
18 倍以上相对 Team Claude
近 1/10相对 Team Claude 的代码量

这不是单纯多写代码换速度。模型用更少代码完成了同等或更好的系统集成。这意味着代理式 AI 的生产力来自选择、调试和执行路径的压缩。

失败点暴露了现实控制的难度

Claude 仍然没有可靠完成自主取回沙滩球。它能把机器人移动到球后面，也能尝试把球撞回起点。

问题在于精细闭环控制。机器人需要持续感知球的位置，判断上一次动作造成的偏差，再快速修正下一步动作。人类经过练习后能做到这一点，Claude 在实验中没有做到。

这个失败很重要。它说明模型已经能处理许多高层工具使用问题，但还没有稳定掌握低层运动控制、误差反馈和连续物理交互。

意义在于“物理代理式 AI”的早期迹象

Anthropic 把这个变化类比为 AI 编程能力的发展。模型最初只是帮助人类使用软件工具，后来能自己调用编辑器、替换字符串和运行命令。

Project Fetch 的第二阶段显示，类似迁移可能正在发生到硬件世界。模型开始能使用现成机器人和传感器，而不是只在屏幕里操作软件。

这不等于通用机器人已经出现。更合理的结论是：现成物理工具正在变成大模型可调用的外部能力。一旦工具接口足够清晰，能力进步可能会非常快。

最值得带走的洞见

  • Opus 4.7 的核心突破是自主完成系统接入和机器人编程，而不是解决机器人学本身。
  • 任务速度从人类团队级别压缩到至少 10 倍以上，说明通用模型的工具使用能力正在快速迁移到物理设备。
  • 代码量接近 Team Claude 的十分之一，表明效率来自更好的路径选择，而不只是更快打字。
  • 自主取球失败说明闭环物理控制仍是关键瓶颈，尤其是实时感知、误差修正和动作稳定性。
  • 真正值得关注的是轨迹：AI 先帮助人类，随后人类帮助 AI，最后 AI 开始独立使用工具。
