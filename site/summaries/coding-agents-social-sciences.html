编程智能体进入社会科学：谁在用，用来干什么

来源：Anthropic ｜ 2026年5月27日 ｜ 作者：Thomas Lyttelton、Maxim Massenkoff、Nathan Wilmers

一句话总括：这是首份大样本实证报告，揭示能自主写代码、跑分析、解读结果的编程智能体（如 Claude Code）正悄然进入社会科学——但截至2026年3月，只有五分之一研究者真正用上它，且使用极不均衡；它更擅长帮人"开新坑"，而非"把论文送进期刊"。

1会用聊天机器人的很多，用编程智能体的很少

"试过AI"和"把智能体纳入工作流"是两件事

81%
曾用AI辅助研究
20%
每周规律使用编程智能体
1,260
受访定量社会科学家

调查于2026年2-3月进行。绝大多数人试过AI聊天机器人，但即使在这群主动报名、对AI好奇的研究者里，也只有1/5把会自主写代码、跑分析的编程智能体纳入了日常。在智能体用户中，86%用 Claude Code（最常见），31%用 Codex。

结论：舆论的热度远跑在采用率前面——"试过"不等于"用上"，真正的工作流改造才刚开始。

2采用极不均衡：学科、资历、性别、校际都分化

这是比"整体AI使用"更陡峭的鸿沟

经济学39%
政治学25%
公共卫生6%
传播学6%
教育学4%

资历上，博士生与博后约四分之一每周使用，终身教职者的比例不到这个的一半——采用者多是更懂代码、更直接处理数据、也更有出成果压力的青年人。性别与校际差距更刺眼：男性名字研究者的采用率是女性名字的两倍多，高声望及私立大学研究者高出约40%（均在 p<0.05 显著）。这些差距即便在同学科、同资历内比较也依然存在。

结论：编程智能体当前比一般AI的分布更不平等，可能放大本就存在的科研资源差距。

3主要用来写代码，而非写论文

这与"AI制造学术垃圾文章"的舆论焦点恰恰相反

生成分析代码97%77%

最普遍的用途是"给定量数据写分析代码"，其次才是改文字、问方法、查文献背景。只有经济学和管理学者较多用AI起草正文。换言之，AI在社会科学里目前主要是"分析助手"，不是"写手"。

结论：关于AI的公共讨论盯着"写作"，但真实的渗透发生在"分析与代码"这一层——而分析决策恰恰塑造我们如何理解经济与社会。

4产出更多，但卡在"最后一公里"

描述性对比，非因果——早期采用者本就可能更高产

控制资历、学科与答题周后，智能体用户在研究流程前端明显更活跃：立项多约10%、工作论文多约75%（约多出半篇），立项与会议投稿也更多。但在后端，没有证据显示他们向期刊投稿更多或修回更快。

结论：智能体擅长"把项目跑起来"，对"把论文打磨到投稿"帮助有限——这道差距可能只是时间问题，也可能是工具能力的真实边界。

5对"个人产量"乐观，对"学科整体"存疑

两种乐观之间存在系统性落差

88%
给"AI提升发表生产力"打分>5（满分10）
50%
打分达到8分及以上
70%
对个人产量的乐观高于对学科影响

受访者普遍相信AI能提高自己写出可发表论文的能力，但同一批人中，七成对"论文产量"的乐观高于对"社会科学整体"的乐观。他们担心：更多论文意味着拥挤与注意力争夺，也担心AI会放大选择性报告、保守化与增量化等既有顽疾。

结论：研究者把"私人收益"与"公共代价"分得很清——个体增产，未必等于学科进步。

最值得带走的洞见

  1. 渗透已开始，但远未普及：81%试过AI，仅20%每周用编程智能体；热度跑在采用率前面。
  2. 不平等是核心信号：男性名字采用率为女性两倍多、顶尖校高约40%、青年人远多于资深者——这道鸿沟比一般AI更深。
  3. 真正被自动化的是"分析"而非"写作"：97%的智能体用户用它写代码，公共讨论的焦点错位了。
  4. 能开坑，难收尾：工作论文多约75%，但期刊投稿无增长——生产力提升尚未走完全程。
  5. 私人乐观 ≠ 公共乐观：研究者相信AI帮自己多发论文，却担心它让整个学科更拥挤、更失真。所有结论均为描述性，因果证据有待后续随机实验。

注意事项：样本为主动报名、可获 Claude Max 账号的定量社会科学家，并非代表性抽样，受访者很可能比一般人更重度使用、也更乐观；产出差异为描述性比较，不能解读为因果；本报告只统计数量、不评估质量。这是一项更大规模研究（含向研究者随机提供 Claude Code 的实验）的基线波次，后续将发布带有干净对照组的结果。
