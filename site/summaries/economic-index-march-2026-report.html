经济指数报告：学习曲线

Anthropic Economic Index · 2026 年 3 月 24 日 · 样本期 2026 年 2 月 5–12 日
一句话总括

用 AI 是一种"越用越会用"的技能：注册更久的老用户带来更难、更高价值的任务，且对话成功率系统性更高——这条学习曲线可能正在悄悄放大劳动力市场的技能偏向型不平等。

01核心发现：经验=成功率

老用户的优势不是"挑了简单任务"，而是真的更会用

报告最重磅的结论是：注册满 6 个月以上的高资历用户对话成功率高出约 10%。关键在于这种优势经得起检验——在控制了具体 O*NET 任务、模型、语言、国家等全部变量后，仍保留 3–4 个百分点的净效应。

+10%原始成功率差距
+3–4pp加全部控制后仍存在
+7pp更可能用于工作目的

老用户个人闲聊类对话少 10%、输入所反映的教育水平高 6%，更倾向与 Claude 反复迭代而非简单"下指令"。注册一年的用户仅 38% 对话是个人用途，而最新用户高达 44%。

洞见：这推翻了"越老练越偏自动化"的旧假设——最资深的用户反而最爱协作式迭代，证据指向"边干边学"，而非单纯的早期采纳者天赋。

02用户会把难任务交给更强的模型

模型选择与任务价值精准匹配，说明用户在主动权衡成本与性能

付费 Claude.ai 用户中，55% 的计算/数学类任务用 Opus，而教育类只有 45%；软件开发任务用 Opus 的比例（34%）是辅导任务（12%）的近三倍。

+1.5ppClaude.ai：任务时薪每+$10，Opus占比
+2.8ppAPI 用户的斜率约为两倍

洞见：用户并非盲目使用最强模型，而是按任务价值精打细算；写程序、跑工作流的 API 用户对任务复杂度的反应尤其敏锐。

03使用场景在变"日常"

早期采纳者偏好高价值编程，后来者带来更广泛的生活化需求

Claude.ai 使用更分散：前 10 大任务占比从 24% 降至 19%，课业从 19% 降到 12%（部分因冬季学期假期），个人用途从 35% 升到 42%（体育、商品比价、家居维护等查询增多）。

$49.3→$47.9任务平均时薪价值微降
12.2→11.9 年输入所需平均受教育年限
~49%职业已有≥1/4任务被Claude做过

与此同时，编程任务正从 Claude.ai 向 API 迁移（Claude Code 占比大增），商业销售外联、自动化交易等 API 工作流的份额至少翻倍——这些更接近"无人在环"的自动化。

洞见：这是经典的"采纳曲线"——技术从少数高价值专业用途，扩散到大众的日常长尾需求。

04不平等：国家间扩大，美国州内收敛但放缓

全球采纳鸿沟在加深，国内差距在缩小

45%→48%前20国占人均使用量份额（上升）
30%→24%美国前五州份额（下降/收敛）
5–9 年美国州际均等所需时间（原估2–5年）

洞见：低采纳国家略微进一步落后；AI 红利的地理分布短期内更可能集中而非普惠。

最值得带走的洞见

  1. 学习曲线是真实的。老用户更高的成功率无法用任务、语言或国家解释，最可信的解释是"边干边学"——熟练使用 AI 本身是一种可积累的技能。
  2. 这可能放大不平等。报告点名"技能偏向型技术变革"机制：高技能早期采纳者既最受 AI 增益，也最暴露于 AI 颠覆，差距或自我强化。
  3. 用户很"理性"。把最难、最值钱的任务交给 Opus，简单任务留给更快更便宜的模型，说明成本意识已深入使用习惯。
  4. AI 正在"大众化"。编程迁向 API 自动化，Claude.ai 则涌入更多生活化轻量需求，平均任务价值因此微降。
  5. 结论仍有保留。作者坦承可能存在群体效应与幸存者偏差，但受控回归已排除最简单的混淆，随时间推移将能更干净地分离"学习"与"选择"。
