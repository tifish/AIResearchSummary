一句话总括

智能体式编码与专业知识的持续回报

Agentic coding and persistent returns to expertise

Claude Code 的真实使用数据表明，AI 代理正在承担更多实现工作，但真正决定成败的仍是用户是否理解自己要解决的问题。

Anthropic Economic Research 2026 年 6 月 16 日 约 40 万个会话

研究规模足够大，信号来自真实工作

这不是实验室小样本测试。研究分析了 2025 年 10 月至 2026 年 4 月约 23.5 万名用户的约 40 万个 Claude Code 会话。它观察的是人和代理如何在实际任务中分工。

400k约 40 万个交互会话
235k约 23.5 万名用户
7覆盖 7 个月使用变化
9会话被划分为 9 类工作模式

人决定方向，Claude 决定实现

典型会话里，用户负责“做什么”。Claude 负责“怎么做”。这说明代理没有简单替代人的判断，而是把执行层大量外包出去。

70%

规划决策由人做出。包括目标、路线和完成标准。

20%

执行决策由人做出。文件修改、代码写法和命令运行更多交给 Claude。

这个分工改变了 AI 对工作的影响。价值不再只来自亲手写代码，而来自能否清楚定义问题并判断结果是否正确。

使用场景从修错转向端到端工作

七个月内，Claude Code 不再主要像调试助手。它越来越多地参与部署、运行、数据分析和文档生成。

修复代码下降

修复损坏代码的会话占比从 33% 降到 19%。这说明工具成熟后，用户开始把它用于更完整的任务链。

操作软件上升

部署、配置、运行流水线和监控系统从 14% 升至 21%。代理开始进入代码周边的运维环节。

写作和数据分析翻倍

相关会话从约 10% 升至约 20%。这暗示 coding agent 正在成为更广义的知识工作代理。

任务价值提高

按自由职业岗位估算，平均会话价值上升 27%。构建、运营和修复类任务的估值都显著提高。

专业知识比职业标签更能解释成功

软件职业仍有优势，但优势不大。只要用户掌握任务领域，非软件职业也能让 Claude Code 完成产出代码的任务。

比较对象 严格成功 较宽松成功 含义
新手会话 15% verified success 77% 至少部分成功 缺少领域判断时，更难验证结果并修正方向。
中级及以上会话 28%-33% verified success 91%-92% 至少部分成功 从新手到中级的收益最大，深度专家只再增加一部分优势。
产出代码的软件职业 34% verified success 89% 至少部分成功 传统编码背景有帮助，但不是唯一决定因素。
产出代码的其他职业 29% verified success 88% 至少部分成功 领域专家可以用代理完成过去需要程序员承担的实现工作。
AI 编程代理削弱的是“必须亲手实现”的壁垒，没有削弱理解问题本身的回报。

困难出现时，专业知识的价值最清楚

当会话遇到错误、失败测试、重复尝试或用户不满时，专家更容易恢复。新手更容易在没有写出代码时放弃。

4%

遇到困难的新手会话最终达到 verified success。问题不是只会不会输入提示，而是能否识别错误路径。

15%

遇到困难的专家会话最终达到 verified success。专业知识让用户能纠偏、补充约束，并判断 Claude 的输出。

最值得带走的洞见

1. 判断力成为杠杆

代理做更多执行，人类的价值转向目标设定、约束表达和结果验收。

2. 编程正在扩散到普通岗位

非软件职业在产出代码任务上的成功率已经接近软件职业。

3. 中级能力拿到多数收益

从新手到中级的提升最大。掌握领域基本规则，比追求极深专精更关键。

4. 任务价值在上升

平均任务估值七个月上升 27%。用户开始把 Claude Code 用在更复杂、更端到端的工作上。

5. 结论仍需谨慎

研究不能直接观察真实业务结果，也排除了大量非交互式使用。它提供的是早期趋势，不是最终答案。
