AI 编程代理扩大了执行能力，但没有削弱专业判断的价值

一句话总括：这项研究真正的贡献，是用约40万次真实 Claude Code 会话展示了智能体式编程中的新分工：人类定义目标和验收标准，AI 承担大量执行，而成功率主要取决于用户对问题本身的理解。

约40万次会话 约23.5万人 2025年10月-2026年4月 隐私保护聚合分析
70% / 20% 人类平均做出约70%的规划决策，但只做出约20%的执行决策。
27% 会话对应的平均任务估计价值在七个月内上升约27%。
15% vs 28%-33% 新手会话验证成功率约15%，中级及以上为28%-33%。
34% vs 29% 产出代码的会话中，软件相关职业与其他职业的验证成功率差距约5个百分点。

智能体式编程的核心分工，是人类定方向、AI 做执行

研究将会话中的决策分成规划和执行两类：规划包括做什么、采取什么方案、什么算完成；执行包括改哪些文件、写什么代码、跑哪些命令。结果显示，人类仍牢牢控制目标和验收，而 Claude 主要接管实现细节。

典型会话约有4轮交互，每条用户提示平均触发约10个 Claude 动作和2400词输出，少数会话甚至每条提示触发超过100个动作。这改变了对 AI 影响的理解：AI 的作用不是简单替代一次性代码补全，而是在用户设定方向后承担一串连续执行工作。

结论和洞见：未来的生产力差异，很大一部分来自“能否把任务交代清楚并判断完成质量”，而不只是打字写代码的速度。

专业知识让用户用更少指令撬动更多 AI 工作

Anthropic 让模型根据用户如何描述目标、要求验证、纠正 Claude 或被 Claude 纠正，将会话中的任务专业度从新手到专家分级。新手每条提示大约触发5个动作和600词输出，专家则约为12个动作和3200词输出。

这个差距出现在不同工作类型和不同任务价值区间中，说明专业知识不是只让人“问得更高级”，而是让 AI 能更放心地连续执行。读者应改变的理解是：AI 代理越强，越会放大清晰问题定义和领域判断的价值。

结论和洞见：专家不是因为亲自写更多代码而占优，而是因为他们更知道该让代理做什么、怎么验证、哪里可能出错。

编程背景的重要性下降，但问题理解仍然决定成败

在产出代码的会话中，软件相关职业的验证成功率约34%，其他职业约29%；按较宽松的“至少部分成功”口径，两者分别约89%和88%。十个最大职业组的成功率都落在软件工程师附近7个百分点以内，管理职业甚至略高。

这说明 AI 编程代理正在让更多非程序员完成技术工作，但并不意味着任何人都能无差别地产出好结果。真正被削弱的是“必须会手写实现细节”的门槛，真正被保留甚至放大的，是对业务、数据、流程、法律、财务或科研问题的把握。

结论和洞见：编程可能从单一职业的专门产出，变成许多职业日常工作的一部分。

任务正在从修 bug 转向端到端工作，价值也在上升

从2025年10月到2026年4月，修复损坏代码的会话占比从33%降到19%；运行和部署软件从14%升至21%；写作与数据分析大约从10%升至20%。研究还用自由职业市场任务进行粗略校准，估计平均会话价值上涨27%。

其中构建、运行和修复类任务的估计价值分别增长约43%、34%和32%。这说明 Claude Code 的使用正在越过“帮我修一段代码”的阶段，进入部署、数据分析、文档和自动化等更完整的工作链条。

结论和洞见：AI 代理能力提升后，用户交给它的不是更碎的任务，而是更完整、更接近真实工作成果的任务。

成功率随专业度上升，但主要收益来自从新手到合格使用者

按最严格的验证成功标准，新手会话成功率约15%，中级及以上为28%-33%；按至少部分成功口径，新手为77%，中级到专家为91%-92%。也就是说，最大跃迁发生在从新手到中级，而不是从中级到专家。

遇到困难时差距更明显：有失败信号的会话中，新手验证成功率约4%，专家约15%；新手中有19%的困难会话以失败且零代码写入告终，其他人约5%-7%。这说明专业能力的关键价值是纠偏和恢复，而不仅是首次提示写得好。

结论和洞见：掌握领域基本功已经能获得大部分收益，深度专家仍有优势，但优势更多体现在处理困难和模糊问题时。

这项研究有边界：它测的是会话证据，不是真实世界结果

Anthropic 没有观察代码后续是否被采用、是否产生经济价值，也排除了非交互式和第三方 IDE/SDK 中的大量 Claude Code 使用。职业、专业度、成功与失败都依赖模型读取 transcript 后分类，虽然研究用遥测数据和参考模型做了校验，但仍不是人工标注的绝对真值。

这些限制不削弱它作为早期信号的价值，却提醒读者不要把数字当作完整劳动力市场结论。最重要的是观察趋势：如果未来专业度收益下降，可能意味着模型开始提供更多关键判断；如果非软件职业成功率继续上升，则软件生产会更深地嵌入普通知识工作。

最值得带走的洞见

AI 编程代理最明显的分工变化，是让人类更多负责目标、约束和验收，让模型负责长链条执行。
领域专业知识仍然有持续回报，因为它决定用户能否准确表达问题、识别错误并指导修正。
编程职业壁垒在降低，非软件职业已经能在许多代码产出任务中接近软件相关职业的成功率。
工具使用正在从修 bug 转向部署、运行、数据分析和文档等端到端工作，任务价值随之上升。
真正值得跟踪的长期问题，是模型是否会逐步接管现在仍由人类提供的判断力。
