AI 使用已形成生活节律

工作相关请求在周末下降，个人用途则从约 35% 升至接近 50%。新闻请求集中在早晨，食谱请求在下午 6 点达到平均水平的 2.3 倍，睡眠建议则在凌晨 5 点左右达到高峰。

这说明 AI 不只是一个统一的生产力工具。它已经嵌入工作、休息、饮食和健康等不同场景，并随现实生活的时间结构变化。

非工作时段的劳动并未平均减少

夜间和周末仍然出现的工作请求，更偏向高薪职业对应的任务。移除计算机和数学职业后，这种趋势依然存在。

这提示我们，AI 使用记录也可能折射既有的劳动差异。灵活工具未必让所有人以同样方式摆脱工作。

调查不能代表总体人口

计算机与数学职业约占受访者的 30%，但只占美国就业的 4%。管理职业占受访者的 23%，也远高于 7% 的就业占比。女性仅占关联样本的 12%。

因此，调查适合解释 Claude 用户内部的差异，不能直接外推到整个劳动力市场。

人们想要合作，而非单纯替代

超过一半的受访者希望 AI 帮助他们完成有意义的工作。略超一半希望它自动处理繁琐任务。约三分之一希望 AI 带来的经济收益得到广泛分享。

即使用户预计能力会快速提升，他们对理想未来的核心想象仍是保留意义、减少苦差事、共享收益。

完整译文

引言

一年前，Claude 的大多数使用形式还是用户与助手之间的对话。随着 Claude Code 和 Cowork 快速增长，Claude 会话如今越来越多地变成长时间运行的智能体任务。聊天记录已经无法完整呈现人们使用 AI 的方式。因此，我们研究 Claude 经济影响的方法也必须随之调整。

为了跟上这种变化，我们对经济指数的数据管线做了几项调整。在本版报告中，我们：

  • 以更高频率抽样数据，从而把使用模式细化到小时层面。
  • 引入一个新分类器，为每次对话的输出加上类别标签。
  • 分享粒度更细的数据。我们分别列出 chat 与 Cowork 对话（合称“Claude 对话”）以及 1P API 的结果，并按月汇总。1

附录介绍了更多方法调整。这些变化合在一起，让我们能更清楚地看到 AI 如何映照经济生活，又如何逐步渗入其中。

此外，我们过去一直看不到 Claude 在用户会话之外产生了什么影响。人们认为 AI 正在怎样改变自己的工作或可获得的机会？AI 的使用方式会不会塑造他们的预期？在理想世界里，他们希望从 AI 那里得到什么？我们报告了 Anthropic Economic Index Survey 的首批发现。这项调查于 2026 年 4 月启动。

下面先概述主要发现。

第一章展示外部世界的节律如何塑造 Claude 的使用。工作相关请求在周末减少，但最高薪职业中的降幅没有那么明显。人们往往在早晨询问新闻，睡眠建议请求在凌晨 5 点左右达到高峰，税务请求则在申报截止日前激增。

第二章研究人们从 Claude 会话中实际带走的成果。这些成果高度依赖所使用的产品。例如，chat 和 Cowork 提供的解释多于 Claude Code。输出性质也会影响人们与 Claude 的互动方式。创建网站时，很多判断可以交给 Claude；翻译文档时，答案则在很大程度上由原文决定。我们还发现，算力越多，通常会对应价值越高的成果。某项输出消耗的 token，会随着该工作的估计价值上升。

第三章给出 Anthropic Economic Index Survey 的首批结果。我们通过隐私保护系统，把调查与 Claude 使用数据关联起来。人们的预期和体验会随 Claude 的使用方式呈现系统性差异。最常以自动化方式使用 Claude 的人，预计未来一年 AI 会承担自己更多任务。然而，他们对这种变化给工作的影响最为乐观，并期待薪酬、就业安全和工作意义得到改善。

节律

我们新的隐私保护遥测系统每天持续抽取一部分对话。这让我们能够研究每日和每小时的使用模式。此前每期经济指数报告只能使用一个为期七天的样本。新分析捕捉到了世界各地工作节奏的起伏。2

我们发现，Claude 的使用会映照工作周的节奏，而个人提示会在周末激增。小时级数据还能捕捉一天之内的规律。人们最常在凌晨 5 点左右询问睡眠建议，在下午 6 点左右索要食谱。使用情况也会反映重要日期。例如，在 4 月 15 日美国报税截止日前，税务请求大幅上升。

工作周

样本期内，chat 和 Cowork3 对话中被归为个人用途的比例，在工作日约为 35%，到周末则升至略低于 50%（图 1.1）。离开工作周后，用户对话会从商务通信、营销文案和演示文稿，转向情感支持、医疗问题和投资建议。这种转变在高收入国家最明显。

Claude Code 和 1P API 流量也呈现类似规律。这里的 1P API 指直接经由 Anthropic 路由的 API 流量。不过，两者的个人用途基准比例都更低。4

请求聚类5让我们可以再深入一层，观察哪些具体的 Claude Code 任务在工作日与周末之间变化最大。到了周末，降幅最大的 Claude Code 使用聚类包括后端架构、API 调试和数据存储。增幅最大的聚类则包括 AI 智能体设计、量化交易和游戏。

周末也可能给人们留出探索新事业的空间。在各个国家，与创业相关的对话都在星期六和星期日达到最高水平。不过，求职活动会与其他工作任务一样在周末下降。6

每日节律

逐小时观察时，Claude 的使用呈现出日常生活的节奏。图 1.2 展示全球流量中不同请求聚类每小时出现的频率，并以各自的整体平均值为基准。7

人们会在当地时间早上 7 点询问新闻。商务通信请求，例如起草电子邮件，会沿着工作日的进程变化，并在上午 10 点至 11 点出现一个小高峰。食谱请求的峰值尤其显著：下午 6 点的出现频率是平均水平的 2.3 倍。媒体推荐最集中于晚间，人们则会在黎明前几个小时寻求睡眠建议。

到了夜间和周末，即使人们仍然使用 Claude 工作，任务也更偏向高薪职业（图 1.3）。我们无法确切识别提出这些请求的人的职业。不过，这可能反映了一个事实：营销经理或计算机程序员等高薪职业从业者，更可能在传统工作时间之外继续工作。相比之下，与工资处于后两个四分位组的职业相关的任务，例如电话营销和文书工作，在全部对话中的占比会降得更低。这种规律并非完全由计算机和数学类任务造成。稳健性检验移除这些职业后，较高工资四分位组的任务在夜间和周末仍然增加。

报税日

本报告的样本期覆盖了美国个人的报税截止日。图 1.4 显示，临近截止日时，税务对话占比大幅上升。4 月 14 日，税务相关聚类的出现频率达到 5 月平均日的 8 倍，并在 4 月 15 日维持在大致相同的高位。到了 4 月 16 日，其频率骤然下降。

成果

本章按照对话产生的成果，对每次 chat 和 Cowork 对话进行分类。下文把它们称为“Claude 对话”。8成果被划分为 30 多个类别。我们把 Claude 在对话中生成的主要输出称为成果。它可能是一份文档、一段解释、一段代码或一篇学术论文，也可能是其他内容。它既可以显示在聊天窗口里，也可以作为单独文档呈现。完整成果类别清单见附录。

分类器认定，93% 的 Claude 对话产生了成果（图 2.1）。9最常见的成果是解释，占全部对话的 17%；其次是文档与报告，占 15%；指导占 11%。解释或指导等对话型输出，与文档或演示文稿等书面交付物，各自约占对话的三分之一。应用或脚本等代码和技术工作约占六分之一。

输出是什么，并不能说明它拿来做什么。同一种成果既可能是工作交付物，也可能是个人项目。接下来我们考察这种划分。

每种成果用于什么目的？

我们在 1 月的经济指数中引入了一项基础分类方法，把每次对话分成工作、个人或课程作业。这里，我们把这种划分应用于 Claude 对话产生的成果（图 2.2）。

有些成果类别几乎总是用于个人目的。产生创意写作、指导和食谱的对话中，超过 80% 被归为个人用途。同一类别内部的个人用途与工作用途也可能截然不同。例如，个人创意写作主要是同人小说、世界观构建和诗歌。工作相关部分占 13%，主要形式是短视频脚本、影视剧本和演讲稿。最可能与工作相关的类别包括制作营销内容，占 80%；撰写博客或文章，占 81%；编写数据库查询，占 82%。

许多输出用于个人目的和工作目的的概率大致相同。计划或策略中，44% 与工作相关，49% 属于个人用途；翻译中，42% 用于工作，44% 用于个人目的。例如，个人规划成果最常见的是旅行行程和健身计划。工作相关计划则最常涉及创业策略或内容策略。

课程作业的典型成果包括学术论文与学位论文、教育材料和数学类查询。不过，每个类别也都有不可忽视的部分被划入工作和个人用途。

我们还可以反过来提问。与其问每种输出用于什么，不如看工作、个人和课程作业对话分别倾向于产生哪些成果。工作对话最常产生文档与报告，占 20%；随后是解释，占 9%；电子邮件草稿占 7%；分析与摘要占 6%。课程作业对话与此大体相似。文档与报告同样排在首位，占 21%；解释紧随其后，占 20%；教育材料占 11%；学术论文占 6%。与之相反，也不令人意外的是，个人对话中只有 6% 会产生文档。其最常见的结果是解释和推荐，分别占 25% 和 22%。

成本随工作价值上升

生成这些输出需要算力。我们发现，算力通常会随工作的价值增加。我们用 token 衡量每次对话的计算成本。这个指标包含处理和生成的文本，也包括 Claude 的内部推理。随后，我们根据分类所得任务，把每次对话映射到通常负责该任务的职业，再比较不同职业。在本节中，分析仅限工作相关对话。

图 2.3 左图显示，单次对话 token 数的中位水平与映射职业的工资中位水平呈正相关。10例如，营销经理的时薪约为编辑的两倍，分别是 80 美元和 37 美元；映射到营销经理任务的对话，消耗的 token 约为编辑任务的 2.5 倍。必须承认，这种关系有很大噪声，也存在明显的离群值。例如，药剂师的收入接近统计助理的三倍，时薪分别为 68 美元和 24 美元；但映射到药剂师任务的对话，token 使用量只有统计助理任务的约二十分之一。

生成不同成果所消耗的 token 也显示出类似规律。更复杂、价值更高的输出，通常会比简单输出消耗多得多的 token。例如，构建应用的对话使用的 token 超过对话中位水平的三倍。处在另一端的典型解释，只使用中位对话约五分之一的 token。token 消耗随工资上升的梯度中，约 44% 可以由输出构成解释。高薪职业更可能产生算力密集型成果。

这在经济层面为什么重要？在映射到高薪职业的对话中，Claude 每轮产生的输出更多，是基准的 1.34 倍；用户参与也更多，轮次达到 1.53 倍；扩展思考的启用频率也更高，为 34%，对照水平为 31%（表 2.4）。关键在于，这些指标会一起上升。Claude 产出更多，并不意味着用户产出更少。如果人类继续参与价值最高的任务，这种模式看起来更像是增强劳动，而不是替代劳动。它也表明，在一定程度上，价值越高的成果，成本也越高。下一节会研究每次对话中的决策有多少被委托给 Claude。

Claude 有多大自主权自行决策？

我们用 1 到 5 分衡量自主性，从“无”到“极高”。容易描述或明确指定的任务只需要很少的自主性。自主性最低的输出包括数学或计算、翻译和问答。高自主性任务需要从许多可能选项中作出选择，例如创建应用与网站、游戏或演示文稿。这类工作需要持续判断，过去一直很难实现自动化。通过比较 Claude chat、Cowork 与 Claude Code 的自主性水平，我们展示这种情况正在发生变化。

在几乎所有输出类别中，也就是图中 31 类成果里的 26 类，Claude Code 的 AI 自主性都高于 chat 或 Cowork。11例如，同样是生成脚本和代码片段，Claude Code 对话的自主性平均比 chat 或 Cowork 高 0.53 分，量表范围为 1 至 5 分。纵观全部对话，平均差异为 0.37 分，主要来自两个方面。12

大约三分之二的差异，可以由同一任务在 Claude Code 上获得了更多委托来解释。博客和文章能说明这一点。两个界面背后的请求和任务相似，但人们与 Claude 合作的方式差别很大。在产生博客或文章的 chat 和 Cowork 对话中，往返互动的中位数是 13 轮；产生博客的 Claude Code 会话中位数则只有一条人类提示。剩余三分之一来自两个界面的输出类型构成不同。

人们可能怀疑，这种差异只反映了模型选择。Claude Code 会话使用最强模型的频率确实高得多。其中 54% 由 Opus 提供服务，chat 和 Cowork 对话只有 10%。然而，比较使用相同模型的对话后，这一差距仍然存在。例如，在使用 Sonnet 的对话中，Claude Code 会话的自主性仍然高出 0.26 分。这表明，所使用的产品很可能比底层模型更重要。

如果不局限于界面之间的比较，用户委托最多的输出类型，也正是消耗算力最多的类型。在不同成果之间，平均自主性会与 token 使用中位数一同上升。在 chat 和 Cowork 上，相关系数 r 为 0.68，详见附录图 A.2。

Claude 回答的理解难度高于提问

分类器为每次对话估算两个阅读水平。一个对应用户提示，另一个对应 Claude 的回答。该指标用理解文本所需的受教育年数表示。13我们发现，阅读水平会随成果类型大幅变化。一条最终产生学术论文的平均查询，需要超过 16 年教育才能理解，大致相当于本科水平。其中 15% 的对话达到博士水平或更高，也就是需要 20 年或更多教育。另一端是产生食谱或指导的对话。理解这些提示所需的教育年数不到 10 年。

一般来说，输出阅读水平较高的成果类型，提示的阅读水平也较高。所有对话中的相关系数为 0.87。不过，我们也观察到，Claude 的输出在几乎每个类别中都比提示具有更高的理解水平，平均差距约为一年教育。用户描述要构建某种内容时，差距最大。图片和图形相差 2.6 年，游戏相差 1.9 年，应用和网站相差 1.7 年。部分差距可能只是语域造成的。提示往往简短且口语化，Claude 则倾向于用经过润色的文字回答。不过，面向受众的写作几乎没有差距。博客为 −0.1 年，学术论文为 +0.0 年，电子邮件为 +0.3 年。可能的原因是，这些提示通常包含草稿语言或来源材料，其语域本来就与目标输出相同。

感受与预期

前两章展示了人们怎样使用 Claude，却没有深入说明人们如何体验工作中的 AI。他们预计自己的岗位和工作场所会怎样变化？他们如何看待 AI 当前及潜在的影响？他们对这项技术抱有什么希望？2025 年 12 月，我们使用 Anthropic Interviewer 访谈了 81,000 名 Claude 用户，并由此得到一幅图景：受访者报告生产力大幅提升，同时也担心自己被替代。这种担忧集中在职业生涯早期的劳动者，以及我们观察到 Claude 承担工作最多的职业中。14

2026 年 4 月，我们启动 Anthropic Economic Index Survey，以推进这项研究。调查让我们能直接询问人们使用 AI 和工作的体验，并探索回答如何随 Claude 使用情况变化。我们使用隐私保护方法，把调查回答与 5 月中旬至 6 月上旬的使用数据关联起来。为了描述每名受访者的使用模式，我们在这个时间窗口内随机抽取每人最多 20 次会话。抽样覆盖 Claude.ai、Cowork 和 Claude Code，使会话构成能够反映每个人通常怎样使用不同界面。为减少抽样噪声，我们排除了会话少于 5 次的受访者。最终关联样本约有 9,700 名调查受访者。

我们发现，大多数受访者预计未来一年 AI 会取得显著进步。人们对 AI 能力的看法取决于自身经验、所在地区以及职业受到 AI 影响的程度。不过，他们对未来进步速度的预期却惊人地一致。这符合“水涨船高”的情形，也就是 AI 能力会普遍提升。

人们对这种进步会怎样影响自身职业，意见就没有这么一致。职业生涯早期的劳动者认为 AI 能完成自己工作中最高比例的任务，也最担心失业。然而，与一种常见担忧相反，把最多任务委托给 Claude 的人，对未来劳动市场结果最为乐观。他们也认为自己的技能正在变得更有价值。尽管身处 AI 前沿，或许也正因为这种接近，普通受访者对未来十年的希望并不是被取代，而是合作。他们希望 AI 能保留有意义的工作，把苦差事自动化，并让收益得到广泛分享。

谁回答了经济指数调查？

经济指数调查不能代表总体人口。我们接触的是随机抽取的 Claude 用户；完成调查的人可能存在自我选择；我们的分析还排除了低频用户。图 3.1 把调查受访者的职业构成以橙色表示，并与美国就业构成的灰色数据并列。计算机与数学职业的代表性过高，约占调查受访者的 30%。这与它们在 Claude 使用中的占比相当，却远高于其在美国就业中 4% 的占比。管理职业占受访者的 23%15，也远高于其 7% 的就业占比，尽管管理任务只占会话的 4%。这种差距符合一种解释：管理者会使用 Claude 完成管理工作以外的任务。在调查中，很多受访者，尤其是经验更丰富的人，都把判断和管理列为 AI 欠缺的能力。运输与物料搬运、餐饮准备与服务、建筑与采掘等体力职业类别，在调查中的代表性不足。它们在 Claude 会话中的占比同样偏低。

AI 与工作任务

研究 AI 影响时，人们经常关注职业暴露程度，也就是某项职业中有多大比例的任务可以用 AI 完成。我们在此前研究中建立了“观察暴露”指标，用来衡量某职业的任务中，有多大比例已经被观察到由 Claude 执行。我们把它与常用的“理论暴露”指标作比较。后者衡量大语言模型理论上能完成某职业中多大比例的任务。

还有一种理解职业暴露的方法，就是直接询问人们 AI 能完成自己多少工作。我们询问受访者，AI 目前能完全独立完成其工作任务中的多大比例。下文称之为“自报暴露”。我们还询问他们预计 12 个月后 AI 能承担多大比例，下文称“预期暴露”。回答分为五档，从“几乎没有”到“几乎全部”。接近六成受访者为明年选择了比今天更高的一档。超过三分之一预计，明年 AI 将能完成其大部分或几乎全部工作任务（图 3.2）。

图 3.3 把自报暴露和预期暴露，与观察暴露和理论暴露进行比较。我们提出两个问题。第一，人们自报和预计 AI 能做的工作，与不同职业的观察及理论暴露指标是否一致？第二，观察或理论暴露较高职业中的受访者，是否预计未来一年进步更快？第一个问题的答案是肯定的。自报暴露，也就是灰色点，与观察暴露和理论暴露都呈正相关。第二个问题的答案是否定的。自报暴露与 12 个月后预期暴露的最佳拟合线，也就是橙色点，基本平行。这意味着，无论所在职业的观察或理论暴露是高是低，人们预计未来一年 AI 可完成任务比例的增幅大致相同。17换句话说，软件工程师和施工经理预计 AI 在各自职业内取得的进展幅度大致相同。

还值得注意的是，自报暴露系统性地高于观察暴露。一种解释是，并非每个人都会执行某职业的全部任务，而我们的调查也不成比例地接触到了 AI 使用较多的人。18类似地，理论暴露是可能性的上限，而不是当前使用情况的衡量指标。因此，理论暴露又会系统性地高估自报暴露。

我们还研究了人们对 AI 当前和未来能力的认知，如何关联到受访者特征与使用模式。图 3.4 左图显示，对 AI 能力的认知与国家 GDP 呈负相关。19在高收入国家，人们自报 AI 当前能替自己完成的任务比例平均低约 10 个百分点。这种规律符合一种可能性：AI 可以替代低收入国家劳动者日常任务中的更大一部分。职业层面的暴露指标往往显示发达经济体暴露更高，但这两者并不矛盾。IMF 指出，发达经济体整体上面临更广泛的 AI 暴露；但低收入国家的劳动者可能较难获得互补技能和基础设施，而这些条件能让 AI 增强工作，而不是替代工作。我们此前的研究也表明，即使对任务构成差异作出调整，低收入经济体仍倾向于以更自动化的方式使用 Claude。

中图显示，自报和预期暴露也与工作经验年数呈负相关。20拥有至少 15 年经验的人，对 AI 可完成任务比例的估计，比工作第一年的人低约 10 个百分点。我们发现，这可能是因为经验丰富的劳动者积累了 AI 难以模仿的隐性知识或特定情境下的专业能力。在后续问题中，我们询问人们认为哪些任务是 AI 永远无法完成的，以及原因是什么。最常见的回答强调，AI 缺少其工作所需的判断力、情境意识和临场推理能力。受访者还指出，建立信任和管理人员等关系及人际维度，是 AI 无法复制的。拥有至少 15 年经验的人尤其常提到这一点。

与职业 AI 暴露的情况一样，我们发现，对 AI 能力未来改善的认知，基本不随人均 GDP 和经验年数变化。人们预计 12 个月后 AI 能完成的任务比例，普遍高于他们对 AI 当前能力的判断。

接下来，我们研究人们与 Claude 的互动方式，和他们目前对 Claude 能力的看法之间有什么关系。与此前报告一样，我们区分同 Claude 协作时的“自动化”与“增强”模式。如果 Claude 被要求在用户几乎不提供输入的情况下完成一项任务，我们就把该对话识别为自动化。具体而言，自动化占比是互动模式属于“指令式”或“反馈循环”的对话占比。前者如“翻译这份文档”，后者如“编辑这封邮件……让语气更随意”。21

图 3.4 右图显示，自报和预期暴露会随自动化占比上升。这可能是因为，委托行为能让人了解 AI 的能力。把整项任务交出去的人，可以直接看到 AI 独立完成了什么。另一种可能是，本来就相信 AI 能完成自己工作的人，更愿意把工作交给它。用工作会话占比或 Claude Code 会话占比替换自动化占比后，规律依然相同。22

AI 与就业

我们还询问人们，预计未来 12 个月自己的工作会怎样变化。超过三分之一的受访者认为，工作职责很可能或极有可能发生显著变化。这个判断适用于他们自己、同级同事、初级同事和高级同事。10% 的人认为，自己很可能或极有可能失业。这个数字略低于美国失业风险的年化发生率。23不过，受访者偏向拥有稳定工作的知识劳动者，这一群体的基准离职风险可能低于平均水平。因此，10% 仍可能表示他们感知到的风险有所升高。在被问及预测依据时，认为自己很可能或极有可能失业的受访者中，有 38% 把预测归因于 AI。24值得注意的是，受访者平均更担心别人失业，而不是自己失业。25他们尤其担心初级同事。超过三分之一的人认为，初级同事在未来一年失去工作的概率高于 60%。低收入国家的受访者也更担心所有人的失业问题。

以更自动化方式使用 Claude 的人，是否也更担心失去工作？我们询问人们，预计未来一年 AI 会怎样影响工作的六个维度。经济维度包括薪酬、就业安全和找到新工作的能力；内在维度包括意义、自主权和人际互动。随后，我们考察这些预期如何随 Claude 使用中的自动化占比变化。

在全部六个维度上，与更多以增强方式使用 Claude 的人相比，自动化会话占比较高的人，对 AI 明年给工作结果带来的影响更为乐观。最明显的差异出现在对未来薪酬和求职能力产生积极影响的预期上。26

一个自然的问题是，为什么自动化使用和态度会同向变化。一种可能是选择效应。对 AI 最热衷的人也最愿意把完整任务交给它。我们无法完全排除这一解释。不过，在控制用户使用 Claude.ai 的时间长短后，这些估计没有出现实质变化。使用时长可以看作热情程度的代理指标，因为它区分了较早和较晚采用产品的人。

另一种可能是，以更自动化方式使用 AI 的人，今天已经体验到了更多收益。与我们此前的发现一致，大多数人报告工作速度、范围和质量有所提升，比例分别为 86%、82% 和 69%。另有 27% 表示，AI 节省了原本需要购买服务的成本。

除了显著提高生产力，大多数人还表示自己借助 AI 学到了更多，占 68%；认为 AI 让自身技能变得更有价值的人占 57%。图 3.7 展示这两项结果如何随自动化会话占比变化。认为 AI 正在提高自身技能市场价值的人所占比例，会随自动化占比上升；表示自己学到更多的人所占比例则大致不变。

人们经常担心，把完整任务委托给 AI 就意味着把思考也交了出去。产出增加的代价可能是学习减少和技能退化。我们在这里没有看到这种规律。委托程度较高的人报告的学习比例与其他人相同。不过，这些都是自我评估。即使技能变得更有价值，即使一个人表示学到了更多，技能仍有可能退化。因此，这些数据不能排除技能侵蚀。

不同性别的使用方式有何差异？

到目前为止，我们研究了使用模式怎样关联到预期和行为。接下来，我们研究哪些人会以不同方式使用 Claude。最显著的差异来自性别。女性只占关联受访者样本的 12%，而且她们使用 Claude 的方式与男性不同。即使控制职业差异，女性使用 Claude 工作的可能性仍略低。她们在 Claude Code 上的会话占比低 0.24 个标准差，也就是 6.3 个百分点；自动化占比低 0.33 个标准差，也就是 7.3 个百分点。女性反而更倾向于迭代式使用 Claude。她们在 chat 上记录的活跃时间也比男性更多，这体现了更具协作性的参与方式。27

人们希望 AI 改造后的经济是什么样子？

Anthropic Economic Index Survey 呈现了人们对 AI 的积极与消极体验和态度。不过，我们以一个充满希望的问题结束调查。最后一个开放式问题请受访者“大胆想象：十年后，你希望由 AI 塑造的经济是什么样子？”我们使用分类器处理每份调查回答，并标记相关主题。下面展示最常被提到的五个主题。每个主题的更多说明见附录。

最常见的主题是 AI 对工作的增强。超过一半的受访者以某种形式表达了类似愿望：希望与 AI 合作完成有意义的工作，希望自己的职业仍然重要，或者希望新产业出现并创造新的就业机会。与此同时，略超一半的受访者希望 AI 实现自动化，尤其是自动处理工作中繁琐的部分，从而让自己拥有更多自由时间，也能在工作之外获得更多追求意义的空间。第三常见的主题是共享繁荣，约三分之一的受访者希望 AI 带来的经济收益得到广泛分享。

讨论

AI 正在迅速扩散到整个经济体系，并进入越来越多的产品界面，产生智能水平越来越高的输出。在早期 AI 聊天界面中，使用方式很简单。它被限制在聊天窗口内，没有网页搜索、工具调用、成果或其他功能。如今，Claude 模型可以通过 Claude Code 和 Cowork 自主运行数小时。随着这些形态发生变化，用户群体也在改变。早期采用者技术背景很强。最近加入的用户，则把 Claude 用于劳动力市场工资较低的任务。

本报告采取了几个步骤，以获得信息更丰富的测量结果。首先，我们开始测量更多数据，并提高测量频率，以小时为单位处理样本。这揭示了日常生活的节律如何刻进使用日志，也为未来研究开辟了新方向。其次，我们开始记录成果，也就是人们从 Claude 那里带走的输出。这些成果让 Claude 的工作产出更容易理解，也呈现了一些符合直觉的规律。

最后，使用数据能提供的信息终究有限。我们的调查让我们第一次能直接询问人们怎样使用 AI，以及对此有何感受。我们发现，调查受访者使用 AI 完成的工作比我们原先认可的更多。他们报告 AI 能完成的工作比例，高于其职业的观察暴露指标所暗示的水平。在预测明年能力时，超过 35% 的人认为 AI 将能完成其大部分工作。

准确分类 Claude 所做的工作，仍会是一项不断变化的任务。例如，随着 AI 能力提高，不同 AI 之间可能会越来越多地互动和交换信息。这些过程甚至可能让人类或简单分类器难以理解。最终，Claude 对经济的影响不仅会体现在使用日志中，也会体现在就业和生产力等经济总量上。不过，AI 最早产生影响的地方，很可能就是它承担工作最多的地方。因此，阐明这些持续变化的使用模式，仍然会是帮助公众了解情况的重要途径。
