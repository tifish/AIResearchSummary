构建云端 Agent 一年，我们学到的五件事

来源：Cursor ｜ 作者：Josh Ma ｜ 2026-06-02 ｜ 原文
一句话总括

云端 Agent 真正的工程难点，不是把本地 Agent「搬上服务器」，而是要为它重建一整套开发环境 + 持久化执行 + 编排的「操作系统层」——模型越聪明，这层基础设施越是决定成败。

01开发环境本身就是产品

过去一年最大的发现：决定云端 Agent 产出质量的单一最大因素，是它有没有一个像真人开发者那样的完整环境。本地 Agent 能「免费」继承你笔记本上的环境，云端却要从零重建，而且做得不到位时几乎没有报错——表现只是产出质量「悄悄变差」，很容易被误判成「模型不行」。为达到「完整环境」，团队不得不重建大量基础设施：VM 的休眠/恢复、镜像的快照/还原/分叉、受控网络访问，乃至密钥脱敏、网络策略、凭证管理——本质上是在为 Agent 搭建企业级 IT。
洞见：当模型足够强，环境就成了产能瓶颈——「模型变笨了」往往其实是「环境没配齐」。

02长时运行的 Agent 需要持久化执行

云端 Agent 跑在隔离 VM 中，可大规模并行、承接动辄数小时的长任务，但也因此暴露在推理服务中断、Pod 被替换、EC2 节点宕机等扰动下。最初的「工作窃取」架构是本地思路的照搬，脆弱到早期 Beta 只有「一个 9」的可靠性。当团队发现自己正在重造 Temporal 已解决的持久化原语（重试、跨机调度、节点故障容错）时，索性迁了过去——可靠性一举跨过「两个 9」，且能扛住跨越数天乃至数周的运行。架构上还从「永生」工作流改为完成单个任务即退出的短工作流，让版本升级更简单。
洞见：遇到「重试、调度、容错」这类通用难题，别自己造轮子——成熟的持久化执行框架比手写 loop 靠谱一个量级。

03让 Agent、机器与对话状态相互解耦

一个云端 Agent 早已不是「一台机器上的一个循环」：它可能在一台机器运行、把异步子 Agent 撒到多台、甚至子 Agent 比父 Agent 活得更久。因此团队把Agent 循环、机器状态、对话状态拆成独立组件——Agent 循环放在 Temporal 而非 VM 里，于是可独立管理 Pod 生命周期，并启用只读 VM、预热 VM 等优化。对话侧则把存储与流式层独立出来，做成追加写（append-only）机制：当某步失败、流出过部分输出后又被重试，客户端能侦测到并「回退」流、用新数据覆盖旧数据。
洞见：解耦让「机器可换、循环可迁、对话可重放」三者互不拖累——这是并行与容错的前提。

04懂得「让开」：把控制权交还给 Agent

早期不信任 Agent，harness 会在每个任务后复查、强制提交并推送。随着模型变强，逻辑被不断从 harness 移交给Agent 自己控制的工具：多仓库从硬编码改为「给它仓库布局 + 分支/PR 工具，让它自己决定」；CI Autofix 从「harness 抓日志写进 VM」简化为「直接给它 GitHub CLI，大输出自动落盘供检索」。harness 没有消失，只是内容在变——比如计算机使用仍需专门子 Agent（独立模型路由、提示、录屏，共享环境里的 VNC 与 Chrome，可直接跑 Playwright），但何时调用由 Agent 自己决定。此外，云端提示词更鼓励自主，因为「阻塞等待」的代价远高于本地（没人盯着，可能一等就是几小时）。
洞见：harness 的演化方向是「确定性逻辑越来越少、交给 Agent 的工具越来越多」——阻塞成本越高，自主性越值钱。

05下一步：自愈式环境

未来的目标是跳出「手把手 vs 完全放手」的二选一，给 Agent 理解周遭系统的工具：当缺密钥、网络被封、或环境阻碍进展时，Agent 能主动上报并自我修复。团队把其中一条路径称为 autoinstall。结论是：云端 Agent 仅最近几个月就进步巨大，且变化速率只会加快——而 Cursor 的价值在于让团队直接享用这片广阔能力面，无需自己搭建和维护底层基础设施。
洞见：从「人来兜底环境」到「环境自愈」，是 Agent 基础设施成熟度的下一个分水岭。

最值得带走的 5 条洞见

  1. 环境 = 产能。模型越强，缺失的开发环境越会以「质量悄悄下降」而非报错的形式拖垮产出。
  2. 可靠性靠框架买。迁到 Temporal 让可靠性从「一个 9」跨到「两个 9」，如今日处理 5000 万+ actions、覆盖 700 万+ 工作流。
  3. 三态解耦是并行与容错的地基。Agent 循环 / 机器 / 对话彼此独立，才能换机、迁移、回放流而互不拖累。
  4. 趋势是不断「让开」。逻辑从 harness 迁往 Agent 工具；阻塞成本高，故云端 Agent 被设计得更自主。
  5. 它正在自我证明。Cursor 内部已有 40%+ 的 PR 出自云端 Agent，下一站是能自报问题、自我修复的「自愈式环境」。
