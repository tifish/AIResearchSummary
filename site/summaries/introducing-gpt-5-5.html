Introducing GPT-5.5

GPT-5.5 的真正贡献，是把前沿模型从更强的问答系统推进为更能持续执行真实工作的代理系统。

核心变化

更强的不只是答案，而是完成任务的能力

文章把 GPT-5.5 描述为更适合“真实工作”的模型。它能更快理解意图，并在多步骤任务中规划、调用工具、检查结果和继续推进。

这改变了读者对 AI 影响的判断。AI 的价值不再只来自单次生成，而来自跨上下文、跨软件、跨时间的执行能力。

效率信号

更高能力没有换来明显延迟惩罚

OpenAI 称 GPT-5.5 在真实服务中的逐 token 延迟接近 GPT-5.4，同时在 Codex 任务中用更少 token 完成同类工作。

这很关键。更强模型如果太慢或太贵，就很难进入日常工作流。GPT-5.5 的叙事重点是能力、速度和 token 效率同时改善。

代理式工作流的四步闭环

GPT-5.5 的能力提升集中在需要持续行动的任务，而不是一次性回答。

1

理解模糊目标。模型先判断用户真正要完成什么。

2

制定计划并调用工具。它能在代码、网页、文件和应用之间移动。

3

验证输出。文章多次强调测试、检查假设和发现失败原因。

4

继续推进直到完成。长期任务中的持续性，是这次升级最重要的信号。

关键能力对比

这些数据说明 GPT-5.5 的提升覆盖编程、办公、科研、工具使用和安全，而不是集中在单一 benchmark。

最值得注意的变化，是模型开始表现出对系统结构的理解：哪里坏了，修复应该落在哪里，改动会影响什么。

科研部分展示了更大的想象空间

GPT-5.5 在 GeneBench 和 BixBench 上取得提升。文章强调这些任务需要处理模糊数据、质量控制失败、隐藏混杂因素和统计方法解释。

这说明 AI 对科研的影响正在从“解释论文”转向“参与分析流程”。它可以帮助研究者从问题走到实验、代码、结果和报告。

一个具体例子：免疫学研究者用 GPT-5.5 Pro 分析 62 个样本、近 28,000 个基因的表达数据，并生成详细研究报告。

这种案例的意义在于，模型开始压缩专家团队的分析周期，而不是只节省写作时间。

风险能力也在上升

安全策略成为发布的一部分

OpenAI 将 GPT-5.5 的生物化学和网络安全能力按 Preparedness Framework 视为 High。文章称模型未达到 Critical 网络安全能力级别，但相较 GPT-5.4 明显提升。

这改变了开放访问的逻辑。越强的模型越需要可信访问、认证使用、监控和针对高风险请求的限制。文章的政策含义是，防御能力要扩大，滥用路径也要收紧。

落地成本

API 定价显示 GPT-5.5 面向高价值任务

标准 gpt-5.5 API 价格为每 100 万输入 token 5 美元、输出 token 30 美元。gpt-5.5-pro 为输入 30 美元、输出 180 美元。

价格高于 GPT-5.4，但文章用 token 效率和任务完成率来解释成本。它更适合交给模型执行长任务，而不是只做低成本闲聊。

内部采用

OpenAI 自己把 Codex 当作生产工具

文章称 OpenAI 每周超过 85% 员工使用 Codex。Finance 团队用它审查 24,771 份 K-1 税表，共 71,637 页。

这些数字比单个 demo 更有说服力。它们说明模型已经进入重复、繁琐、需要审查的组织流程。

最值得带走的洞见

代理能力是主线GPT-5.5 的重点是持续执行，而不是更漂亮的单轮回答。
编程是先行场景复杂工程任务最能暴露模型是否真的理解系统。
科研开始被重塑模型正在进入数据分析、假设检验和工具构建环节。
效率决定普及接近 GPT-5.4 的延迟和更低 token 使用量，让更强模型更容易进入工作流。
安全是能力的配套条件网络安全和生物能力越强，可信访问和监控越重要。
