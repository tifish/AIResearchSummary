GPT-5.5 正式发布

Introducing GPT-5.5

一句话总括：GPT‑5.5 把更强的推理、工具操作和长期执行能力结合起来，让模型能以接近上一代的延迟，用更少 token 完成更复杂的真实工作。

能力增长开始转化为完整的工作执行

GPT‑5.5 的关键变化不是单项答题分数上升。它更能理解任务意图，也更能在模糊环境中持续行动。模型会规划步骤、调用工具、检查结果，并把修改扩展到相关代码或文档。

这改变了对 AI 能力的判断标准。重点不再只是“能否回答”，而是能否把一个混乱的多阶段任务真正做完。

更强，也更节省 token

GPT‑5.5 在真实服务中的逐 token 延迟与 GPT‑5.4 相当，但整体智能水平更高。它完成相同 Codex 任务时通常消耗更少 token，也需要更少重试。

推理团队还利用 Codex 分析数周的生产流量，并设计负载划分启发式算法。相关优化让 token 生成速度提高逾 20%。

这说明模型规模增长并不必然意味着实际工作流变慢。软硬件协同设计可以把能力提升转化为可用效率。

能力越强，部署边界越重要

OpenAI 将 GPT‑5.5 的生物与化学能力、网络安全能力评定为 Preparedness Framework 下的 High。

GPT‑5.5 未达到网络安全的 Critical 等级，但相较 GPT‑5.4 已明显增强。因此，发布同时引入了更严格的分类器、重复滥用保护和可信访问机制。

真正可持续的开放路径，需要把广泛防御用途与高风险操作控制同时纳入部署设计。

洞见是：模型开始从解释已有知识，迈向提出分析、构建工具和贡献可核验结果。

评测结果

2026 年 4 月 24 日更新：GPT‑5.5 和 GPT‑5.5 Pro 现已在 API 中提供。系统卡也已更新，介绍适用的额外防护措施。

我们正式发布 GPT‑5.5。这是我们迄今最聪明、使用起来也最直观的模型。它代表着一种全新计算机工作方式的下一步。

GPT‑5.5 能更快理解你想做什么，并独立承担更多工作。它擅长编写和调试代码、在线研究、分析数据、创建文档和电子表格、操作软件，以及跨工具持续推进，直至任务完成。

你不再需要仔细管理每一步。可以直接交给 GPT‑5.5 一个杂乱的多阶段任务，并让它自行规划、使用工具和检查结果。它还能在模糊条件下确定方向，并持续执行。

提升在智能体编程、计算机操作、知识工作和早期科学研究方面尤其明显。这些领域的进展依赖跨上下文推理，也需要模型长时间采取行动。

GPT‑5.5 在智能水平显著提高的同时，没有牺牲速度。规模更大、能力更强的模型通常需要更长的服务时间，但 GPT‑5.5 在真实服务中的逐 token 延迟与 GPT‑5.4 相当。

它完成相同 Codex 任务时使用的 token 也显著减少。因此，它不仅能力更强，效率也更高。

我们为 GPT‑5.5 配备了迄今最强的一套防护措施。目标是在保留有益用途访问权的同时，降低滥用风险。

我们依据完整的安全和 Preparedness Framework 体系评估了该模型。我们与内部及外部红队合作，还针对高级网络安全和生物能力进行了专项测试。发布前，我们也收集了近 200 家可信早期访问合作伙伴对真实用例的反馈。

今天，GPT‑5.5 开始向 ChatGPT 和 Codex 的 Plus、Pro、Business 与 Enterprise 用户推出。GPT‑5.5 Pro 则开始向 ChatGPT 的 Pro、Business 与 Enterprise 用户推出。

API 部署需要不同的防护措施。我们正在与合作伙伴和客户密切协作，确定大规模服务所需的安全要求。GPT‑5.5 和 GPT‑5.5 Pro 很快也会进入 API。

模型能力

OpenAI 正在建设面向智能体 AI 的全球基础设施，让世界各地的个人和企业都能借助 AI 完成工作。

过去一年，我们看到 AI 显著加快了软件工程的速度。随着 GPT‑5.5 进入 Codex 和 ChatGPT，同样的转变也开始延伸到科学研究，以及人们在计算机上完成的更广泛工作。

在这些领域，GPT‑5.5 的提升不只体现在智能水平上。它解决问题的方式也更加高效，通常能以更少 token 和更少重试获得质量更高的结果。

在 Artificial Analysis 的 Coding Index 中，GPT‑5.5 以竞争性前沿编程模型一半的成本，实现了当前最佳水平的智能表现。

Artificial Analysis Intelligence Index（在新窗口打开）是外部机构对 10 项评测进行加权平均后得到的指标。这些评测是 AA-LCR、AA-Omniscience、CritPt、GDPval-AA、GPQA Diamond、Humanity’s Last Exam、IFBench、SciCode、Terminal-Bench Hard 和 τ²-Bench Telecom。

智能体编程

GPT‑5.5 是我们迄今最强的智能体编程模型。

Terminal-Bench 2.0 测试需要规划、迭代和工具协作的复杂命令行工作流。GPT‑5.5 在这项评测中取得了当前最佳的 82.7% 准确率。

SWE-Bench Pro 用于评估真实 GitHub issue 的解决能力。GPT‑5.5 得分为 58.6%，单次端到端完成的任务多于以往模型。

Expert-SWE 是我们的内部前沿评测，针对长期编码任务设计。这些任务预计由人类完成所需时间的中位数为 20 小时。在这项评测中，GPT‑5.5 同样超过了 GPT‑5.4。

在这三项评测中，GPT‑5.5 的成绩都高于 GPT‑5.4，同时使用的 token 更少。

模型的编程优势在 Codex 中尤其明显。它可以承担实现、重构、调试、测试和验证等不同类型的工程工作。

早期测试显示，GPT‑5.5 更擅长真实工程工作所依赖的行为。它能在大型系统中保持上下文，对模糊故障进行推理，使用工具核查假设，并把修改贯彻到周边代码库。

知识工作

让 GPT‑5.5 擅长编程的同一组能力，也让它成为日常计算机工作的强大工具。

模型更能理解意图，因此可以更自然地完成知识工作的整个闭环。它会查找信息、识别重点、使用工具、检查输出，并把原始材料转化为有用成果。

在 Codex 中，GPT‑5.5 生成文档、电子表格和幻灯片演示文稿的能力优于 GPT‑5.4。

Alpha 测试者表示，它在运营研究、电子表格建模，以及把杂乱业务输入转化为计划等工作中超过了以往模型。

结合 Codex 的计算机操作技能后，GPT‑5.5 更接近一种“模型能与你一起使用计算机”的体验。它能看见屏幕内容，执行点击和输入，浏览界面，并精确地在工具之间移动。

OpenAI 内部团队已把这些能力用于真实工作流。目前，公司每周使用 Codex 的员工比例超过 85%。使用者来自软件工程、财务、传播、营销、数据科学和产品管理等职能。

传播团队使用 Codex 中的 GPT‑5.5 分析了六个月的演讲请求数据。他们建立了评分和风险框架，还验证了一个自动化 Slack 智能体。低风险请求可以自动处理，高风险请求则仍会转交人工审查。

财务团队使用 Codex 审查了 24,771 份 K-1 税表，总计 71,637 页。该工作流排除了个人信息，并帮助团队比上一年提前两周完成任务。

在 Go-to-Market 团队，一名员工自动化生成每周业务报告，每周节省了 5 至 10 小时。

科学研究

GPT‑5.5 在科学和技术研究工作流中也有提升。这些工作并不只是回答一个困难问题。

研究人员需要探索想法、收集证据、检验假设、解释结果，并决定下一步尝试什么。与其他模型相比，GPT‑5.5 更能持续完成这个循环。

值得注意的是，GPT‑5.5 在 GeneBench（在新窗口打开）上相较 GPT‑5.4 有明显进步。这是一项新评测，重点关注遗传学和定量生物学中的多阶段科学数据分析。

这些问题要求模型在监督指导很少的情况下，对可能含糊或存在错误的数据进行推理。模型还要处理隐藏混杂因素或质量控制失败等真实障碍，并正确实现和解释现代统计方法。

这项表现尤其引人注目，因为其中的任务通常相当于科学专家需要数天完成的项目。

在 BixBench（在新窗口打开）上也有类似结果。该 benchmark 围绕真实的生物信息学和数据分析任务设计。GPT‑5.5 在已有公开成绩的模型中取得了领先表现。

模型的科学能力如今已经足够强，可以作为真正的共同科学家，为生物医学前沿研究带来有意义的加速。

另一个案例中，配备定制执行框架的 GPT‑5.5 内部版本，帮助发现了一个关于 Ramsey 数的新证明（在新窗口打开）。Ramsey 数是组合数学的核心对象之一。

组合数学研究图、网络、集合和模式等离散对象如何组合。粗略地说，Ramsey 数研究的是：一个网络必须大到什么程度，才能保证其中出现某种秩序。

该领域很少出现新结果，技术难度通常也很高。GPT‑5.5 找到了一个有关非对角 Ramsey 数长期渐近事实的证明，之后又通过 Lean 验证。

这是一个具体案例。它表明 GPT‑5.5 的贡献不只限于代码或解释，还可以在核心研究领域提出令人意外且有用的数学论证。

早期测试者使用 ChatGPT 中的 GPT‑5.5 Pro 时，并没有把它当作一次性回答引擎，而是把它作为研究伙伴。

他们让模型多轮评议论文，压力测试技术论证，提出分析方法，并结合代码、笔记和 PDF 上下文工作。

这些用法的共同点是：GPT‑5.5 更能帮助研究人员完成从问题到实验，再到成果的整个过程。

Jackson Laboratory for Genomic Medicine 的免疫学教授兼研究员 Derya Unutmaz 使用 GPT‑5.5 Pro 分析了一个基因表达数据集。数据集包含 62 个样本和近 28,000 个基因。

模型生成了一份详细研究报告。报告不仅总结了发现，还提出了关键问题和洞见。他表示，这项工作如果交给团队完成，原本需要数月。

波兰波兹南 Adam Mickiewicz University 数学助理教授 Bartosz Naskręcki 使用 Codex 中的 GPT‑5.5，仅用一个提示词就在 11 分钟内构建了一款代数几何应用。

该应用可以显示二次曲面的交线，并把得到的曲线转换成 Weierstrass 模型。

之后，他又扩展了应用，加入更稳定的奇点可视化和可在后续工作中重复使用的精确系数。

对他而言，更大的变化在于 Codex 现在可以帮助实现定制的数学可视化和计算机代数工作流。过去，这些工作需要专门工具。

这些案例共同表明，GPT‑5.5 可以把专家意图转化为实际可用的研究工具和分析成果。

下一代推理效率

要让 GPT‑5.5 以 GPT‑5.4 的延迟提供服务，必须把推理重新视为一个集成系统，而不是一组相互孤立的优化。

GPT‑5.5 的设计、训练和服务都与 NVIDIA GB200 和 GB300 NVL72 系统协同完成。

Codex 和 GPT‑5.5 对团队实现性能目标发挥了重要作用。Codex 帮助团队更快地把想法变成可以运行 benchmark 的实现。它会勾勒方案、连接实验，并帮助判断哪些优化值得投入更多精力。

GPT‑5.5 也帮助团队发现并实现了技术栈中的关键改进。简单地说，模型参与改进了为它提供服务的基础设施。

其中一项改进涉及负载均衡和分区启发式算法。

在 GPT‑5.5 之前，我们会把加速器上的请求划分成固定数量的分块，以平衡计算核心之间的工作。这样一来，大型和小型请求就能在同一块 GPU 上运行。

但预先确定的静态分块数量并不适合所有流量形态。

为了提高 GPU 利用率，Codex 分析了数周的生产流量模式，并编写了定制启发式算法，用于优化工作分区和负载平衡。

这项工作的影响远超其规模，使 token 生成速度提高了 20% 以上。

推进网络安全，守护每个人的安全

世界需要为非常擅长发现和修补安全漏洞的模型做好准备。这是一项需要整个团队共同完成的工作。

整个生态系统都必须努力增强韧性。我们也需要通过普及模型访问和迭代部署，为网络防御的下一个时代做准备。

前沿模型的网络安全能力正在持续增强。这些能力也会广泛传播。

我们认为，最佳前进路径是确保这些能力能够用于加快网络防御，并增强整个生态系统。

GPT‑5.5 是迈向能够解决网络安全等全球艰难问题的 AI 的一次渐进但重要的进步。

去年 12 月发布 GPT‑5.2 时，我们主动部署了必要的网络安全防护，以限制模型可能遭受的网络滥用。

现在，GPT‑5.5 配备了更严格的潜在网络风险分类器。随着我们不断调整这些分类器，一些用户起初可能会觉得它们令人烦恼。

多年来，我们一直在 Preparedness Framework 中把网络安全列为一个类别。随着模型逐步增强，我们也持续开发、校准并迭代缓解措施，以便负责任地发布具备实质性网络安全能力的模型。

针对这一级别的网络能力，我们正在部署行业领先的防护措施。

我们去年随 GPT‑5.2 首次引入了网络安全专用防护。此后的每次部署中，我们都在继续测试、完善和扩展这些措施。

对于 GPT‑5.5，我们围绕高风险活动和敏感网络请求设计了更严格的控制，并增加了针对重复滥用的保护。

我们在模型安全、身份验证使用方式，以及违规用途监控方面的投入，使广泛访问成为可能。

数月以来，我们一直与外部专家合作，共同开发、测试和迭代这些防护措施的稳健性。

借助 GPT‑5.5，我们希望开发者可以轻松保护自己的代码。同时，我们也会更严格地控制恶意行为者最可能用来造成伤害的网络工作流。

我们正在扩大访问范围，以加快各层级的网络防御。

我们会通过 Trusted Access for Cyber 提供对网络安全用途更宽松的模型。该计划将从 Codex 开始。

发布时，符合特定信任条件的已验证用户，可以在限制更少的情况下使用 GPT‑5.5 的高级网络安全能力。

负责保护关键基础设施的组织，可以申请使用 GPT‑5.4‑Cyber 等网络安全用途更宽松的模型。这些组织必须满足严格的安全要求，并仅使用模型保护内部系统。

这能让更多经过验证的防御者获得更强的合法安全工具，同时减少不必要的阻碍。我们的目标是普及重要的防御能力。

用户可以在 chatgpt.com/cyber 申请可信访问。经过验证的防御工作在使用 GPT‑5.5 时，可以由此减少不必要的拒绝。

我们正在与政府合作伙伴协作，帮助保护公众赖以生存的关键基础设施。

我们共同探索高级 AI 如何支持可信官员的防御工作。这些官员负责的系统包括保护重要纳税人数据的数字系统，也包括地方社区的电网和供水设施。

在 Preparedness Framework 下，我们把 GPT‑5.5 的生物与化学能力和网络安全能力评定为 High。

GPT‑5.5 没有达到 Critical 网络安全能力等级。不过，我们的评测和测试显示，它的网络安全能力相较 GPT‑5.4 已有所提升。

此外，GPT‑5.5 在发布前经过了完整的安全和治理流程。流程包括准备情况评测、特定领域测试、面向高级生物与网络安全能力的新专项评测，以及与外部专家共同进行的严格测试。

这些工作反映了我们更广泛的 AI 韧性方案。我们认为，随着模型能力提升，这套方案十分必要。

我们希望，强大的 AI 能够提供给那些用它保护系统、机构和公众的人。

可行的路径包括可信访问、随能力同步增强的稳健防护，以及发现和应对严重滥用所需的运营能力。
