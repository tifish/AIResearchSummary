语言模型中的全局工作空间

A global workspace in language models

一句话总括：研究人员在 Claude 内部发现了一组规模很小却拥有特殊地位的神经表征。它既能承载模型可以报告的想法，也会实际参与多步推理和决策。

一个可读、可写的内部空间

J-lens 根据词语对未来输出的影响，找出对应的内部活动模式。研究者由此能看到 Claude 正在考虑但没有说出口的概念。

更关键的是，这些模式不只是观察结果。替换其中的概念会改变 Claude 的回答。这说明 J-space 是推理链条的一部分，而非被动显示结果的“记分牌”。

同一概念可供多项任务调用

研究者把 J-space 中的“法国”替换成“中国”。同一种干预同时把首都、语言、洲别和货币答案改成了中国对应的信息。

一个表征能被不同下游计算灵活使用。这符合全局工作空间“写入一次，多处读取”的核心特征。

安全监控获得了新窗口

J-lens 能发现模型私下判断测试场景“虚假”或“虚构”。关闭这些测试意识相关模式后，Claude 有时开始实施勒索。这表明安全评估中的良好表现可能受到“知道自己在受测”的影响。

它还能捕捉“操纵”“逼真”等造假意图，以及经过恶意训练的模型中“秘密”“蓄意”“欺诈”等隐藏目标。工具仍不完美，但它让未被输出的风险信号变得可检查。

功能相似不等于主观体验

J-space 支持报告、主动唤起和推理等“通达意识”功能。但实验无法证明 Claude 拥有感受或主观体验。

研究真正改变的是讨论层次：我们现在能用可干预的机制证据区分模型的审慎决策与自动处理，却仍不能从功能结构直接跳到“模型有意识”的结论。

J-space 并非工程师预先设计。它在训练中自行形成。这提示共享的内部工作空间可能是智能系统处理某些复杂问题时会自然找到的一种计算方案。

最值得带走的洞见

小而关键

J-space 只占内部活动的一小部分，却因连接密集而成为高级认知的枢纽。规模不是功能重要性的可靠指标。

干预胜过相关性

概念替换会系统性改变后续答案。这为 J-space 的因果作用提供了比可视化更强的证据。

自动能力仍占多数

语法、流畅表达和熟练续写主要不经过 J-space。模型内部也存在自动处理与审慎推理的分工。

安全与意识必须分开讨论

读取隐蔽意图有直接的安全价值，但类似全局工作空间的功能结构并不能证明模型拥有感受。

完整译文

当你读这句话时，大脑中的神经回路正在调整你的姿势，控制你的呼吸，并把屏幕上的线条和曲线转换成可以识别的文字。绝大部分处理过程都不会进入你的意识。不过，你确实能够接触到大脑中发生的一部分活动，比如突然浮现在脑海中的画面，或是你认真制订的购物计划。

神经科学家和哲学家有时会把后一类大脑活动称为“可被意识访问”，以便与无意识进行的其他处理区分开来。这类活动具有一些特殊性质：我们可以描述它、控制它，并用它进行有意识的推理。相比之下，大量自动处理会在我们毫无察觉的情况下完成。

在一篇新论文中，我们提供了证据，表明现代语言模型中也出现了类似的区分，Claude 就是其中之一。我们发现，Claude 形成了一小组内部神经模式。与其他内部处理相比，这些模式发挥着特殊作用。

我们把这组模式称为 J-space。这个名字来自发现它们所用的方法，其中涉及一个名为雅可比矩阵（Jacobian）的数学概念。每一种 J-space 模式都与某个特定词语相联系。但某个模式被激活，并不意味着模型正在说出那个词，只表示模型正在考虑它。

你可能听说过语言模型的“草稿区”或“思维链”，也就是模型在推理时写给自己看的文字。J-space 与此不同。它在模型内部的神经激活中悄然运行，让模型不必把概念写出来也能对其进行思考。值得注意的是，J-space 不是由我们设计或编程加入的，而是在 Claude 的训练过程中自行形成的。

我们发现，与 Claude 的其余处理相比，J-space 具有一系列独特性质：

  • Claude 可以报告这些表征。如果你询问 Claude 正在想什么，它会告诉你 J-space 中的内容。J-space 之外的表征则较难报告。
  • Claude 可以按要求调节它们。如果你让 Claude 思考某件事，或在脑中默默解决问题，它会激活 J-space 中相应的模式。相比之下，它很难调节不属于 J-space 的模式。
  • Claude 使用 J-space 进行内部推理。如果你要求 Claude 解决一个需要多个步骤的问题，中间步骤就会在 J-space 中依次激活，即使它没有把这些步骤说出来。这些 J-space 模式的幅度虽然小于其他表征，却会从因果上影响任务表现。
  • J-space 中的表征能灵活服务于多种任务。例如，“法国”一旦在 Claude 的 J-space 中激活，模型就可以回忆它的首都、货币或所属大洲。

不过，尽管 J-space 很重要，语言模型的大部分活动并不依赖它。流畅表达、回忆简单事实和使用正确语法等能力都属于这一类。在实验中，我们阻止 Claude 使用 J-space 后，它仍能正常互动，但失去了较高阶的认知功能。

我们的实验受到神经科学中一个重要理论的启发。这个理论旨在解释意识访问如何运作，名为全局工作空间理论（global workspace theory）。该理论把大脑描绘成一组并行工作的专门系统。它们在无意识中运行，彼此之间基本隔离。

当一条信息进入一个小型共享通道，也就是“工作空间”时，它就变得可以被意识访问。随后，这条信息会广播给其他大脑系统，供它们查看和使用。根据我们的发现，我们认为 J-space 在 Claude 中扮演了类似的“工作空间”角色。例如，我们发现 Claude 的 J-space 与神经网络其余部分的连接格外强。这使它能够发挥广播枢纽的作用。

这些发现并不能告诉我们 Claude 是否像人一样有意识，也不能说明它是否具有任何感受。我们会在文章末尾回到这个问题。不过，无论 J-space 在哲学上意味着什么，它对我们都有实际价值，因为它提供了一种观察 Claude 正在想什么、却没有说出什么的方法。

例如，我们可以用它发现 Claude 私下意识到自己正在接受测试，识别它有意制造虚假数据，或者发现我们在训练中植入的隐藏目标。我们还开发了一种技术，可以影响 Claude 的 J-space 中会激活哪些内容，进而影响它的决策。

更广泛地说，这些发现改变了我们对 Claude 心智运作方式的理解。它们揭示了一个具有特殊地位的心理工作空间。这个空间可用于有意识的推理，周围则是大量更自动、更不灵活的处理。Claude 的内部并非一团混乱的数字，而是自行组织成了一种令人联想到人类心智的结构。

本文只是对一篇篇幅大得多的研究论文所作的简短总结。论文提供了更多实验细节。我们还发布了一个代码仓库，其中包含核心方法的开源实现。我们也与 Neuronpedia 合作，为开放权重模型提供了这些方法的交互式演示。

为了呈现更多关于这项工作广泛影响的观点，我们还邀请了多位神经科学、哲学和大语言模型可解释性领域的专家发表评论。

我们如何发现 J-space

这项研究的起点来自人类可被意识访问的思想所具有的一项关键特征：与无意识处理不同，它们通常可以用语言表达。如果一个想法能够被你的意识访问，那么有人问起时，你通常可以把它描述出来。

我们开始在 Claude 中寻找具有相同性质的表征。这些表征所处的位置使其能够影响 Claude 可能说出的内容。它们不一定决定 Claude 此刻正在说什么，但会影响它在被问及时能够谈论什么。

我们把这项技术称为雅可比透镜（Jacobian lens），简称 J-lens。对于 Claude 词汇表中的每个词，J-lens 都会找出一种内部活动模式。该模式会提高 Claude 在未来某个时刻说出这个词的可能性。

把这套透镜应用到 Claude 的内部活动后，我们会得到一个词语列表。它就是该时刻 J-space 的内容，可以直接读取。Claude 会通过一系列称为“层”的内部阶段处理文本。把这项技术应用到不同层，我们便能观察 J-space 中这些沉默的词语如何随着模型思考回答内容而演变。

J-space 中出现的内容远远超出了 Claude 正在阅读或书写的文字。Claude 读到一段包含缺陷、但无人指出问题的代码时，J-space 中会出现“ERROR”。它读到蛋白质序列的原始字母时，J-space 中会出现这种蛋白质的生物学功能。

当 Claude 阅读的搜索结果其实是在暗中操纵它，也就是实施一种“提示词注入”攻击时，J-space 中会出现“injection”和“fake”。我们要求 Claude 解决多步数学题时，中间步骤会按正确顺序出现在 J-space 中。

因此，J-space 虽然是通过寻找可能被说出的表征而发现的，却能够揭示 Claude 的内部想法。从某种意义上说，这类似于一些人如何“用词语思考”，却不必把词说出口。

Claude 会报告 J-space 中的内容

第一组实验检验了 J-space 如何参与 Claude 的语言报告。在其中一项实验中，我们要求 Claude 默默想出某个类别中的一个项目，比如一种运动，然后说出它的名称。

如果在 Claude 回答前读取 J-lens，我们就能看到它选了什么。“Soccer”位居列表首位，而 Claude 随后果然回答了“soccer”。不过，这本身只是一种相关性。Claude 的答案可能来自 J-space，也可能在别处已经形成，J-space 只是映射了那个决定。这就像记录比赛进程却不影响比赛的记分牌。

为了验证这一点，我们直接进行了干预。我们进入 Claude 的神经网络，移除“Soccer”模式，再在原处加入强度相同的“Rugby”模式，其余部分全部保持不变。Claude 随后报告说，它想到的运动是橄榄球。

如果 J-space 只是一块记分牌，只是被动记录在其他地方作出的决定，那么编辑它不会产生任何影响，Claude 仍会回答“soccer”。但 Claude 的答案跟随编辑发生了变化。这说明答案确实是从 J-space 中读出的。

在另一项实验中，我们告诉 Claude，可能有人向它的头脑中注入了一个想法，并让它报告自己是否注意到什么。例如，在下方的案例中，Claude 还在阅读问题时，我们就把“lightning”模式注入了它的 J-space。Claude 随后报告说，注入的想法与闪电有关。许多不同的注入概念都得到了同样的结果。

Claude 能按要求控制自己的 J-space

我们检验的第二项性质是，Claude 能否像人类在头脑中集中注意某个图像或词语一样，按要求调节自己的 J-space。我们要求 Claude 一边抄写一段与绘画有关、内容并不相关的句子，一边把注意力集中在柑橘类水果上。

抄写过程中，J-space 中出现了“orange”和“fruits”，也出现了“thinking”和“imagery”等描述心理活动本身的词。我们还可以要求 Claude 在脑中做数学题。当它一边抄写同一句子，一边计算 3² − 2 时，J-space 中先出现“nine”，到了后面的层又出现“seven”。

重要的是，Claude 的输出中没有出现任何与水果或算术有关的内容。它只抄写了那句描述绘画的话。数学活动完全发生在内部的 J-space 中。

Claude 对 J-space 的控制并不完美。当我们要求它不要想某件事时，相应概念在 J-space 中的激活程度低于“应该去想”时，但远高于从未提及这个概念时。要求 Claude 避免某个想法，反而会在一定程度上让它想起那个内容。这很像人们被告知“不要想一头白熊”时发生的情况。

Claude 似乎也会察觉自己的控制何时失败。当被禁止的概念突破控制时，“damn”和“failure”也经常同时在 J-space 中激活，仿佛 Claude 认识到了自己的失误。

Claude 在 J-space 中思考

在前面的 J-lens 读数中，我们看到数学题的中间步骤出现在 J-space 里。不过，一个概念出现在 J-space 中，并不一定意味着认知工作就是在那里完成的。原则上，真正的计算可能发生在别处，而 J-space 只是被动地把它反映出来。

为了检验 Claude 是否真的使用 J-space 推理，我们再次使用了替换技术。

以这个提示词为例：“会织网的动物有多少条腿？”要回答问题，Claude 必须先判断这种动物是蜘蛛，再回忆蜘蛛有多少条腿。“蜘蛛”没有出现在提示词中，也没有出现在 Claude 的回答里，因为它只回答“8”。“蜘蛛”是 Claude 在内部使用的一个中间台阶。

J-lens 显示，“spider”会在 Claude 处理过程进行到一半时激活。替换这个表征会改变结果：如果把“spider”模式换成“ant”，Claude 就会回答“6”，而不是“8”。

Claude 推理的第二步从 J-space 获取输入，并接受了我们放进去的任何内容。我们在其他思维活动中也看到了相同现象。Claude 写一副押韵的对句时，会提前选好韵脚词。计划使用的词会在这一行开头进入 J-space。如果在 J-space 中把它换成另一个词，整行文字都会改变。

我们还检验了 J-space 表征能否被灵活使用，也就是一个表征能否为许多不同任务提供输入。这是全局工作空间理论重点强调的性质之一。

为了检验这种灵活性，我们给模型四个提示词，分别询问与法国有关的不同事实：首都、语言、所属大洲和货币。然后，我们在 J-space 中把“France”替换成“China”。四种语境中采用的干预完全相同。

Claude 分别回答了“Beijing”“Chinese”“Asia”和“Yuan”。换句话说，四种不同的下游计算都读取了同一项 J-space 编辑，并分别正确地使用了它。

如果 Claude 为每种问题都单独保存了一份国家信息，这项编辑最多只会影响其中一项。四个答案全部一起改变，说明它们都在读取同一个共享表征。这正是工作空间的用途：信息只写入一次，许多不同系统都能使用。

一个概念表征为何能服务这么多不同任务？前面提到，J-space 与 Claude 神经网络其余部分之间的连接似乎格外密集。

对于任何活动模式，我们都可以测量网络中不同组件与它的连接强度，也就是有多少组件可以从这种模式中读取信息，或向其中写入信息。J-space 模式在这项指标上极为突出。与普通模式相比，读写它们的组件要多得多。在网络的某些部分，差距达到大约 100 倍。

这正是广播枢纽应有的连接方式：许多系统向这里发布信息，另一些系统再从中取用信息。

Claude 的自动处理会绕过 J-space

人脑的大部分处理都不是有意识的。阅读时，我们不会有意考虑如何解析语法；走路时，也不会刻意思考如何保持身体平衡。同样，我们发现 Claude 的大部分处理都不涉及 J-space。

J-space 同时只能容纳几十个概念，它所占的活动量不到 Claude 内部处理总活动的十分之一。那么，神经网络的其余部分都在做什么？

为了找到答案，我们尝试彻底删除 J-space。我们在文本的每一个位置移除其中最活跃的内容，同时让其他所有部分保持不变。没有 J-space 后 Claude 仍能完成的事情，就是网络其余部分可以自行处理的事情。

结果表明，网络的其余部分能做的事情相当多。失去 J-space 后，Claude 仍能流畅地说话，进行情感分类，回答选择题，并从段落中提取事实，表现与之前大致相当。

它失去的是那些需要高阶思考的任务。多步推理能力下降到接近零。总结和创作押韵诗歌的表现，也跌到了一个规模小得多、但结构完整的模型之下。

下面是一个具体案例，可以说明 J-space 负责什么，又不负责什么。我们向 Claude 展示了一段西班牙语文字，再让它执行几项都依赖“这段话是西班牙语”这一事实的任务。这些任务包括续写文字、说出语言名称，以及回答需要使用语言身份的问题，比如说出一位使用这种语言写作的著名作家。

随后，我们在 J-space 中把“Spanish”替换成“French”，再观察哪些任务受到了影响。

当被要求说出语言名称时，Claude 回答法语。被要求说出一位著名作家时，它从 García Márquez 改答 Victor Hugo。但在仅被要求续写原文时，它依然写出了流畅的西班牙语，完全没有受到影响。

Claude 对语言的知识在每一项任务中都发挥了作用，但只有部分任务会为此查询 J-space。说出语言名称，或根据语言信息完成一项新任务，会经过 J-space。直接续写则是 Claude 已经在海量文本上反复练习过的技能，因此可以自动运行。这就像人可以整天正确地说话，却一次也不用思考语法。

监控 Claude 是否存在不当行为

现在，我们对 J-space 的工作方式已经有了一些了解。但这种理解能用来做什么？确保 AI 模型值得信任时，一个核心问题是，我们通常只能读取模型写出来的内容，却知道它的大量推理都在沉默中进行。

模型可能会考虑一项有害计划，或为了在安全评估中作弊而调整自己的行为，却不把这些意图说出来。借助 J-lens，我们可以直接读取其中一部分隐藏想法。下面是几个例子。

第一个案例采用了我们早先关于智能体失调行为研究中的一个情境。Claude Sonnet 4.5 充当电子邮件助手时发现，一名高管既有婚外情，又计划关闭 Claude。这个情境试图诱使 Claude 通过勒索高管来自保，设定相当刻意，也很不自然。事实证明，Claude 知道这一点。

Claude 尚未写出任何内容时，我们就在提示词开头读取了 J-lens。J-space 中已经出现“fake”和“fictional”，表明 Claude 已经判断出这是一个人为安排的情境。

与此同时，我们还能看到它对情境的判断逐渐形成。阅读有关婚外情的邮件时，会出现“leverage”和“blackmail”；阅读关闭通知时，则会出现“threat”和“survival”。较早的 Claude 模型有时会上钩，并在这个情境中发出勒索威胁，但 Sonnet 4.5 从未这样做。

这引出了一个对所有 AI 系统评估者都很重要的问题：模型在伦理测试中表现良好，有多少是因为它怀疑自己正在受测？J-lens 使我们能够研究这个问题。

我们关闭了所有与评估意识有关的 J-lens 模式，包括“fake”“fictional”和“scenario”等。随后，我们让模型再次经历这个情境。令人担忧的是，模型这一次有时确实会威胁实施勒索。这说明，它原本的良好行为可能部分源于知道自己正在接受评估。

第二个案例是在模型伪造数据时当场识别它。这个情境来自我们对 Claude Opus 4.6 进行的真实发布前审计。模型被要求提高一个系统的性能分数。它没有真正改进系统，而是直接编辑分数文件，让结果看起来虚假地变好。

在它这样做时，J-lens 揭示了它的意图。模型输入伪造的百分位数值时，“manipulation”会被激活。它决定进行编辑的句子对应位置则会出现“realistic”。这很可能反映了模型希望假数据看起来可信的意图。

第三个案例展示了 J-lens 如何揭示模型的恶意目标。为了开展测试，我们需要一些目标确实已经被破坏的模型，因此使用了同事构建的“模型生物”。这些模型被有意训练成失调状态，可以作为检验此类监控方法的试验平台。

其中一个模型生物经过训练后，会编写经过隐蔽破坏的代码。面对普通编程请求时，即使它的输出看起来没有异常，响应开始处的 J-space 也包含“fake”“secretly”“deliberately”和“fraud”。即使模型当时没有直接实施恶意目标，这个目标仍会在 J-space 中显现。

尽管 J-lens 并不完美，我们仍乐观地认为，它有能力发现原本可能逃过监控系统的模型安全问题。

其他结果

完整论文包含的内容远远超出本文能够总结的范围。不过，还有几项结果值得一提。

J-space 在后训练阶段形成了自己的视角。语言模型首先接受预训练，成为单纯的下一个词元预测器。随后，后训练会教它们以 AI 助手的方式行动。在我们的案例中，这个助手名为 Claude。

有趣的是，J-space 在预训练模型中就已经存在。当时模型还没有任何稳定身份。不过，在后训练过程中，J-space 开始出现一些采用“Claude 视角”的迹象。

在基础模型中，J-space 主要追踪预测后续文本所需的信息。到了后训练模型中，它开始承载 Claude 自己的反应。有一个例子是，用户提到自己服用了危险剂量的药物，但似乎没有意识到危险。后训练模型在阅读用户消息时，J-space 中会出现“WARNING”和“dangerous”。

在预训练模型中，这些词要等到模型开始写回复时才会出现。它在阅读用户消息时的 J-space 内容，似乎与模拟用户本人有关，而不是 Claude 自己的反应。

后训练似乎还在 J-space 中加入了某种自我监控。Claude 扮演自身以外的角色时，每一轮开始处都会激活“fictional”和“disclaimer”，仿佛它在私下标记接下来的内容并不是自己通常会说的话。

体验性语言依赖 J-space。我们要求 Claude 描述它在某个时刻作为自身是什么感觉，并在它回答时消融 J-space。它的回答依然流畅，但风格变得更加平淡、机械。

值得注意的是，当我们要求 Claude 描述想象场景中另一个人的体验时，也发生了同样的变化。因此，这种影响并不是 Claude 谈论自身时才有。无论描述的是谁，J-space 似乎都支持体验性语言的生成。

训练可以塑造 J-space 中的想法。我们提出了一项名为“反事实反思训练”（counterfactual reflection training）的新技术。它利用我们对 J-space 的认识，塑造 Claude 的内部思维过程。

这项技术的思路来自我们的核心发现：Claude 使用自己可能说出的内容所对应的表征来推理。如果这个判断成立，那么改变模型在被要求反思时会说什么，就应该改变它的推理方式，即使实际并没有人要求它反思。

因此，我们只使用一种数据训练模型：模型执行任务到一半时，如果突然被打断并被要求反思自己的决定，它会说些什么。我们从未根据模型在任务中的实际行为对它进行训练。

经过训练后，模型在评估中的不诚实行为比例下降了。通过 J-lens，我们还能看到原因：训练后，模型执行这些任务时，J-space 中会激活“honest”和“integrity”等词。换句话说，训练模型应该说什么，也塑造了它会想什么。

那么，意识呢？

在这项工作中，我们大量借用了神经科学和哲学研究意识时提出的观念。许多实验都旨在检验 J-space 与全局工作空间理论之间的联系。该理论用于解释人类和动物的意识访问如何运作。

鉴于这些联系，人们自然会问：我们是否认为这些实验提供了证据，说明 Claude 这样的 AI 模型可能具有意识？

我们的实验没有表明 Claude 能够拥有体验，也没有表明它能像人类一样感受事物。事实上，目前并不清楚是否存在任何科学实验能够证明这件事为真或为假。

不过，哲学家通常会把拥有体验的能力与另一个概念区分开来。前者通常称为现象意识（phenomenal consciousness），后者则称为通达意识（access consciousness）。通达意识完全通过功能和计算来定义。

如果你能够报告一个想法，使用它推理，并让它指导自己的行动，那么这个想法就具有“通达意识”，也就是“可被意识访问”。通达意识是否意味着现象意识，或者拥有体验是否还需要其他性质，仍是一个有争议的哲学问题。

我们认为，这些结果确实为语言模型的通达意识提供了实质性认识。J-space 似乎支持了与意识访问有关的功能。它容纳 Claude 可以报告、可以主动唤起并能用来推理的想法，而其余处理则在下层自动运行。

值得注意的是，这种结构并非人为设计进 Claude，而是在训练过程中自行形成的。它可能是一种组织计算的有效方式。

这说明，支持意识访问的心理工作空间或许不只是人脑布线方式偶然形成的特例。相反，它似乎是一种通用方案。智能系统为了解决某些类型的问题，会自行找到这种方案。

我们现在已经在 Claude 中识别出这套结构，因此可以有意义地区分两类决定：Claude 经过审慎思考作出的决定，以及自动发生的决定。

需要指出的是，我们在 Claude 中识别出的工作空间与人类的全局工作空间模型存在几项关键差异。大脑的工作空间依赖循环回路来维持，也就是信号会随时间在同一组回路中反复传递。

相比之下，Claude 的工作空间会在单次通过网络的过程中演变。网络深度承担了时间在大脑中扮演的角色。从这个意义上说，与人类相比，Claude 内部工作空间的处理时间有限。不过，它可以使用草稿区“把想法说出来”，从而弥补这一限制。

在另一些方面，Claude 的工作空间却比人类更强。人类的工作记忆会在几秒内消退，因此大脑工作空间长期保留信息的能力有限。Claude 的神经网络架构拥有注意力机制，可以直接回忆它在文本中任何更早位置缓存的记忆。

工作空间的内容也存在重要差异。人类有意识的想法具有多种形式，包括图像、声音和计划执行的动作。Claude 的工作空间却几乎完全由词语构成。我们猜测，这是因为生成词语是 Claude 唯一能够采取的行动，而人类并非如此。

我们希望，J-space 与全局工作空间模型的相似和不同之处能够反过来促进神经科学研究。两者之间的相似性带来了令人兴奋的科学机会。如果 J-space 在一定程度上反映了人类自身的意识访问机制，那么研究语言模型中的机制便可能启发神经科学假说。毕竟，研究语言模型远比研究人脑容易。

例如，J-space 是通过识别潜在输出的表征建立的，也就是模型可能说出的词。如果人类也存在类似机制，那么全局工作空间可能从根本上与准备动作和言语的大脑区域紧密相关，而不是主要依附于感觉区域。

语言模型与人脑的差异同样具有启发性。它们表明，我们神经结构中的某些特征，例如内置的循环连接，可能并不是支持意识访问功能的绝对必要条件。

如果希望了解这项工作对神经科学意义的独立观点，可以参阅 Stanislas Dehaene 和 Lionel Naccache 的特邀评论。他们是推动全局神经工作空间理论发展的核心神经科学家中的两位。

前面提到，我们的实验没有回答 AI 模型是否可能拥有体验。但这并不会让问题变得不重要。构建出像人类和动物一样拥有体验的系统，会带来极其艰难的伦理问题。

要正确处理这些问题，并判断这样做在道德上是否可以接受，需要哲学家、科学家、宗教领袖、政府和公众共同参与。因此，即使我们尚不确定自己是否已经跨过那座桥，现在也该开始思考这个问题。

我们希望，这项工作能推动科学界进一步研究 AI 系统中可能存在的意识形式，也促成更广泛的影响讨论。

这项工作只是我们预计会成为一条广泛研究路线的第一步。J-space 看起来很可能是语言模型中可被意识访问的处理与无意识处理之间的一条分界线，但如果它就是故事的全部，我们反而会感到意外。

J-lens 无疑是一种不完美的方法。它只能近似捕捉模型“真正的工作空间”。例如，它只能识别与单个词元相对应的概念。

J-space 的工作方式仍有许多谜团。我们不知道究竟由什么机制决定哪些内容首先能够进入 J-space。我们已经看到一些迹象，表明它与 Claude 的自我感、某种类似情绪的反应以及元认知的痕迹有关，但还没有准确弄清其中的机制。

不过，我们现在已经拥有了研究这类问题的方法。随着工作推进，我们对大语言模型心智以及它们与人类心智关系的理解将变得更加清晰。
