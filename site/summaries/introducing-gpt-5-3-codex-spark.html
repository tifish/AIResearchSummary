GPT‑5.3‑Codex‑Spark：把"快"做成一种新的编码模式

来源 OpenAI ｜ 2026-02-12 ｜ 分类 Product

一句话总括：这次发布的真正贡献不是又一个更聪明的编码模型，而是把"交互延迟"提升为与智能同等重要的产品维度——一个跑在 Cerebras 晶圆级硬件上、超过 1000 tokens/秒 的小模型，让 Codex 第一次拥有"实时协作"这一档，与既有的"长时间自主跑任务"互补。

定位Codex 现在有了两种节奏

过去 OpenAI 的前沿模型擅长长时程任务——可以连续自主工作数小时、数天甚至数周。Codex‑Spark 走的是相反方向：专为当下的即时编辑而生——做定点修改、重塑逻辑、调整界面，并立刻看到结果。你可以在它工作时随时打断、改变方向，像结对一样高速迭代。

原有模式 · 长时程
自主跑数小时/数天，适合"交给它去办"的大任务。
新增模式 · 实时
近乎瞬时响应，默认轻量：只做最小化定点修改，不主动跑测试（除非你要求）。

结论：编码助手不再只比"能不能办成大事"，而是开始区分"什么时候该慢、什么时候该快"，并把选择权交还给开发者。

速度1000+ tokens/秒，靠 Cerebras 而不只是靠小模型

Codex‑Spark 是 GPT‑5.3‑Codex 的小型化版本，运行在 Cerebras Wafer Scale Engine 3 这一为高速推理定制的加速器上，给 Codex 接入了一条"延迟优先"的服务层。OpenAI 强调 GPU 仍是训练与推理的基座、提供最具性价比的 token，而 Cerebras 专攻"对延迟极度敏感"的工作流，两者甚至可在单个任务里组合以达到最佳表现。这也是 1 月宣布的 Cerebras 合作的首个里程碑。

基准能力够强，耗时却只是大模型的一小部分

在两个评估智能体编码能力的基准——SWE‑Bench Pro 与 Terminal‑Bench 2.0——上，Codex‑Spark 表现强劲，而完成同样任务的耗时只是 GPT‑5.3‑Codex 的一小部分。OpenAI 给出的耗时口径很具体：等于"输出生成时间 + 预填充时间 + 工具执行总时间 + 网络开销总和"，即把真实端到端体验都算进去，而不仅是采样速度。需要诚实指出：原文以图表呈现分数，文字部分未给出具体数值，故本页不臆造任何 benchmark 数字。

结论：对实时编码来说，"单位时间内能办成多少事"比"峰值智能分数"更贴近真实生产力——这正是它选择优化的目标。

工程提速一半来自模型，另一半来自重写管道

训练过程中 OpenAI 发现：光让模型快还不够，整条请求-响应链路都得提速。于是他们引入持久 WebSocket 连接、重写推理栈、优化会话初始化与 Responses API。成果惠及所有模型：

优化项降幅
每次客户端/服务器往返开销-80%
每 token 开销-30%
首 token 时间（TTFT）-50%

WebSocket 路径已对 Codex‑Spark 默认开启，并将很快成为所有模型的默认。

结论：当模型本身越来越快，瓶颈就转移到管道；"端到端延迟"开始成为和模型架构同等重要的工程战场。

可用性 & 安全预览期独立额度，安全评估未触高能力阈值

即日起以研究预览形式向 ChatGPT Pro 用户开放（最新版 Codex app、CLI、VS Code 扩展），并对少数设计合作伙伴开放 API。因运行在专用低延迟硬件上，它有独立的速率限制、不计入标准额度，但高需求时可能限流或排队。安全方面，Codex‑Spark 沿用主力模型的安全训练（含网络相关训练），评估认定其在网络安全与生物领域均未达到 Preparedness Framework 的高能力阈值。

当前局限：纯文本、128k 上下文、预览期可能限流。OpenAI 表示这是"超快模型家族"的第一款，后续将带来更大模型、更长上下文与多模态输入。

结论：先用受控的小模型预览跑通体验与硬件产能，再逐步放大，是一条务实的"延迟优先"上线路径。

最值得带走的洞见

  1. 竞争维度在变。编码 AI 的较量从"谁更聪明"扩展到"谁的交互闭环更快"——延迟被正式当成可优化目标。
  2. 速度成了基础设施。把 Cerebras 晶圆级硬件接入主流服务栈、并可与 GPU 组合，说明"低延迟推理"正变成一种可独立采购的能力层。
  3. 端到端比模型更关键。往返开销 -80%、TTFT -50% 来自重写管道而非模型本身，且惠及全部模型——瓶颈已从算力转向链路。
  4. 两种模式将融合。未来 Codex 可一边与你紧密实时协作，一边把长任务委派给后台子智能体或并行扇出，不必预先二选一。
  5. 读基准先读口径。耗时被定义为含工具执行与网络开销的真实端到端时间，这种诚实的测量口径比一个孤立的速度数字更可信。
