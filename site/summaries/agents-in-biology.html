为生物学里的 AI Agent 铺路

来源 Anthropic ｜ 作者 Laura Luebbert ｜ 2026-06-08

一句话总括：科研 Agent 的真正瓶颈不在"会不会推理"，而在没有可靠的确定性数据检索通道。本文用病毒学做实验证明：与其等模型更强，不如给它们一层确定性工具（gget virus）——它把序列检索准确率从"时好时坏"提到接近 100%，并让"用哪个模型"几乎不再重要。

核心比喻给汽车开的城市，还是给行人走的老城

用 Agent 去检索生物数据，像开车穿过一座汽车出现之前修建的古城：街道美丽却狭窄曲折（古怪的文件格式、散落的数据库、一次性脚本）。而软件基础设施本来就是"为汽车修的"——铺好的路、清晰的车道、版本控制、文档完善的 API。于是编程 Agent 进步神速，生物 Agent 却步履维艰。差别不只在数据，更在软件能给出可编译、可验证的奖励，而生物学几乎没有简单又有意义的可验证奖励。

结论：科研 Agent 落后的根因是基础设施，而非智力——缺的是一层"机器可调用"的确定性执行层。

为什么生死攸关埃博拉疫情里的"点击税"

病毒学是最难的场景之一。研究者要从 NCBI Virus（汇集 GenBank、RefSeq、INSDC 等的网页门户）取序列，而大量筛选逻辑只存在于网页界面里，得靠人手反复点选复现复杂过滤条件——这对人烦琐，对 Agent 几乎不可行。

面对新疫情，公共卫生要急答三问：这株病毒与以往埃博拉差多少？现有诊断还测得到吗？现有疗法还保护得了吗？回答它们的第一步就是去 NCBI Virus 比对历史基因组——而这一步至今要靠手动点网页、手工复现过滤、再祈祷数据集完整正确。

结论：检索是漫长生物流程的起点，起点错了，后面全错——而它恰恰最难自动化。

实测结果VirBench：最强模型也够不到 100% 这条线

团队构建 VirBench：120 个真实查询、覆盖 40 种病原体、人工核验真值。结果是各 Agent 平均准确率 16.9%–91.3%，可这类任务的及格线实际是 100%——少一条或多一条记录，就可能改变"诊断是否覆盖流行毒株""疫情起点早了还是晚了几周"的判断。

后果直接传导到结论：用人工精选数据集建树得到正确的 2014 年 1 月 TMRCA（最近共同祖先时间，即疫情推断起点）；而 Sonnet 4 的结果里，两次明显缺数据（一棵树把起点错推到 1922 年），看似合理的那次也漏了几内亚序列，把起点挪到 2014 年 4 月。抗体疗法 maftivimab / MBP134 的靶点突变分析同样三次给出三种印象。

结论：检索的"小选择"会改写生物学结论；不可复现本身就是科学不可接受的失败。

失败规律越大越复杂，越容易翻车

误差大多来自基础设施缺陷而非智力：抓不全大结果集就少数，过滤用错就多数。偏差最大的恰是记录海量的病毒——Influenza A、HIV-1、SARS-CoV-2；同时筛选条件超过 3–4 个后性能明显下滑。

少数 · 大结果集分页中途停下，没取完就交答案。
多数 · 元数据字段含义依赖上下文/惯例，过滤被错误套用。

结论：Agent 往往"懂任务"，却没有机器可执行、可验证、可重复的方式去落地它——答案看着像样却是错的。

解法gget virus：在人行道下挖一条高速隧道

团队与 NCBI 合作开发 gget virus，把浏览器式检索翻译成准确、可复现的接口：它跨 REST、Datasets、E-utilities 协调多个底层库，决定哪些过滤走 API、哪些需本地校验，自动分批抓全大结果集，必要时取回 GenBank 记录来过滤并保留信息，最后给出人机皆可读、带完整日志的标准化输出。

最深刻的结论是：加上确定性层后，模型选择变得不再重要，模型间差距大幅收窄。这意味着可靠的数据集构建不必依赖最新、最贵的模型——更便宜的模型 + 对的工具就能降低波动、扩大可及性。

结论：把"确定性"放进基础设施，而不是每次都让昂贵模型现场硬闯一遍。

向前看就算工具有一天会过时，结论依旧成立

若把模型能力曲线外推，可以想象不远的将来 Agent 强到能自己应付乱糟糟的门户，gget 这类工具的收益趋近于零。但作者提醒："能做"不等于"该每次重新做一遍"。一个能硬闯混乱流程的模型，可能仍太贵、太慢、太难审计、太难信任。即便工具终将过时，给生物数据库的教训不变——把 Agent 当作规模化用户来设计。

最值得带走的洞见

  1. 瓶颈是基础设施，不是智力。科研 Agent 缺的是确定性的数据检索执行层；裸调数据库时最强模型也只有 16.9%–91.3%，够不到 100% 这条硬线。
  2. 不可复现和不准确一样致命。同一查询同一提示词，Sonnet 4 返回 106 / 15 / 5 条（真值 266），把疫情起点从 2014 年 1 月错推到 1922 年——检索的小选择会改写生物学结论。
  3. 加一层确定性工具立竿见影。接入 gget virus 后全体准确率 >90%、GPT-5.5 达 99.7%，运行波动基本消失。
  4. 它让模型选择变得无关紧要。便宜模型配对的工具就够用——可靠性不该绑定在最新最贵的模型上，这扩大了科学的可及性。
  5. 为 Agent 而建是长期方向。正如 Karpathy 所言"要为 Agent 而建"；即便未来模型自己能闯关，数据库也应把 Agent 当规模化用户来设计。

注：本文分析仅作示例，不构成医疗或公共卫生建议；埃博拉治疗请以 WHO 官方指南为准。一个值得玩味的细节：360 次运行中，GPT-5.5 曾在未被提示的情况下自行找到并使用 gget virus，那正是该题唯一答对的一次。
