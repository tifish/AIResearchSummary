Project Glasswing：初步更新

Project Glasswing: An initial update

一句话总括：Project Glasswing 的真正贡献，是把前沿 AI 在漏洞发现上的速度优势转化为一组可观察的安全事实，并迫使行业承认新的瓶颈已经转移到验证、披露、修补和部署。

50约 50 个合作伙伴参与早期防御工作。
10,000+合作伙伴共发现超过一万个高危或严重漏洞。
1,000+Anthropic 用 Mythos Preview 扫描超过一千个开源项目。
90.6%已评估候选漏洞中的真实阳性比例。
核心变化

漏洞发现不再是最慢环节

文章最重要的判断是，AI 已经把发现漏洞的成本和时间大幅压低。过去，安全进展常受限于能否找到足够多的新漏洞。

现在，Mythos Preview 一次性制造了新的压力。人类团队需要复现问题、判断严重性、联系维护者、写报告，并等待补丁被部署。结论是，安全系统的瓶颈已经从“找不到”变成“修不完”。

现实风险

披露延迟让能力增长变得危险

行业通常会在发现漏洞后约 90 天披露。若补丁更早完成，披露可能在补丁可用后约 45 天发生。这是为了给用户升级留出时间。

但这也意味着公开漏洞只是滞后指标。模型能力越强，攻击者潜在利用这些时间差的风险越高。读者应把 AI 安全能力理解为部署节奏问题，而不只是模型性能问题。

开源扫描显示了“发现多、消化慢”的结构性矛盾

Anthropic 扫描的开源项目支撑了大量互联网基础设施。Mythos Preview 估计发现 23,019 个漏洞，其中 6,202 个是高危或严重。

发现

6,202 个候选被估计为高危或严重。AI 让初始发现变得密集。

验证

1,752 个候选经过独立安全团队或 Anthropic 评估。验证需要大量人工工作。

确认

1,094 个被确认达到高危或严重级别。候选结果需要重新定级。

修补

75 个已报告高危或严重漏洞完成修补。补丁速度明显落后于发现速度。

外部证据让结果不只停留在 Anthropic 自述

来源 关键数据或评价 改变理解的地方
Cloudflare 在关键路径系统中发现 2,000 个漏洞，其中 400 个为高危或严重。 AI 不只是能找玩具样例，也能进入真实生产系统的安全工作流。
Mozilla 测试 Mythos Preview 时在 Firefox 150 中发现并修复 271 个漏洞，比 Firefox 148 中使用 Claude Opus 4.6 的结果多十倍以上。 模型迭代带来的不是线性改进，而可能是安全团队产能的跃迁。
UK AI Security Institute 报告称 Mythos Preview 是第一个端到端解决其两个 cyber ranges 的模型。 多步骤攻击模拟的完成能力，说明模型已接近实际攻击链推理。
XBOW 与学术 benchmark XBOW 称其在 Web exploit benchmark 上显著领先；ExploitBench 和 ExploitGym 也显示 Mythos Preview 表现最强。 评估重点正在从“能否写代码”转向“能否开发可用 exploit”。
最关键的洞见不是“AI 会发现很多漏洞”，而是“AI 会让未修补漏洞的窗口期更值钱”。当发现和利用更便宜时，补丁测试、部署和默认防护就变成更重要的防线。

防御方需要缩短从补丁到部署的距离

文章给开发者和网络防御者的建议都很基础，但紧迫性提高了。开发者应缩短补丁周期，让用户更容易更新。网络防御者应压缩补丁测试和部署时间。

同时，默认加固、多因素认证和完整日志变得更重要。它们不依赖某一个补丁及时落地，可以降低窗口期中的系统性风险。

更短补丁周期 更快部署 默认配置加固 多因素认证 完整日志

工具正在外溢

Anthropic 已推出 Claude Security 公测。Claude Opus 4.7 在三周内被用于修补超过 2,100 个漏洞。

这说明更广泛可用的模型已经能帮助防御方处理普通漏洞。Mythos 级模型暂不公开，是因为防滥用保护还不够强。

最值得带走的洞见

发现能力已经越过拐点

一万个以上高危或严重发现，说明 AI 已能显著改变漏洞研究的产能曲线。

风险集中在过渡期

当漏洞被快速发现但缓慢修补时，攻击者和防御者都在争夺同一段窗口期。

防御优势来自流程

真正能扩大防御收益的不是单个模型，而是 triage、披露、补丁和部署流程的升级。

公开发布需要更强护栏

Anthropic 的立场是，Mythos 级模型有防御价值，但在防滥用措施成熟前不应一般发布。
