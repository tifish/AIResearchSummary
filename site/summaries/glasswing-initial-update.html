Project Glasswing：初步进展

Project Glasswing: An initial update

一句话总括：Claude Mythos Preview 已把大规模漏洞发现变成现实，但验证、披露和修补仍依赖有限的人力，网络安全因此进入了一个发现飞快、修复偏慢的高风险过渡期。

能力跃升已有外部证据

多数合作伙伴在一个月内各自发现了数百个高危或严重漏洞。Cloudflare 找到 2,000 个漏洞，其中 400 个达到高危或严重级别。

Mozilla 在 Firefox 150 中发现并修复 271 个漏洞。数量超过使用 Claude Opus 4.6 测试 Firefox 148 时的十倍。英国 AI Security Institute 还报告，Mythos Preview 首次完整攻克了其两套网络攻击模拟环境。

结论：这不只是实验室 benchmark 的提升。模型已经在真实、关键的软件系统中显著提高漏洞发现产量。

瓶颈已经转移

Anthropic 在开源项目中发现了 23,019 个疑似漏洞，其中 6,202 个被初评为高危或严重。

但目前仅约 530 个高危或严重漏洞已向维护者披露，其中 75 个完成修补。高危或严重漏洞平均需要两周才能修复。

结论：稀缺资源不再只是漏洞研究能力，而是人工复现、评级、协调披露和部署补丁的容量。

从模型发现到用户安全，中间仍有漫长的人工作业

01 · 发现

AI 可以低成本扫描大量代码，并生成可复现的攻击路径。

02 · 核验与披露

安全团队必须复现问题、重评严重程度、检查现有修复，再向维护者提交高质量报告。

03 · 修补与部署

维护者设计补丁后，仍需等待软件发布、测试和最终用户安装。

关键洞见：Mythos 级模型同时缩短了发现和利用漏洞的时间，却没有自动消除后续流程中的延迟。这会扩大攻击者可利用的窗口。

短期是风险，长期则可能重塑软件工程

在过渡期内，漏洞会被迅速发现，却只能缓慢修补。软件开发者应缩短补丁周期，并降低用户安装更新的难度。网络防御者也需要加快补丁测试和部署，同时强化多因素认证、默认配置和日志记录。

长期来看，同类模型可以在代码部署前发现缺陷。若工具、流程和维护者容量同步提升，AI 有机会让重要软件比今天安全得多。

真正改变网络安全格局的，不只是 AI 能找到多少漏洞，而是整个软件生态能否以相近速度完成核验、修补和部署。

漏洞发现已跨过规模化门槛约 50 家合作伙伴一个月内发现超过 1 万个高危或严重漏洞，说明能力提升已进入实际生产环境。
高精度并未解决容量问题已核验样本的真实漏洞比例达到 90.6%，但人工复现、评级和协调披露仍然昂贵而缓慢。
防御者需要压缩全部补丁链路只加快漏洞发现并不够。补丁制作、测试、发布和安装的每个环节都要提速。

完整译文

上个月，我们启动了 Project Glasswing。这是一项协作计划，目标是在能力越来越强的 AI 模型被用于攻击之前，保护全球最关键的软件。

此后，我们与约 50 家合作伙伴使用 Claude Mythos Preview，在全球最具系统重要性的软件中发现了超过 1 万个高危或严重漏洞。过去，软件安全进展受限于发现新漏洞的速度。现在，限制因素已经变成核验、披露和修补 AI 所发现的大量漏洞的速度。

本文将介绍 Project Glasswing 启动最初几周以来，我们对这一关键网络安全挑战的认识。内容重点包括 Mythos Preview 性能的早期公开证据，以及扫描数千个开源软件项目的初步结果。我们还会讨论这些进展对当今网络防御者的意义、Project Glasswing 接下来的安排，以及未来发布 Mythos 级模型时的考虑。

早期结果

我们如何介绍 Mythos Preview 的发现

软件行业长期以来的惯例，是在新漏洞被发现 90 天后将其公开。如果补丁在 90 天期限届满前完成，则通常会在补丁可用约 45 天后公开。这种安排让最终用户有时间更新软件，避免攻击者在此之前利用漏洞。我们自己的协调漏洞披露政策也采用了这种方式。

但这也意味着，已经公开的漏洞只能滞后地反映 AI 模型网络能力边界的快速提升。现在，我们还不能详细披露合作伙伴通过 Mythos Preview 发现的全部问题，否则可能让最终用户面临风险。因此，我们会提供能够说明模型性能的示例，以及截至目前的总体进展数据。等 Mythos Preview 所发现漏洞的补丁得到广泛部署后，我们会更详细地分享相关发现。

合作伙伴和外部测试者提供的证据

Project Glasswing 的首批合作伙伴负责开发和维护互联网及其他关键基础设施赖以运行的基础软件。修复这些代码中的缺陷，可以降低众多依赖组织的风险，进而减少数十亿最终用户面临的风险。

一个月后，多数合作伙伴都在各自的软件中发现了数百个高危或严重漏洞。总数超过 1 万个。多家合作伙伴告诉我们，其漏洞发现速度提高了十倍以上。例如，Cloudflare 在关键路径系统中发现了 2,000 个漏洞，其中 400 个为高危或严重漏洞。Cloudflare 团队认为，其误报率优于人工测试人员。

这些结果与外部测试者使用 Mythos Preview 的体验一致，也与近期对该模型开展的其他评估相符：

  • 英国 AI Security Institute 报告称，Mythos Preview 是第一个能够从头到尾完成其两套网络靶场的模型。这些靶场模拟了多步骤网络攻击。
  • Mozilla 在测试 Mythos Preview 时，于 Firefox 150 中发现并修复了 271 个漏洞。这个数字是使用 Claude Opus 4.6 在 Firefox 148 中发现数量的十倍以上。
  • 独立安全平台 XBOW 表示，在其 Web 漏洞利用 benchmark 上，Mythos Preview 相比所有现有模型都有“显著提升”。按相同 token 用量比较，它展现出了“前所未有的精度”。
  • ExploitBench 和 ExploitGym 是两个近期发布的学术 benchmark，用于衡量模型开发漏洞利用程序的能力。两项结果都显示 Mythos Preview 表现最强。我们在 Frontier Red Team 博客中更详细地讨论了这些 benchmark 所揭示的模型能力。

总体而言，我们还看到，经过修补的软件正以更快的速度发布。Palo Alto Networks 最新版本包含的补丁数量超过平时的五倍。Microsoft 表示，他们发布的新补丁数量将在一段时间内“继续增加”。Oracle 发现并修复旗下产品及云服务漏洞的速度，也比过去快了数倍。

Mythos Preview 还证明了自己可以用于其他类型的安全工作。例如，Project Glasswing 的一家银行合作伙伴曾遇到威胁行为者入侵客户电子邮箱并伪造电话的事件。Mythos Preview 协助发现并阻止了一笔 150 万美元的欺诈电汇。

开源软件

过去几个月里，Anthropic 使用 Mythos Preview 扫描了超过 1,000 个开源项目。这些项目共同支撑着互联网的大量基础设施，也支撑着我们自己的许多基础设施。

截至目前，Mythos Preview 在这些项目中发现了 23,019 个疑似漏洞。其中 6,202 个被模型评估为高危或严重漏洞，其余包括模型评估为中危或低危的漏洞。

在这些被评为高危或严重的漏洞中，已有 1,752 个由六家独立安全研究公司之一进行仔细评估，少数则由我们自己评估。其中，90.6%，即 1,587 个，被证明是真实有效的漏洞；62.4%，即 1,094 个，被确认属于高危或严重漏洞。这意味着，即使 Mythos Preview 不再发现任何新漏洞，按照当前人工筛选后的真实漏洞比例计算，它预计也将从开源代码中找出近 3,900 个高危或严重漏洞。这还不包括它为 Project Glasswing 合作伙伴发现的漏洞。需要说明的是，我们打算继续扫描开源代码，因此预计这个数字还会增加。

Mythos Preview 检测到的一个开源漏洞位于 wolfSSL。wolfSSL 是一个以安全性著称的开源密码学库，全球数十亿台设备都在使用它。Mythos Preview 构造了一种漏洞利用方法，攻击者可以借此伪造证书。例如，攻击者可以搭建一个冒充银行或电子邮件服务商的网站。对最终用户来说，这个网站看起来完全合法，实际上却由攻击者控制。该漏洞现已修复，编号为 CVE-2026-5194。我们将在未来几周发布完整的技术分析。

正如前文所述，修复此类漏洞的瓶颈，是人类筛选、报告以及设计和部署补丁的能力。相比之下，使用 Mythos Preview 发现漏洞已经容易得多。我们建立了一个开源漏洞扫描看板，展示漏洞披露流程中的不同阶段，并持续跟踪进展。看板包含各种严重级别的漏洞，并不局限于 Mythos Preview 最初评估为高危或严重的子集。可以看到，进入每个后续阶段的漏洞数量都会急剧下降。这反映出核验和修复每个漏洞所需的大量人力。

我们的漏洞筛选流程非常严格。首先，我们或合作的外部安全公司会复现 Mythos 发现的问题，并重新评估其严重程度。确认漏洞真实存在后，我们会检查是否已有修复措施，再为软件维护者撰写详细报告。我们在这里非常谨慎。开源软件维护本就面临许多常规挑战，而维护者现在还要应对大量低质量的 AI 生成漏洞报告。事实上，多位维护者告诉我们，他们目前面临严重的容量限制。有些人甚至要求我们放慢披露速度，因为他们需要更多时间设计补丁。Mythos Preview 发现的高危或严重漏洞，平均需要两周才能修复。

应维护者要求，我们有时会直接披露漏洞，不再开展进一步评估。目前，我们已经报告了 1,129 个未经核验的漏洞。其中，Mythos Preview 估计有 175 个属于高危或严重漏洞。

据估计，我们目前已向维护者披露 530 个高危或严重漏洞。对于直接披露的问题，该估计采用 Claude 给出的严重程度评估；如果维护者或安全合作伙伴提供了评估，则以他们的判断为依据。此外，还有 827 个已经确认的漏洞等待尽快披露。按照相同方法估计，这些漏洞也属于高危或严重级别。

在已经报告的 530 个高危或严重漏洞中，75 个现已修复，其中 65 个已经发布公开安全公告。补丁数量仍然较少，原因有三点。第一，我们仍处在协调漏洞披露政策规定的 90 天期限早期，预计很快会有更多补丁完成。第二，我们可能低估了补丁数量，因为有些漏洞修复后不会发布公开安全公告。遇到这种情况，我们只能使用 Claude 自行扫描和识别补丁。第三，补丁数量较少也反映了一个真实问题：即使我们的披露速度相对缓慢，Mythos Preview 仍在给早已超负荷运转的安全生态增加压力。

与修复漏洞相比，发现漏洞已经变得容易得多。这种反差构成了网络安全领域的一项重大挑战。如果能够成功应对，我们的软件会比过去安全得多。下面将讨论网络防御者可以采取的一些调整措施。

适应网络安全的新阶段

具备与 Mythos Preview 相近网络安全能力的模型，很快会得到更广泛的使用。软件行业显然需要采取更大规模的行动，以处理这些模型将产生的大量发现。

目前，从漏洞被发现，到补丁完成，再到最终用户广泛部署补丁，往往存在很长的延迟。这为攻击者利用关键软件留下了显著的时间窗口。Mythos 级模型大幅减少了发现和利用漏洞所需的时间与成本，因此也放大了这些延迟带来的风险。最终，Mythos 级模型将帮助开发者在软件部署前发现缺陷，从而构建安全性高得多的软件。但在这个过渡期内，漏洞被快速发现，修补速度却仍然缓慢，这会带来新的风险。

软件开发者和用户现在就应采取行动，减少自身面临的风险。下面的建议并不新颖。许多研究人员，包括 Anthropic 的研究人员，正在研究更好、更持久的解决方案。在此期间，首先要把基础工作做好：

  • 软件开发者应缩短补丁周期，并尽快提供安全修复。合理使用公开可用的 AI 模型可以提供帮助。我们也在开发工具并分享研究成果，以支持这项工作，详情见下文。开发者还应尽可能降低用户保持软件更新的难度。对于仍在运行已知存在漏洞的软件的用户，在可行范围内应更主动地督促其更新。
  • 网络防御者应缩短补丁测试和部署周期。National Institute of Standards and Technology、英国 National Cyber Security Centre 等机构提出的关键控制措施，如今变得更加重要，因为它们不依赖任何一个补丁能否及时发布，也能改善安全状况。这些措施包括强化网络默认配置、强制使用多因素认证，以及保留完整日志以支持检测和响应。

使用公开 AI 模型开展网络防御的工具

许多普遍可用的模型已经能够发现大量软件漏洞。它们可能还无法发现最复杂的漏洞，也不能像 Claude Mythos Preview 那样有效地利用漏洞。即便如此，Project Glasswing 已推动许多其他组织使用这些普遍可用的模型检查自己的代码库。我们正在努力让这项工作变得更容易。

首先，我们已经面向 Claude Enterprise 客户公开测试 Claude Security。该工具可以帮助团队扫描代码库中的漏洞，并生成建议的修复方案。上线三周以来，Claude Opus 4.7 已被用于修补超过 2,100 个漏洞。这个速度高于前文介绍的开源漏洞修补速度，主要原因是企业可以直接修复自己的代码，而开源软件的修复通常需要志愿维护者通过协调披露流程处理。

我们还启动了 Cyber Verification Program。该计划允许安全专业人员使用我们的模型开展合法的网络安全工作，例如漏洞研究、渗透测试和红队测试，同时不受部分用于防止网络滥用的安全限制影响。

现在，我们还将自己和合作伙伴配合 Mythos Preview 使用的工具，按请求提供给符合条件的客户安全团队。我们的目标是减少复杂配置，让用户更容易发挥高能力公开模型的最佳性能。本次发布包括：

  • 我们与合作伙伴共同构建和分享的 skills，也就是用于重复性工作的自定义指令；
  • 一套 harness，可以帮助 Claude 理解代码库、启动扫描 subagents、筛选发现并撰写报告；
  • 一个威胁模型构建器，用于分析代码库、识别潜在攻击目标，并据此确定模型工作的优先级。

Project Glasswing 的合作伙伴 Cisco 最近还将 Foundry Security Spec 开源，帮助其他防御者建立与 Cisco 自用系统类似的评估体系。

支持整个生态

我们已经与 Open Source Security Foundation 的 Alpha-Omega 项目建立合作关系。这将支持该基金会帮助维护者处理和筛选漏洞报告。我们也会继续发布研究成果，探索如何让前沿模型的能力更好地服务于网络防御者。

我们还支持开发了 ExploitBench 和 ExploitGym。这两个新 benchmark 让研究人员能够持续跟踪前沿 AI 模型开发漏洞利用程序的能力。我们在相关内容中进行了进一步讨论。通过 External Researcher Access Program，我们也在支持其他高质量定量 benchmark 的开发。最后，Claude for Open Source 会为维护者和贡献者提供支持。我们还承诺，未来只要 Anthropic 采用某个开源软件包，就会对其进行扫描。

Project Glasswing 的下一步

AI 进步的速度意味着，许多不同的 AI 公司很快都会开发出能力与 Mythos Preview 相当的模型。目前，没有任何公司，包括 Anthropic，已经建立足够强大的安全保障，可以防止这类模型遭到滥用并可能造成严重伤害。这正是我们尚未向公众发布 Mythos 级模型的原因。但这也是我们启动 Project Glasswing 的原因：如果能力相近的模型在缺少此类安全保障的情况下发布，那么世界上几乎任何人利用有缺陷软件的成本和难度，很快都会大幅下降。

Glasswing 可以让最具系统重要性的网络防御者获得不对称优势。不过，现在迫切需要尽可能多的组织加强网络防御。我们希望，普遍可用的模型，以及配套提供的新工具、资源和研究成果，能够帮助这些组织改善自身的网络安全状况。

接下来，我们将与关键合作伙伴合作，包括美国及其盟国政府，把 Project Glasswing 扩展到更多合作伙伴。我们也期待在不久的将来，当所需的、更强大的安全保障开发完成后，通过面向公众的常规发布提供 Mythos 级模型。

越过这些风险之后，一个令人鼓舞的未来正在等待我们：重要代码的安全加固程度将远胜今天，黑客攻击也会少得多。前方仍有许多障碍，但我们依然相信，Project Glasswing 能够帮助我们抵达这样的未来。
