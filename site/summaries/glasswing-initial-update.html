Project Glasswing 初步进展

来源 Anthropic ｜ 2026-05-22

一句话总括：Project Glasswing 上线一个月的真正结论不是"AI 很会找漏洞"，而是瓶颈彻底反转了——用 Claude Mythos Preview（一个因风险太高尚未公开发布的前沿模型），约 50 家伙伴已在全球最关键软件里找出逾一万个高危/严重漏洞。过去"修软件"卡在找漏洞太慢，现在卡在人类来不及验证、披露和打补丁。这篇文章的贡献是用大量一手数据把这个新的攻防失衡讲清楚，并给出防御方该如何提前应对。

核心反转找漏洞变白菜价，修漏洞成了瓶颈

以往网络安全进展受限于"找新漏洞有多快"，现在受限于"验证、披露、打补丁有多快"。Mythos Preview 让发现漏洞变得极其容易，但每一个都需要密集的人力去复现、定级、写报告、设计并部署补丁。一个高/严重漏洞平均要两周才能打上补丁——部分维护者甚至主动请求 Anthropic 放慢披露速度，因为他们没有足够人手设计补丁。

结论：当"发现"被自动化、"修复"仍靠人力，安全生态会被自己人压垮——这是本文最重要的一句话。

伙伴战果找漏洞速度普遍快 10 倍以上

合作一个月，多数伙伴各自在自家软件里找到数百个高/严重漏洞，合计 逾一万个；几家报告找漏洞速度提升了 10 倍以上。Cloudflare 在关键路径系统中找到 2,000 个 bug（其中 400 个高/严重），其团队认为误报率优于人类测试者。外部评测也印证了这一点。

外部背书还包括：英国 AISI 称它是首个端到端攻破其两个网络靶场的模型；XBOW 称其"显著超越所有现有模型"、精度"前所未有"；学术基准 ExploitBench / ExploitGym 均把它列为最强。补丁端也随之放量——Palo Alto Networks 一次发布的补丁是平时的 5 倍多，微软称补丁数将"持续走高"，Oracle 修复速度快了数倍。

结论：这不是单一基准的炫技，而是多家独立机构在真实代码与靶场上交叉验证的能力跃迁。

开源扫描1,000+ 项目，真阳性率 90.6%

Anthropic 用 Mythos Preview 扫描了 1,000 多个开源项目（它们支撑着大半个互联网，也包括 Anthropic 自己的基础设施）。模型估计找到 6,202 个高/严重漏洞（连同中/低危共 23,019 个）。其中 1,752 个高/严重已由 6 家独立安全机构（少数由 Anthropic）复核：90.6%（1,587 个）为真阳性，62.4%（1,094 个）确认为高/严重。据此推算，仅开源代码就将浮现近 3,900 个高/严重漏洞——而且扫描还在继续。典型案例是密码库 wolfSSL 的证书伪造漏洞（已修，CVE-2026-5194），可被用来伪造银行或邮箱的假网站。

结论：每一阶段都断崖式缩水——漏斗陡降处不是模型不准，而是人类工时跟不上。

新阶段"已发现却慢慢修"的危险空窗期

与 Mythos Preview 能力相近的模型很快会被多家公司做出并广泛可得。Mythos 级模型大幅压低了发现与利用漏洞的时间和成本，放大了"漏洞已知、补丁未到"这段时间差里的攻击风险。文章给出的应对都是老建议、但此刻更要紧的"基本功"：

软件开发者
缩短补丁周期、让用户更易更新，并对仍在跑已知漏洞版本的用户更持续地催更。
网络防御方
缩短补丁测试与部署时限；落实 NIST、英国 NCSC 的关键控制——加固默认配置、强制多因素认证、保留完整日志——不把安全押在单个补丁能否及时落地上。

Anthropic 也在降低门槛：Claude Security 公测（Claude Opus 4.7 三周内已修 2,100+ 漏洞）、Cyber Verification Program、把内部用过的扫描脚手架/威胁模型构建器开放给合格客户、与 OpenSSF 的 Alpha-Omega 合作帮维护者处理报告。

结论：防御方的窗口很窄——趁能力还没普及，先把补丁流水线和基础控制做扎实。

为何不公开没有足够防护，就不发布

Mythos 级模型一旦在缺乏防护下发布，几乎任何人都能廉价地利用有缺陷的软件——这正是 Anthropic 至今不公开该级别模型的原因，也是发起 Glasswing 的原因：先让最关键的防御方拿到"不对称优势"。下一步是联合美国及盟友政府扩展伙伴，并在做出足够强的防护后，才考虑把 Mythos 级模型普遍发布。

结论：发布与否被明确绑定在"防护是否到位"上，而非能力是否就绪。

最值得带走的洞见

  1. 瓶颈反转是全文主轴。AI 让"发现漏洞"变廉价，安全的真正约束转移到"验证—披露—修复"的人力端。
  2. 规模是真实的，不是 demo。约 50 家伙伴合计 >10,000 个高/严重漏洞，开源侧复核真阳性率 90.6%，多方独立评测交叉印证。
  3. �amount斗陡降揭示痛点。6,202 → 530 已披露 → 75 已修补：每一级断崖都是人类工时的缺口，而非模型误报。
  4. 它用的是"还没发布的"模型。Mythos Preview 因风险太高未公开；这意味着这套能力很快会被多家复制、廉价普及，留给防御方加固的时间不多。
  5. 防御方现在能做的很具体。缩短补丁与部署周期、强制 MFA、加固默认配置、保留日志——别把安全押在"某个补丁能否及时到位"上。
