Warp Decode：把并行轴从"专家"翻转到"输出"，让 MoE 解码更快也更准

一句话总括：Cursor 发现，在 Blackwell GPU 上做小批量、逐 token 的 MoE 解码时，围绕"输出值"而非"专家"组织内核能同时拿下两件通常此消彼长的事——吞吐 提升 1.84 倍，精度 逼近 FP32 1.4 倍。其本质是认清解码阶段"没有足够共享计算可摊销"，于是删掉了为专家视图服务的所有数据搬运。

01传统 MoE 路径的浪费，在解码阶段被放大

问题不在数学，在数据搬运。

现代 MoE 每个 token 只激活一小撮专家（例如每层从 128 选 8）。围绕专家来收集 token、计算、再重组，在 prefill 和大批量时很划算，因为每个专家的共享计算能摊销组织开销。但解码每次只产出一个 token——八个流水线阶段里有五个纯粹在搬数据、不做任何计算。

解码瓶颈不是算力而是内存带宽与冗余搬运；沿用大批量的内核结构，等于为不存在的"共享工作"反复付费。

02核心改动：一个 warp 只算一个输出值

两个融合内核取代八个阶段。

GPU 以 32 条通道为一组（warp）执行。warp decode 让每个 warp 终身只负责一个输出标量：直接从内存流式读取所需权重，把 8 个路由专家的贡献折叠进一个 FP32 累加器，最后只写一次结果。整个 MoE 计算被压缩成 moe_gate_up_3d_batched 和 moe_down_3d_batched 两个内核。down 内核用 __shfl_xor_sync 蝶形归约（编译为单条 shfl.sync.bfly 硬件指令）在寄存器间交换数据，绕过共享内存，连同步都"烧"进了指令里。

删除阶段不是简单"融合"，而是从并行结构的根部重组——这才让 padding、scatter、combine 彻底消失，而非被合并。

03消除的开销：阶段、缓冲区、跨 warp 协调

不形成 per-expert 批次，也就没有相应的成本。

5→0
删除的纯搬运阶段（含 padding/scatter/combine）
32 KB+
每 token 省下的中间缓冲流量（8×2048×2B 的 per-expert 输出缓冲）

传统路径要把激活复制重排成"专家主序"缓冲，还要为 8 个专家各写一份中间结果再读回归约。warp decode 把这 8 份贡献折叠进 32 条通道的寄存器累加器，直到最后一次单标量写出前，什么都不落到全局内存，腾出的 L2 缓存留给真正决定性能的权重行。

省下的不仅是计算，更是"内存往返"——在带宽受限的解码里，少搬一次数据往往比少算一次更值钱。

04为什么更快：天然"令人尴尬地并行"

每个 warp 完全独立，调度器随便排都对。

每个 warp 只读自己需要的权重行、累加器在私有寄存器、输出写到唯一地址，warp 之间没有任何共享可变状态。于是 GPU 调度器可以任意顺序发射任一 warp：某个 warp 等内存时立刻切到另一个。B200 的 148 个 SM 上有成千上万 warp 在飞，内存延迟几乎被其他 warp 的有效计算完全掩盖。输出维度翻倍就线性翻倍独立 warp 数，无需新增同步。

把工作变成"无边的扁平命名空间"，等于把延迟隐藏交给硬件调度器——这是比手写流水线更可靠的加速来源。

05结果：更快、更准、且贴近硬件极限

在 B200 上跑 Qwen-3 风格模型的实测。

1.84×
解码吞吐提升（各上下文长度区间均平坦）
1.4×
输出更接近 FP32 全精度参考
3.95 TB/s
实测带宽（B=32），达 6.8 TB/s 峰值的 58%

精度提升来自全程保持 BF16 激活、FP32 累加，省掉了 BF16↔MXFP8 量化往返带来的、会逐层累积的舍入误差。正确性也很硬：最小余弦相似度 >0.999996，最大绝对误差 0.001953。剩下到峰值的差距，主要是专家路由造成的随机访问（一个 token 可能命中 5、8、14、19 等不相邻专家）带来的内存延迟。

"更快"常以"更糙"为代价；warp decode 反而因删掉量化步骤而更准——这是同时改善性能与精度的罕见内核。

06边界：它不是万能替代品

解码场景的专用武器，而非通用方案。

prefill 和大批量推理仍然适合专家中心式打包，因为大量 token 共享同一专家，组织成本能被足够多的真实计算摊销。warp decode 只在"每个专家共享工作不足以摊销开销"时取胜——而这正是 MoE 解码的常态。它服务于更快迭代 Composer：预训练数据和 RL 决定模型输出的质量，而 warp decode 这类推理投入决定这些输出多快、多准地到达开发者手中。

最值得带走的洞见

  1. 瓶颈决定结构。大批量推理的"专家中心"结构在逐 token 解码里大半是无用搬运——优化第一步是认清当前负载的真实瓶颈，而非沿用既有范式。
  2. 翻转并行轴是关键杠杆。从"warp 算专家"改为"warp 算一个输出值"，一举消除了填充、scatter、combine 和两个中间缓冲，把八阶段压成两个融合内核。
  3. 减少内存往返胜过减少计算。带宽受限场景下，让数据"直到最后一刻才落全局内存"释放出的 L2 缓存，是 1.84× 加速的主因。
  4. 独立性即可调度性。每个 warp 完全无共享状态，使内核"令人尴尬地并行"，把内存延迟隐藏交给硬件调度器，并随维度线性扩展。
  5. 快与准可以兼得。去掉 MXFP8 量化往返后，warp decode 既提速又把精度拉近 FP32 1.4 倍——推理工程的投入直接转化为模型迭代速度。
