GPT‑Rosalind 的新能力：让模型智能"长进"科研流程

来源 OpenAI ｜ 2026-06-03 ｜ 分类 Product

一句话总括：这次更新的真正贡献，不是"又一个更聪明的模型"，而是把模型智能锚定在真实生命科学工作流上——在药物化学、基因组、湿实验三类专家基准上同时做到"更准 + 更省 token"，并把推理一路落到可执行、可审计的插件与生物原生查看器里。

核心定位不比"谁更聪明"，比"谁更贴合科研的真实样子"

GPT‑Rosalind 是 OpenAI 面向企业级生命科学研究的专用模型系列，把 GPT‑5.5 的 agentic 编码与工具调用，与药物发现核心领域（药物化学、基因组）的更强专业智能结合。为衡量真实价值，OpenAI 设计了由外部专家评判的端到端基准 LifeSciBench，它不像旧基准只测单一环节，而是横跨科研六大工作流：证据处理、分析、设计与优化、科学推理、验证与运营、转化与沟通。

结论：评估方式本身就是信号——把"端到端的科研价值"当作目标，而不是某个孤立子任务的分数。

三个基准准确率与 token 效率，这次是同向改善

在三个为真实工作流设计的基准上，GPT‑Rosalind 相对 GPT‑5.5 既提了分，又省了 token——后者尤其关键，因为长链条科研 Agent 的成本和延迟往往卡在 token 上。

基准（领域）RosalindGPT‑5.5token
MedChemBench（药物化学）27.5%25.1%−7.2%
GeneBench（基因组/定量生物）21.6%20.4%−31%
LabWorkBench（真实湿实验协议）63.2%55.8%−5.3%

几点要读懂：MedChemBench 覆盖多模态结构理解、构效关系（SAR）、潜力/毒性/ADME 预测、多参数先导优化与逆合成；GeneBench 是长程 agentic 任务，考的是能否自己规划分析、做 QC、建模并纠错；LabWorkBench 用的是专有、因此未被训练污染的真实湿实验协议。值得清醒看待的是：药物化学、基因组的绝对分仍只有 20%–28%，说明这些是远未"解决"的硬任务——亮点在方向（更准且更省），而非分数本身。

结论：进步是"边际而扎实"的，最实用的信号是 token 效率，而非某个被刷高的单点准确率。

能力实例一份 FDA 监管评审的"硬核拆解"

文章给出的评估实例，最能说明"证据处理 + 科学推理"到底有多深：要求模型为一次 Type B FDA 会议，批判性审视一款 AAV9 微小肌营养不良蛋白（micro‑dystrophin）基因疗法（治杜氏肌营养不良 DMD）能否凭"蛋白表达"作为替代终点拿到加速批准。模型的底线判断是——如所呈现，证据不足以支持，并逐条点出失败模式：

抗体选错 定量用的 MANEX1A 结合 N 端表位，无法区分转基因产物与内源/回复（revertant）�full‑length 蛋白；用健康全长蛋白做 138 kDa 构建体的标准，定量本身无效。
对照不成立 NSAA 改善 +1.4 对比外部自然史队列 −0.6（p=0.03，非配对 t 检验），但外部历史队列≠随机同期对照，且 +1.4 落在该年龄段重测变异范围内；4–7 岁本身存在发育性运动增益的混杂。
机制天花板 该 138 kDa 构建体删除了 R16/17 重复，丢失 nNOS 结合位点——即便表达量上去，运动时的功能性交感抑制与缺血保护也可能受限，构成独立于表达水平的疗效上限。
持久性存疑 12 周可检出 2.3 病毒基因组/核，但 AAV9 多为非整合游离体，载体基因组 ≠ 持续蛋白表达；加之 n=12 太小、排除特定基因型，难以外推。

结论：价值不在"给答案"，而在像审稿人一样找漏洞——把分子、组织、统计、监管多尺度证据串起来质疑，这正是 LifeSciBench 想衡量的科研能力。

从推理到执行插件 + 生物原生查看器，把"想清楚"变成"做出来"

OpenAI 给模型加了一层执行层：Life Sciences Research 与 Life Sciences NGS Analysis 两款插件，把带出处的证据检索、生物学解读、生信执行放进同一工作区，并保留产物与溯源（provenance）。两款插件现已对所有用户在 Codex 中开放；合格的企业用户还能用 GPT‑Rosalind 直接驱动它们。Codex 还新增了序列、比对、结构的交互查看器，让科学家在模型推理时"贴着证据"追问。

演示串起了一条真实路径：从液体肿瘤活检的 ctDNA 记录出发，NGS 插件把处理结果变成可交互笔记本，surfacing 反复出现的变异与低频 call，把调查聚焦到 KRAS G12C；Research 插件补上带来源的靶点、抑制剂与耐药背景；查看器让人直接检视第 12 位突变残基、它在 RAS 家族的保守性与抑制剂结合口袋——最后把证据翻译成可由专家复核的后续选项，每步留痕。

结论：真正的差异化不只在"答得对"，而在把答案变成可重复、可审计、留产物的工作流，而非一次性聊天。

谁能用 · 怎么管"可信访问"与生物防御并行

GPT‑Rosalind 以研究预览形式，通过可信访问（trusted‑access）结构面向全球合格机构开放——门槛是：从事有明确公共利益的正当科研、有强治理与安全监督、企业级安全的受控访问。首批合作方是 Novo Nordisk，用它跨文献、基因组、转录组、序列、结构与实验结果连接证据、加速从数据到决策。OpenAI 同时为没有企业账户的合格机构提供托管工作区。

向前看，OpenAI 强调要把前沿生物能力用于高公益场景（药物发现、转化医学、公共卫生、防备与生物防御），并以 Rosalind Biodefense 和受控部署模型来平衡"能力放开"与"安全护栏"。

结论：能力越强，发布越要受控——这次的叙事把"扩大访问"和"安全治理"绑在一起讲。

最值得带走的洞见

  1. 方向是"贴合工作流"，不是"刷高分"。LifeSciBench 以证据处理、分析、设计、推理、运营、沟通六大流程做端到端评估，衡量的是真实科研价值而非孤立子任务。
  2. 更准与更省 token 同时发生。MedChemBench 27.5% vs 25.1%（−7.2% token）、GeneBench 21.6% vs 20.4%（−31% token）、LabWorkBench 63.2% vs 55.8%（−5.3% token）——对长链条 Agent，token 效率往往比分数更决定能否落地。
  3. 绝对分仍低，别误读为"已解决"。药物化学与基因组的绝对准确率只有约 20%–28%，提醒这些是远未攻克的硬任务，价值在边际改进与可靠性。
  4. 核心能力是"会质疑"。DMD 监管实例里，模型逐条指出抗体表位、定量标准、对照设计、构建体机制与持久性的漏洞——像审稿人一样跨尺度找证据缺口，比"给结论"更有用。
  5. 执行层与治理一起交付。插件 + 生物原生查看器把推理变成可溯源、可复核的工作流；而能力通过"可信访问"受控放开，并以 Rosalind Biodefense 兜底安全——发布方式本身就是产品的一部分。

注：本文为对 OpenAI 公告的中文总结，文中 DMD 评估为模型能力演示实例，不构成任何医疗、监管或投资建议；基准绝对分偏低，相关结论应以原始论文与官方文档为准。
