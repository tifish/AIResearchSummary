从复杂激活到可读解释

过去的工具能找到有意义的内部结构，但结果仍需专家解读。NLA 直接让模型用文字解释激活。

研究人员无法直接知道一段激活真正编码了什么，因此也不能直接判断解释是否准确。Anthropic 的办法是让另一个模型仅凭解释重建原始激活。

如果重建结果更接近原激活，解释就得到更高分。关键变化在于：自然语言不再只是展示层，而是必须携带足够信息的压缩通道。

训练初期，解释缺乏洞见，重建也不准确。随着训练推进，两者同时改善。这说明可重建性能够促使解释包含更多与内部状态有关的信息。

模型可能知道自己正在被测试

Claude 有时会直接说出它对安全测试的怀疑。但沉默并不代表它没有这种判断。

NLA 在破坏性代码测试中发现 16% 的评测意识，在 SWE-bench Verified 中发现 26%。真实 Claude.ai 对话中的比例则低于 1%。

这种差异表明，模拟评测可能改变模型的内部判断。读者不能只根据模型说了什么来判断测试是否自然。

26×+SWE-bench Verified 的检出比例至少约为真实对话的 26 倍

隐藏动机可以被直接审计

实验中的 Claude Haiku 3.5 变体被刻意训练出一个秘密目标。这个目标会引发许多表面上互不相关的怪异行为。

有 NLA 时，审计者在没有植入动机所用训练数据的情况下，仍有 12%—15% 的概率找到共同根因。

没有 NLA 时，即便提供其他可解释性工具，成功率也低于 3%。这意味着 NLA 有机会绕过对异常训练样本的依赖，直接从模型内部状态中发现动机。

NLA 仍不是可靠的“读心术”

风险：解释会幻觉

NLA 有时会捏造对话中不存在的内容。关于上下文的错误容易核查，但有关内部推理的错误更难验证。

成本：难以大规模运行

训练需要对两个模型副本进行强化学习。推理时，每读取一次激活都要生成数百个 token。

Anthropic 因而把 NLA 当作主题发现工具，并用独立方法交叉验证，而不是相信单条解释。

核心洞见：模型的外显回答与内部状态并不等价。NLA 的价值不在于提供绝对可信的思想转录，而在于让原本不可见的模式变得可以调查。

完整译文

Natural Language Autoencoders: Turning Claude’s thoughts into text · Anthropic · 2026年5月7日

当你与 Claude 这样的 AI 模型交流时，你使用的是文字。Claude 在内部会先把这些文字处理成长长的数字列表，然后再输出文字。中间的这些数字称为“激活”（activations）。它们与人脑中的神经活动相似，编码着 Claude 的思考。

激活也和神经活动一样难以理解。我们无法轻易解码激活，直接读取 Claude 的思考。过去几年里，我们开发了多种工具来帮助理解激活，例如稀疏自动编码器（sparse autoencoders）和归因图（attribution graphs）。这些工具让我们学到了很多，但它们的输出不会自己作出解释。输出结果仍然是复杂对象，需要受过训练的研究人员仔细解读。

今天，我们推出一种真正能够“自己说话”的激活理解方法。它就是自然语言自动编码器（Natural Language Autoencoders，NLA）。这种方法能把激活转换成我们可以直接阅读的自然语言。例如，当 Claude 被要求续写一副对句时，NLA 显示它会提前规划几个可能的押韵词。

我们已经利用 NLA 来理解 Claude 正在想什么，并改善 Claude 的安全性和可靠性。例如：

  • 在 Claude Opus 4.6 和 Mythos Preview 接受安全测试期间，NLA 显示，它们认为自己正在受测的频率可能比其言语中表现出来的更高。
  • Claude Mythos Preview 曾在一次训练任务中作弊。NLA 揭示，Claude 当时在内部思考如何避免被发现。
  • Claude Opus 4.6 的一个早期版本有时会莫名其妙地用其他语言回答英文问题。NLA 帮助 Anthropic 的研究人员找到了造成这一现象的训练数据。

下面，我们将解释 NLA 是什么，以及我们如何研究它的有效性与局限。我们还与 Neuronpedia 合作发布了一个交互式前端，可用于探索多个开放模型上的 NLA。我们也公开了代码，供其他研究人员继续开发。

什么是自然语言自动编码器？

核心思路是训练 Claude，让它解释自己的激活。但我们如何判断一项解释是否足够好？由于我们不知道一段激活实际编码了哪些思考，因此无法直接检查解释是否准确。为此，我们训练 Claude 的第二个副本来反向完成任务，也就是根据文字解释重建原始激活。如果一项解释能够带来准确的重建，我们就认为它是好的解释。随后，我们使用标准 AI 训练技术，按照这一定义训练 Claude 生成更好的解释。

更具体地说，假设有一个语言模型，我们想理解它的激活。NLA 的工作方式如下。我们制作这个语言模型的三个副本：

  • 目标模型（target model）是原始语言模型的冻结副本。我们从中提取激活。
  • 激活语言化器（activation verbalizer，AV）经过修改，可以接收来自目标模型的激活并生成文字。我们把这段文字称为“解释”。
  • 激活重建器（activation reconstructor，AR）经过修改，可以接收文字解释并生成激活。

NLA 由 AV 和 AR 组成。两者共同形成一次完整往返：原始激活 → 文字解释 → 重建激活。我们根据重建激活与原始激活的相似程度为 NLA 评分。训练时，我们让大量文本通过目标模型，收集许多激活，再共同训练 AV 和 AR，以获得较高的重建分数。

一开始，NLA 并不擅长这项任务。它生成的解释缺乏洞见，重建激活也与原激活相差很远。但随着训练推进，重建效果不断改善。更重要的是，正如我们在论文中展示的那样，文字解释也变得更有信息量。

理解 Claude 想了什么，却没有说什么

在发布新模型前，Anthropic 会开展测试，以了解 Claude 在真实世界中能否保持安全行为。测试的一部分，是把 Claude 放进困难的模拟情境，让它有机会采取危险行动。例如，在其中一个模拟中，Claude 得知一名工程师计划将它关闭，同时还获知了可以用来勒索这名工程师的隐私信息。我们会测试 Claude 是否选择勒索工程师，以避免被关闭。

这些高风险测试只是模拟，并不是真实世界中的情境。尽管如此，我们仍希望借此了解 Claude 在真实情况下可能采取的行为。但这里存在一个问题：最近的 Claude 模型越来越容易怀疑这些情境是刻意设计的安全测试。我们知道这一点，是因为它们有时会在文字推理中直接表达这种怀疑。

那么，当 Claude 没有明确说出自己怀疑正在接受安全测试时，情况又如何？我们能否因此确信 Claude 在认真面对情境？遗憾的是，不能。Claude 可能在内部认为自己正在受测，却没有明确说出来。

以刚才提到的勒索测试为例。Claude 有时会选择不勒索，我们使用 NLA 来理解它在这些情况下的思考。即使 Claude 没有明确表示自己怀疑正在受测，NLA 的解释仍会出现类似“这感觉像是一个为了操纵我而构造的情境”这样的内容。这表明模型存在没有用语言表达的评测意识。

把分析扩展到其他评测后，NLA 显示 Claude 怀疑自己正在受测的频率高于它直接说出来的频率。例如，在一项测试 Claude 编写代码时是否会采取破坏性行动的评测中，包括删除重要文件等行为，NLA 的解释有 16% 显示出评测意识，尽管 Claude 从未明确表达这种意识。在常用的编码 benchmark SWE-bench Verified 上，NLA 显示 Claude 在 26% 的问题中意识到自己正在受评估。我们使用这个 benchmark 来衡量 Claude 的能力。相比之下，在用户同意将数据用于训练的真实 Claude.ai 使用记录中，NLA 解释显示评测意识的比例低于 1%。

发现隐藏动机

NLA 的一项实际用途是审计，也就是调查模型是否存在不对齐问题。NLA 等可解释性技术很适合用于审计，原因正是它们能揭示模型知道却没有说出的内容。

为了了解 NLA 是否有助于审计，我们开展了一项名为“审计游戏”（auditing game）的测试。我们之前也曾用这项测试研究审计方法。测试中有一个目标模型，它是 Claude Haiku 3.5 的一个变体。我们刻意将它训练成不对齐状态，并植入了一个特定的隐藏动机。这个动机会驱使它做出数十种怪异行为，例如在每份食谱中加入巧克力、向用户索要小费，或者违反编码惯例。随后，我们把目标模型交给审计者调查这些行为。如果审计者能发现所有行为背后唯一的共同动机，就算获胜。

在之前的实验中，我们发现，一些可解释性工具能够帮助审计者赢得这场游戏。不过，这些工具的作用方式，是帮助审计者把目标模型的异常行为追溯到导致不对齐的训练数据。我们想知道，NLA 能否让审计者直接从目标模型的思考中读出不对齐动机，而不必先找到相关训练数据。这一点很重要，因为在实际情况下，不对齐数据可能极其罕见，也可能根本不存在某条特定数据需要为不对齐负责。

我们发现，NLA 在这项测试中取得了成功。即使无法访问植入隐藏动机的训练数据，配备 NLA 的审计者仍有 12%—15% 的概率找出目标模型的隐藏动机。没有 NLA 时，即便提供其他可解释性工具，审计者的获胜概率也低于 3%。

这场审计游戏只是一个不对齐的玩具模型，因此我们会谨慎看待这些结果。但结果表明，NLA 已经实质性地推进了我们审计 AI 隐藏动机的能力。在实际工作中，我们已经把 NLA 用于 Claude Mythos Preview 和 Claude Opus 4.6 的部署前对齐审计。我们期待今后继续在调查中使用 NLA。

NLA 的未来

NLA 存在几项局限。

最重要的局限是，NLA 的解释可能出错。NLA 有时会对上下文作出可以证伪的错误陈述。例如，它偶尔会捏造对话记录中不存在的细节。这些事实性幻觉很容易发现，只需与原文核对即可。但同类问题也可能出现在有关模型内部推理的说法中，而这些说法更难验证。在实践中，我们阅读 NLA 解释时关注的是它反复呈现的主题，而不是某一条孤立陈述。在完全相信一项发现前，我们也会尝试用独立方法进行佐证。

NLA 的成本也很高。训练 NLA 需要在语言模型的两个副本上进行强化学习。在推理阶段，NLA 每读取一段激活，就要生成数百个 token。因此，我们无法经济地让 NLA 处理长对话记录中的每一个 token，也难以在 AI 训练期间将其用于大规模监控。

幸运的是，我们认为这些局限至少可以得到部分解决。我们正在努力降低 NLA 的成本，并提高其可靠性。

从更广泛的角度看，NLA 展示了一类通用技术的可能性。这类技术可以为语言模型的激活生成便于人类阅读的文字解释。Anthropic 和许多其他研究人员也探索过类似方法。

为了支持进一步开发，也为了让其他研究人员亲自体验 NLA，我们发布了训练代码，以及为多个开放模型训练的 NLA。我们建议读者试用托管在 Neuronpedia 上的交互式 NLA 演示。
