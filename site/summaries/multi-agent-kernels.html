多智能体系统把 GPU kernel 提速 38%

作者：Wilson Lin、Sahil Modi、Yuan Zhang、Edward Lin　|　与 NVIDIA 合作
真正的贡献不是“又快了一点”，而是：一个自主运行的多智能体系统在 3 周 内，从零写出并优化了 235 个 面向 NVIDIA Blackwell B200 的 CUDA kernel——这类工作通常需要资深工程师数月到数年。

为什么拿 kernel 优化当试金石

开放式、有可量化目标、连出题人都不知道最优解。

kernel 优化提供了可迭代测量的目标，而不是去对齐一个已知 diff——这正是检验长时间运行多智能体系统的理想场景。今天工程师把模型拆成单个数学算子逐个调优，虽可控却会因“分段优化”错失跨系统的整体收益。结论：让智能体跳出人工简化、探索更大解空间，是这次实验真正想验证的事。

SOL-ExecBench：题目来自真实生产模型，作弊会被判废

124+ 个开源模型，27 块 B200 做基准。

NVIDIA 用 SOL-ExecBench 从 Deepseek、Qwen、Gemma、Kimi、Stable Diffusion 等 124+ 个生产级开源模型 生成 235 个问题，覆盖 LLM、扩散、视觉、音频、视频与多模态等真实训练/推理约束，而非玩具 kernel。评测在 27 块 B200 上进行，并对照软件基线与硬件理论极限；若智能体用缓存等手段“跑出”超过 B200 物理上限的成绩，结果会被直接判废。结论：这让 38% 的数字更可信——它是被真实硬件和反作弊机制约束过的。

如何运行：一个 markdown 文件协调，全程零人工干预

规划者分发任务，工人按性能指标自我再平衡。

整套协调协议只写在一个 markdown 文件里（规定输出格式、规则与测试）；系统自己学会调用基准流水线，形成“测试—调试—优化”的闭环，开发者全程不介入。为压力测试能力，团队让它用两种处于 GPU 抽象谱两端的语言各跑一遍：CUDA C + 内联 PTX（直达寄存器与 ISA，考低层硬件推理）与 CuTe DSL（公开训练数据极少，考从文档现学新 API）。结论：系统能在两个极端都自洽运行，说明能力不是靠“背过的代码”。

三个案例：面对不同问题，自发选出不同策略

BF16 分组查询注意力 · 带分页预填充

取自 Llama 3.1 8B 的 SGLang 推理，对照 FlashInfer 人工基线。智能体用上专用访存/计算指令、持久化 kernel 调度并按输入尺寸超调，做出 SOL 0.9722（逼近硬件极限）、84% 加速 的方案；替换进 SGLang 后端到端 TTFT 提升 3%。

洞见：注意力仅占预填充的 2–5%，仍换来可观的端到端收益，说明逼近极限的 kernel 有真实落地价值。

NVFP4 MoE 线性层 · 带门控

类 Qwen3 的双 kernel MoE 模式，输入与中间结果量化到 NVFP4（4-bit 浮点）。智能体识别出量化是主瓶颈，融合了缩放与舍入：因为 NVFP4 只有 16 个可能取值，它用预计算阈值桶把 FP32 直接映射到 FP4 编码。最终超越优化版 PyTorch 基线，39% 加速、SOL 0.58。

洞见：真正的算法巧思（“只有 16 个值”）由系统自己发现，而非套用模板。

BF16 矩阵乘法（GEMM）

GEMM 历来是专家专属领域，需要内联 PTX、流水线与分级。系统从零写出专用 CUDA C++ GEMM，达到 NVIDIA cuBLAS 精调人工基线的 86%；在对 LLM 解码尤为关键的 小 M 场景，甚至反超库实现最高 9%。

洞见：在最难的 kernel 问题上，多智能体系统已显露出局部超越领域专家的迹象。

仍有很大上行空间：算力是当前瓶颈

中位 SOL 仅 0.56。

尽管 geomean 加速达 38%，中位 SOL 只有 0.56（0.5 为优化版 PyTorch 基线，1.0 为硬件极限），离极限尚远。团队认为这主要是因为数百个问题和智能体只能挤在 27 块 GPU 上，算力受限。结论：更多 GPU 意味着更深、更新颖的解——这些技术也将很快反哺 Cursor 主产品。

最值得带走的洞见

  1. 速度量级的改变：资深工程师数月到数年的活，自主多智能体系统 3 周完成 235 个 kernel，平均提速 38%。
  2. 真实而非作弊：题目来自 124+ 个生产模型，27 块 B200 实测，超物理极限的结果会被判废——数字经得起检验。
  3. 策略多样性是关键能力：注意力、量化 MoE、GEMM 三类问题各自演化出不同优化路径，包括自发利用“NVFP4 只有 16 个值”的算法巧思。
  4. 开始触碰专家天花板：GEMM 达 cuBLAS 的 86%，小 M 解码场景反超最高 9%，预示在最硬问题上局部超越人类专家。
  5. 瓶颈在算力而非方法：中位 SOL 仅 0.56，受限于 27 块 GPU；更多算力可能解锁更大、更新颖的优化空间——这是多智能体架构走向“默认软件构建方式”的论据。
