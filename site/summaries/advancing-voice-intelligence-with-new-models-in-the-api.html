用 API 新模型推进语音智能

Advancing voice intelligence with new models in the API

发布日期：2026 年 5 月 7 日 来源：OpenAI 类别：产品

一句话总括

三款新模型把实时语音从简单的问答界面，推进为能够理解、推理、调用工具、翻译和转写的工作入口。

变化不只是响应更快

GPT‑Realtime‑2 可以在通话继续进行时推理、调用多个工具，并处理纠正和打断。它还能先说“让我查一下”，让用户知道系统仍在工作。

这意味着语音代理开始具备完整的任务执行能力，而不只是生成自然的声音。

评测提升指向真实工作流

高推理档在 Big Bench Audio 上提升 15.2%。xhigh 档在 Audio MultiChallenge 上提升 13.8%。

提升集中在推理、上下文管理和指令遵循。这些能力直接决定语音代理能否可靠完成复杂请求。

三条语音应用路径

语音到行动

用户说出目标。系统理解约束，调用工具，并完成任务。

系统到语音

软件把航班、日历或客户支持信息转成即时口头指引。

语音到语音

模型跨语言实时传递对话，让交流在任务和语境变化中继续。

企业实测给出了更直观的尺度

Zillow 在最难的对抗 benchmark 上进行提示词优化后，通话成功率从 69% 升至 95%，提高了 26 个百分点。印度市场的 Hindi、Tamil 和 Telugu 评测中，GPT‑Realtime‑Translate 的词错误率比其他受测模型低 12.5%。这些结果说明，生产价值取决于任务完成率、合规性和对区域语音的适应能力，而不只是声音是否自然。

真正的转折点，是语音模型开始一边维持自然交流，一边完成需要上下文、工具和判断的工作。

我们在 API 中推出三款音频模型，为开发者开启一类全新的语音应用。借助这些模型，开发者可以构建更自然的语音体验。应用能够更智能地响应，也能实时采取行动：

GPT‑Realtime‑2：我们首款具备 GPT‑5 级推理能力的语音模型。它可以处理更困难的请求，并自然地推动对话继续。

GPT‑Realtime‑Translate：一款新的实时翻译模型。它可以跟上说话者的速度，把 70 多种输入语言的语音翻译成 13 种输出语言。

GPT‑Realtime‑Whisper：一款新的流式语音转文字模型。它能在人们说话时实时转写语音。

试用 GPT-Realtime-2

启动会话后，可以尝试说出以下任一句话：

  • 我今晚临时要请客吃饭。我只有 30 分钟，两位朋友吃素，其中一人讨厌蘑菇，而且厨房很小。请帮我安排一份简单的菜单。
  • 我要在日本的一场现场活动中欢迎来宾。请用日语说一段温暖、自然的欢迎词，就像主持人在开启一场特别活动。
  • 我的订单号是 Orbit-742Q。请清楚地重复一遍，让我确认是否正确。
  • 请帮我练习告诉团队，我们已经达成发布里程碑。先用沉稳自信的语气说一遍，再用更兴奋的语气说一遍。
  • 我正在为公路旅行准备知识问答。请给我三道看似简单、实际容易答错的问题，然后各用一句话解释答案。

语音正在成为人们使用软件时最自然的方式之一。开车时，用户可以开口寻求帮助。穿过机场时，用户可以调整旅行计划。用户还能用自己偏好的语言获得支持，或在不必停下来打字的情况下完成任务。

但要打造真正有用的语音产品，仅有快速的对话轮转或自然的声音还不够。语音代理需要理解用户的真实意图，持续掌握上下文，并在请求发生变化时恢复。它还要在对话继续时使用工具，并以符合当下情境的方式回应。

本次发布的模型共同推动实时音频摆脱简单的一问一答，走向真正能够工作的语音界面。随着对话展开，它们可以倾听、推理、翻译、转写并采取行动。

语音：连接用户与产品的界面

随着语音逐渐成为更自然的软件使用方式，我们看到开发者正围绕语音 AI 的三种新模式构建产品：

语音到行动（Voice-to-action）：用户描述自己的需求，系统理解并推理这项请求，调用工具，然后完成任务。例如，Zillow 正在构建一款助手。它能够听取请求、进行推理并采取行动，例如：“帮我寻找 BuyAbility 预算范围内的房子，避开繁忙街道，并预约周六看房。”

系统到语音（Systems-to-voice）：软件把已有上下文转化为实时语音指引。例如，旅行应用可以主动告诉旅客：“你抵达本站的航班延误了，但仍然赶得上转机。我找到了新的登机口，规划了穿过航站楼的最快路线。你的行李预计也仍能顺利转运。”

语音到语音（Voice-to-voice）：AI 可以帮助实时对话跨越语言、任务或不断变化的上下文。例如，Deutsche Telekom 正在构建语音支持体验。客户可以使用自己最习惯的语言说话，模型则实时翻译对话。

这些模式也可以结合使用。Priceline 正在为一种未来体验做准备：旅客只需使用语音，就能管理整段旅程。他们可以通过对话搜索航班和酒店，也能处理行程变化。例如，在航班延误后调整酒店预订，或实时查询 TSA 安检等待时间。到达目的地后，他们还可以翻译现场对话。

实时语音：帮助语音模型推理并采取行动

GPT‑Realtime‑2 面向实时语音交互构建。模型可以一边对请求进行推理，一边维持对话。它能调用工具，处理纠正或打断，并以符合当下情境的方式回应。

开场提示语

开发者可以让模型在主要回复前先说一句简短的话，例如“让我查一下”或“请稍等，我正在查询”。这样，用户就知道代理正在处理请求。

并行工具调用与工具透明度

模型可以同时调用多个工具，还能通过“正在查看你的日历”或“现在为你查询”等话语，把这些操作告知用户。代理在完成任务时仍能及时回应。

更强的故障恢复能力

模型可以用更自然的方式应对问题。例如，它会说“我现在处理这件事遇到了一些困难”，而不是无声失败或直接让对话中断。

面向代理式工作流的更长上下文

我们把上下文窗口从 32K 扩展至 128K，以支持更长、更连贯的会话，以及更复杂的任务流程。

更强的领域理解能力

模型能够更好地保留专业术语、专有名词、医疗健康术语，以及生产环境中重要的其他词汇。

更可控的语气与表达

模型能更准确地调整语气。解决问题时，它可以保持冷静。用户感到沮丧时，它可以表达共情。确认操作成功时，它也可以表现得轻快积极。

可调节的推理强度

开发者现在可以选择 minimal、low、medium、high 和 xhigh 五档推理强度，默认值为 low。面对简单交互时，可以用较低延迟换取快速响应。遇到复杂请求时，则可以选择更审慎的推理。

这些进步也体现在与生产级语音代理高度相关的音频评测中。GPT‑Realtime‑2（high）在衡量音频智能的 Big Bench Audio 上，比 GPT‑Realtime‑1.5 高 15.2%。GPT‑Realtime‑2（xhigh）在衡量指令遵循的 Audio MultiChallenge 上高 13.8%。它优于 GPT‑Realtime‑1.5，也展现出更强的推理、上下文管理和实时对话控制能力。

Big Bench Audio 用于评估支持音频输入的语言模型所具备的高难度推理能力。Audio MultiChallenge 用于评估口语对话系统的多轮对话智能，包括指令遵循、上下文整合、自洽性，以及处理自然口语纠正的能力。

实时翻译：构建实时多语言语音体验

GPT‑Realtime‑Translate 可以帮助开发者构建实时多语言语音体验。每个人都能使用自己偏好的语言说话，实时听到翻译结果，同时阅读实时转写文本。它支持 70 多种输入语言和 13 种输出语言，适用于客户支持、跨境销售、教育、活动、媒体，以及服务全球受众的创作者平台。

对开发者而言，实时翻译必须在跟上说话速度的同时保留原意。即使人们采用自然口语、切换上下文，或使用地区发音和领域术语，也必须做到这一点。例如，Deutsche Telekom 正在测试该模型的多语言语音交互能力。更低的延迟和更流畅的表达，可以让跨语言对话更加自然。

实时转写：构建低延迟转写体验

GPT‑Realtime‑Whisper 是一款新的流式转写模型，专为低延迟语音转文字而构建。它在人们说话时转写音频，让实时产品变得更快、更灵敏，也更自然。字幕可以即时出现，会议记录也能跟上对话节奏。

模型让实时语音在发生的同时就能进入业务工作流。团队可以为会议、课堂、广播和活动生成字幕。在对话仍在进行时，他们就能生成记录和摘要。他们也可以构建需要持续理解用户的语音代理，并为客户支持、医疗健康、销售、招聘及其他具有大量语音交互的场景建立更快的后续工作流。

安全

Realtime API 包含多层防护和缓解措施，用于帮助防止滥用。我们会在 Realtime API 会话中运行主动分类器。这意味着，如果系统检测到某些对话违反有害内容准则，就可以中止会话。开发者也可以使用 Agents SDK，方便地加入自己的额外安全护栏。

我们的使用政策禁止把服务输出改作或分发于垃圾信息、欺骗或其他有害用途。开发者还必须明确告知最终用户，他们正在与 AI 互动。只有在上下文已经明显表明这一点时，才可以不再另行说明。

Realtime API 完整支持欧盟数据驻留，适用于基于欧盟的应用，也受到我们的企业隐私承诺保护。
