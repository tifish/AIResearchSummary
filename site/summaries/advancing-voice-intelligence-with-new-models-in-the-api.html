一句话总括

OpenAI 把实时语音 API 的核心目标，从自然对话扩展为可推理、可执行、可翻译、可转写的实时工作流。

这次发布的重点不是单一模型升级，而是把语音入口拆成三类生产能力：GPT‑Realtime‑2 负责复杂语音代理，GPT‑Realtime‑Translate 负责跨语言实时沟通，GPT‑Realtime‑Whisper 负责低延迟语音转文字。

70+ → 13翻译模型支持 70 多种输入语言和 13 种输出语言。
$0.017/分钟GPT‑Realtime‑Whisper 的实时转写价格。

语音代理的变化：从“轮流说话”到“边说边做事”

文章把语音 AI 的新场景分为 voice-to-action、systems-to-voice 和 voice-to-voice：用户可以直接说需求，系统可以调用工具、给出实时引导，或帮助不同语言的人继续对话。

这改变了读者对 AI 影响的理解：语音不再只是输入法替代品，而可能成为软件执行任务的主界面，尤其适合出行、客服、销售、教育和现场活动。

GPT‑Realtime‑2 的核心贡献：复杂请求也能保持对话连贯

GPT‑Realtime‑2 支持短前置语、并行工具调用、工具透明提示、更强失败恢复、专业词汇保持和可控语气，还把上下文窗口从 32K 提升到 128K。

评测上，高推理档在 Big Bench Audio 上比 GPT‑Realtime‑1.5 高 15.2%，xhigh 档在 Audio MultiChallenge 上高 13.8%。这说明实时语音模型的竞争点正在从延迟和音色，转向推理、上下文管理和任务控制。

业务早测信号：可靠工具调用决定能否进生产

Zillow 的早测反馈显示，在最难对抗基准中，经过提示优化后通话成功率从 69% 提升到 95%，也就是 26 个百分点提升。

这个数据的意义在于，企业语音代理不是只要会聊天就能上线；它必须在合规、工具调用和复杂条件筛选中稳定完成任务。

实时翻译的价值：跨语言对话开始接近实时产品能力

GPT‑Realtime‑Translate 支持 70 多种输入语言到 13 种输出语言，并强调在自然语速、口音、上下文切换和专业词汇下保持意思与节奏。

在印度相关评测中，它在印地语、泰米尔语和泰卢固语上的词错误率比其他被测模型低 12.5%。这提示多语言语音 AI 的关键不只是覆盖语言数量，而是能否在真实口音和区域发音中保持可用。

实时转写的影响：语音内容进入业务流程的时间被提前

GPT‑Realtime‑Whisper 面向低延迟流式语音转文字，能在说话过程中生成转写，用于实时字幕、会议记录、客服跟进、医疗记录、招聘和销售场景。

它的产品意义是把“会后处理”前移到“对话正在发生时”，让摘要、检索、行动项和后续流程更早启动。

安全与成本：平台把可控性放进发布叙事

Realtime API 包含会话级主动分类器，检测到违反有害内容规则的对话可被中止；开发者还可以通过 Agents SDK 增加自己的安全护栏，并需明确告知终端用户正在与 AI 交互。

价格上，GPT‑Realtime‑2 为音频输入 $32/100万 token、缓存输入 $0.40/100万 token、音频输出 $64/100万 token；翻译为 $0.034/分钟，Whisper 为 $0.017/分钟。这让开发者可以按任务复杂度区分高推理语音代理、翻译和转写三类成本。
