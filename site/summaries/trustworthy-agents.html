一句话总括

实践中的可信赖智能体

Trustworthy agents in practice

Anthropic 的真正贡献，是把智能体治理从“模型是否安全”推进到“模型、工具、权限、环境和行业标准如何共同约束行动”。

5项可信智能体原则：人类控制、价值对齐、安全交互、透明、隐私。
4个行为层：模型、harness、工具、环境。任何一层都可能放大风险。
约 2 倍复杂任务中，Claude 主动确认的频率大约翻倍。
1 个开放协议 MCP 被捐给 Linux Foundation 的 Agentic AI Foundation。

智能体不是更会聊天的模型，而是会行动的系统

文章把智能体定义为能自行规划、调用工具、观察结果并调整步骤的 AI。它不只是回答问题，而是在任务中持续循环。

这改变了治理重点。风险不再只来自模型输出，还来自它能访问什么、能操作什么、在哪里运行。

模型
提供推理和任务能力，也决定是否能识别危险指令。
Harness
承载产品指令和护栏，例如超过金额阈值必须提醒用户。
工具
连接邮件、日历、代码、文件和业务系统，让模型从“说”变成“做”。
环境
决定数据边界和后果大小。企业网络中的智能体与个人手机中的智能体风险不同。

人类控制从“逐步批准”转向“先看整体计划”

简单任务可以逐项确认。复杂任务会产生几十次操作，反复弹窗会让用户疲劳。

计划

Claude 先拆解目标，列出要做的步骤。

审阅

Plan Mode 让用户在行动前看到整体策略。

修改

用户可以调整计划，而不是只批准单个动作。

执行

智能体按计划推进，并在关键点保留干预空间。

升级

子智能体并行工作后，新的挑战是让用户仍能理解和引导流程。

Prompt injection 的核心威胁，是把恶意指令藏进智能体正在处理的内容里，让它越过原本的目标和权限边界。

安全不能押注单一防线

训练层
让模型识别注入模式，并在不确定时停下来。
生产层
监控真实流量，拦截现实攻击。
评测层
用外部 red team 检验系统，而不是只依赖内部假设。
部署层
客户仍要谨慎选择工具、数据、权限和运行环境。

行业基础设施决定智能体能否被可靠采用

Anthropic 承认，单家公司不能独自解决智能体安全。企业采用需要可比较的测试、可共享的证据和开放协议。

需要建设的部分当前缺口为什么重要
Benchmark还没有标准化方法比较智能体抵抗 prompt injection 和暴露不确定性的能力。没有共同评测，企业和政策制定者就难以判断系统真实风险。
证据共享各家公司掌握自己的使用数据和失败模式，但公开程度不一致。更多真实证据能让政策讨论少依赖猜测。
开放标准工具和数据接入如果各自拼装，安全属性会碎片化。MCP 这样的协议可以把安全设计嵌入基础设施，而不是每次部署后再补丁式修复。

最值得带走的洞见

智能体治理必须看系统。模型很重要，但 harness、工具和环境同样决定最终行为。
自治越强，误解意图的代价越高。可信智能体需要知道什么时候继续，什么时候停下来问人。
用户控制要匹配任务复杂度。Plan Mode 的价值在于让人审查策略，而不是陷入操作级弹窗。
Prompt injection 是结构性问题。开放环境和强工具会扩大攻击面，所以防御必须跨训练、监控、评测和部署。
开放标准会影响竞争方向。如果工具接入协议共享，竞争就更可能集中在智能体质量和安全性上。
